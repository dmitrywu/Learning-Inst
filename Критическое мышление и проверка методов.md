# Критическое мышление и проверка методов: инструкция пользователя

## О чём эта инструкция

Эта инструкция о том, как трезво проверять учебные советы, методики, курсы, приложения, нейрообъяснения, ИИ-workflow, карточки, конспекты и собственные привычки обучения.

Главный вопрос:

```text
Этот метод действительно помогает мне учиться,
или он только выглядит убедительно, научно, красиво или продуктивно?
```

В эту тему входят:

- критическое мышление;
- трезвое мышление;
- мифы об обучении;
- нейромифы;
- булшит-фильтр;
- научный метод;
- нулевая гипотеза;
- 7-дневные эксперименты;
- проверка методов без самообмана;
- разделение фактов, гипотез и преувеличений;
- ограничения исследований;
- неуниверсальность учебных техник;
- риск механической зубрёжки;
- проверка карточек, ИИ и конспектов;
- оценка образовательных продуктов;
- защита от маркетинговых обещаний;
- контроль иллюзии компетентности.

Короткая формула:

```text
не верить методу на слово,
а переводить его в проверяемое утверждение, действие, метрику и решение.
```

## Главная мысль

В обучении вредят не только плохие методы. Вредят и хорошие методы, если применять их механически, не к месту или без проверки.

Примеры:

- retrieval practice полезна, но плохой тест может проверять узнавание, а не знание;
- Anki полезен, но карточки "на всё подряд" создают долг и иллюзию работы;
- ИИ полезен как feedback, но прямые ответы до попытки могут снижать самостоятельное мышление;
- конспекты полезны, если перерабатывают материал, но бесполезны как дословное копирование;
- мнемотехники полезны для списков и фактов, но не заменяют понимание;
- 7-дневный эксперимент полезен для настройки личной системы, но не доказывает универсальную причинность;
- нейрообъяснение может быть правдоподобным, но не доказывать бытовой протокол.

Поэтому рабочее правило такое:

```text
сначала понять, что именно утверждается;
потом проверить уровень доказательности;
потом проверить применимость к моей задаче;
потом протестировать в маленьком цикле.
```

## Базовый словарь

| Термин | Что означает |
|---|---|
| Факт | То, что подтверждено наблюдением, источником, измерением или проверкой. |
| Гипотеза | Проверяемое предположение, которое может оказаться неверным. |
| Интерпретация | Объяснение фактов, зависящее от рамки и контекста. |
| Преувеличение | Слишком сильный вывод из ограниченных данных. |
| Миф | Устойчивое популярное убеждение, которое не подтверждается или сильно упрощает реальность. |
| Нейромиф | Псевдонаучное или чрезмерно упрощённое утверждение о мозге. |
| Нулевая гипотеза | Предположение, что эффекта нет, а наблюдение может быть случайностью или искажением. |
| Фальсифицируемость | Возможность представить проверку, которая могла бы показать, что идея неверна. |
| Контроль | Сравнение с условием, где метод не применялся или применялся другой метод. |
| Рандомизация | Случайное распределение условий, чтобы снизить предвзятость. |
| Ослепление | Участник или оценщик не знает, какое условие применяется. |
| Размер эффекта | Насколько велико изменение, а не только "есть ли статистическая значимость". |
| Практическая значимость | Достаточно ли эффект велик, чтобы менять поведение. |
| Перенос | Работает ли навык в новой ситуации, а не только в учебном примере. |
| Performance | Текущий результат выполнения задачи. |
| Learning | Устойчивое знание или навык, который остаётся после помощи. |

## Базовый протокол проверки метода

Используй для любого учебного совета:

- "делай карточки";
- "учись с ИИ";
- "переписывай конспекты";
- "смотри видео на скорости 2x";
- "используй дворец памяти";
- "делай паузу после обучения";
- "учи только утром";
- "покупай курс";
- "слушай классику для мозга";
- "делай 5 минут в день, и мозг изменится".

### Шаг 1. Выпиши утверждение

Не проверяй туманную фразу.

Плохо:

```text
Anki полезен.
```

Лучше:

```text
Если я буду делать 5-10 карточек из реальных ошибок и повторять их через интервалы, я буду реже повторять эти ошибки в задачах.
```

Плохо:

```text
ИИ помогает учиться.
```

Лучше:

```text
Если я буду использовать ИИ только после своей попытки для feedback, я лучше найду ошибки и смогу решить похожую задачу без ИИ.
```

### Шаг 2. Раздели утверждение на части

Любой метод обычно содержит несколько слоёв:

| Слой | Пример |
|---|---|
| Практическое действие | Делать карточки каждый день |
| Ожидаемый результат | Лучше помнить материал |
| Механизм | Retrieval + spacing |
| Условия | Карточки короткие, точные, по важным ошибкам |
| Ограничения | Не заменяет задачи и применение |
| Маркетинг | "Запомни всё навсегда без усилий" |

Частая ошибка - доказанность одного слоя переносить на все остальные.

Например:

```text
Retrieval practice подтверждена лучше, чем конкретный шаблон карточек,
конкретный плагин Anki,
конкретный курс,
конкретный AI-workflow
или обещание "плюс 40% к обучению".
```

### Шаг 3. Спроси про нулевую гипотезу

Главный вопрос:

```text
Что я ожидал бы увидеть, если метод не работает?
```

Примеры:

- часть дней будет лучше просто из-за сна;
- новая методика временно повысит мотивацию из-за новизны;
- после покупки курса появится энтузиазм, но не навык;
- после ИИ-ответа будет казаться, что понятно;
- после красивого конспекта появится чувство порядка;
- после зубрёжки получится сдать тест завтра, но не помнить через неделю.

Нулевая гипотеза защищает от поспешного вывода:

```text
мне стало легче -> значит, метод доказан.
```

Более аккуратно:

```text
мне стало легче -> возможно, помог метод;
возможно, помог сон;
возможно, помог эффект новизны;
возможно, задача была проще;
возможно, я оценил результат слишком мягко.
```

### Шаг 4. Найди альтернативные объяснения

Перед выводом спроси:

- что ещё могло повлиять;
- изменил ли я сразу несколько факторов;
- не была ли неделя проще обычной;
- не стал ли я больше времени уделять теме;
- не улучшился ли результат из-за повторения, а не из-за нового метода;
- не выбираю ли я только удачные примеры;
- не сравниваю ли я разные по сложности задачи;
- не путаю ли удовольствие с обучением.

### Шаг 5. Посмотри на цену метода

Метод может работать, но быть невыгодным.

Проверь:

- сколько времени занимает;
- сколько внимания требует;
- не создаёт ли долг;
- не откладывает ли практику;
- не повышает ли тревогу;
- не создаёт ли зависимость от инструмента;
- не заменяет ли задачу ритуалом;
- не мешает ли другим важным вещам: сну, практике, feedback, восстановлению.

### Шаг 6. Проверь перенос

Главный тест обучения:

```text
Могу ли я использовать это без исходной подсказки, в новой задаче?
```

Если метод улучшил только ощущение знакомости, он слаб.

Если метод помогает:

- объяснить;
- вспомнить;
- решить;
- сравнить;
- исправить ошибку;
- применить в новом контексте;
- повторить через несколько дней;

он вероятно полезнее.

### Шаг 7. Прими решение

После проверки метод получает один из статусов:

| Статус | Что делать |
|---|---|
| Оставить | Работает, цена приемлема, помогает закрывать учебные циклы |
| Упростить | Идея полезна, но реализация перегружена |
| Ограничить | Полезно только для узкой задачи |
| Отложить | Сейчас не подходит к цели или состоянию |
| Убрать | Не помогает, мешает или создаёт иллюзию прогресса |
| Проверить позже | Данных мало, но гипотеза интересная |

## Научный метод в бытовом обучении

Научный метод не нужен, чтобы превращать каждую учебную привычку в лабораторию. Он нужен как образ мышления:

```text
наблюдение -> гипотеза -> прогноз -> проверка -> корректировка -> повтор
```

### Что важно

Наука не даёт абсолютной безошибочности. Она снижает количество ошибок через процедуру проверки.

В обучении это означает:

- не верить одному примеру;
- не доверять только ощущению;
- проверять результат действием;
- учитывать альтернативные объяснения;
- признавать "не знаю";
- пересматривать метод, если данные против него.

### Семь практических столпов проверки

| Принцип | Как применить к обучению |
|---|---|
| Контроль | Сравнить новый метод с обычным способом, а не с хаосом |
| Размер выборки | Не делать вывод по одному удачному дню |
| Рандомизация | По возможности чередовать условия, чтобы не подбирать удобные дни |
| Ослепление | Проверять себя задачей/тестом, где ответ не виден заранее |
| Воспроизведение | Повторить результат в другой день и на другой теме |
| Открытость данных | Записывать не только успехи, но и провалы |
| Честность | Не переписывать критерии после результата |

### Что считать хорошей гипотезой

Хорошая гипотеза:

- конкретна;
- проверяема;
- ограничена во времени;
- имеет метрику;
- допускает отрицательный результат;
- указывает, что делать после проверки.

Пример:

```text
Если 7 дней после каждого учебного блока я буду закрывать источник и писать 5 тезисов по памяти,
то через сутки смогу точнее восстановить главные идеи,
а количество возвратов к исходнику снизится.
```

Плохая гипотеза:

```text
Если я буду учиться осознаннее, всё станет лучше.
```

Она слишком широкая.

## Факты, гипотезы, выводы и преувеличения

## Как разделять

Используй четыре блока:

```text
Факты:
что прямо известно?

Гипотезы:
что может быть правдой, но ещё не доказано?

Выводы:
что осторожно следует из фактов?

Преувеличения:
где вывод сильнее данных?
```

### Пример 1. ИИ в обучении

Факт:

```text
Есть исследования и отчёты, где ИИ может улучшать текущий результат выполнения задачи.
```

Гипотеза:

```text
ИИ может помогать обучению, если работает как тьютор, задаёт вопросы и даёт feedback.
```

Осторожный вывод:

```text
ИИ лучше использовать после собственной попытки и проверять самостоятельным повтором.
```

Преувеличение:

```text
ИИ автоматически делает обучение эффективнее.
```

### Пример 2. Карточки

Факт:

```text
Retrieval practice и интервальное повторение имеют сильную доказательную базу.
```

Гипотеза:

```text
Мои конкретные карточки в Anki помогают мне лучше решать задачи.
```

Осторожный вывод:

```text
Карточки полезны, если проверяют важные различия, ошибки и правила, а не копируют конспект.
```

Преувеличение:

```text
Любая карточка полезна.
```

### Пример 3. Конспект

Факт:

```text
Переработка материала своими словами помогает глубже обрабатывать информацию.
```

Гипотеза:

```text
Переписывание моего конспекта помогает мне лучше понимать лекцию.
```

Осторожный вывод:

```text
Переписывание полезно, если это переработка, структурирование и recall, а не механическое копирование.
```

Преувеличение:

```text
Конспекты всегда нужно переписывать полностью.
```

### Пример 4. Нейрообъяснение

Факт:

```text
Некоторые нейромеханизмы памяти, внимания и replay изучаются в лабораторных условиях.
```

Гипотеза:

```text
Пауза после учебного блока может помогать снижать интерференцию и поддерживать воспроизведение.
```

Осторожный вывод:

```text
После сложной темы стоит закрыть источник, восстановить 3 идеи и сделать паузу без нового потока.
```

Преувеличение:

```text
Ровно 5 минут паузы после каждой темы доказанно запускают гиппокампальный replay и гарантируют обучение.
```

## Мифы об обучении

## Миф 1. У каждого есть свой стиль обучения

Популярная версия:

```text
Я визуал / аудиал / кинестетик, значит, мне нужно учиться только своим способом.
```

Более точная версия:

```text
Формат обучения должен соответствовать материалу и задаче.
```

Если учишь:

- произношение - нужны слух, речь, запись себя, feedback;
- программирование - нужны код, тесты, ошибки, документация;
- анатомию - нужны схемы и визуальные структуры;
- аргументацию - нужны тексты, примеры, контраргументы;
- движение - нужны повторения и сенсорная обратная связь.

Практический вывод:

```text
Не подгоняй весь мир под свой "тип".
Выбирай формат под навык.
```

## Миф 2. Перечитывание равно обучению

Перечитывание создаёт знакомость:

```text
я это видел -> значит, знаю.
```

Но знание проверяется иначе:

```text
я могу вспомнить, объяснить, применить и отличить от похожего.
```

Практический вывод:

```text
После чтения закрыть источник и восстановить материал.
```

## Миф 3. Чем больше часов, тем лучше

Количество часов важно, но не само по себе.

Плохие часы:

- перечитывание;
- механическое копирование;
- бесконечные видео;
- карточки без внимания;
- настройка приложений;
- ИИ-диалоги без попытки;
- усталое сидение.

Хорошие часы:

- попытка;
- feedback;
- исправление;
- повтор;
- применение;
- сон и восстановление после нагрузки.

Практический вывод:

```text
Считать не часы, а закрытые учебные циклы.
```

## Миф 4. Зубрёжка - основной способ учиться

Зубрёжка может помочь:

- выучить стих;
- запомнить короткую формулировку;
- быстро удержать факты перед проверкой;
- подготовиться к тесту на узнавание.

Но она плохо работает как основная стратегия для:

- понимания;
- переноса;
- задач;
- письма;
- программирования;
- языка;
- долгосрочного владения темой.

Практический вывод:

```text
Зубрёжка допустима как узкий инструмент,
но не как система обучения.
```

## Миф 5. Мнемотехника заменяет понимание

Мнемотехника помогает помнить:

- списки;
- последовательности;
- даты;
- термины;
- выступления;
- наборы фактов.

Но она не заменяет:

- анализ;
- причинные связи;
- решение задач;
- перенос;
- критическое сравнение;
- понимание ограничений.

Практический вывод:

```text
Мнемотехника - опора для памяти, а не замена мышления.
```

## Миф 6. База не нужна, нужны только актуальные навыки

База может устаревать в деталях, но она даёт:

- язык темы;
- историю решений;
- понимание ограничений;
- возможность читать новые источники;
- способность отличать тренд от принципа.

Практический вывод:

```text
Не застревать в старой базе,
но и не строить обучение только из актуальных фрагментов.
```

## Миф 7. Нужно сразу брать самую сложную задачу

Сложные задачи полезны, если есть опора.

Опасность:

- перегруз рабочей памяти;
- быстрое выгорание;
- ложный вывод "я не способен";
- копирование чужого решения без понимания.

Практический вывод:

```text
Сложная задача -> откат к базовому примеру -> своя попытка -> возврат к сложной.
```

## Миф 8. Нужны идеальные условия

Комфорт помогает, но ожидание идеального места часто становится прокрастинацией.

Практический вывод:

```text
Главное - защищённый минимальный контекст:
20-40 минут, одна задача, минимум переключений.
```

Иногда полезно тренироваться в разных условиях, чтобы знание не было привязано только к одному месту.

## Миф 9. Ошибка доказывает неспособность

Ошибка чаще означает:

- недостаточную опору;
- неверную стратегию;
- сложность задачи;
- усталость;
- пробел в feedback;
- недостаток повторения.

Практический вывод:

```text
ошибка -> данные -> исправление -> повторная попытка
```

Неудача не является доказательством общей несостоятельности.

## Нейромифы

## Что такое нейромиф

Нейромиф - это утверждение, которое звучит как наука о мозге, но:

- слишком упрощает реальность;
- переносит лабораторный механизм прямо в бытовой совет;
- использует красивые термины без проверки;
- объясняет всё одной причиной;
- обещает быстрый эффект;
- подменяет результат ссылкой на "дофамин", "гиппокамп", "нейропластичность", "полушария" или "нейронные связи".

## Частые нейромифы

| Миф | Более точная позиция |
|---|---|
| Мы используем только 10% мозга | Мозг работает распределённо; не существует простого "разгона скрытых 90%" |
| Левополушарные - аналитики, правополушарные - творцы | Полушария взаимодействуют; поведение не сводится к такой схеме |
| У каждого фиксированный стиль обучения | Формат должен соответствовать задаче и материалу |
| Классическая музыка делает умнее | Музыка может влиять на настроение, но не является универсальным усилителем интеллекта |
| Нейропластичность означает "можно быстро стать кем угодно" | Изменения возможны, но требуют практики, времени и ограничены контекстом |
| Любой мозговой механизм даёт бытовой протокол | Механизм не равен инструкции |
| Нейрообъяснение делает метод доказанным | Нужны поведенческие результаты, не только красивая история о мозге |

## Фильтр нейроутверждений

Когда слышишь "это работает, потому что мозг...", спроси:

1. Это механизм или проверенный учебный результат?
2. На ком исследовали: люди, животные, пациенты, студенты, дети?
3. Какая задача была в исследовании?
4. Похож ли мой контекст на этот контекст?
5. Есть ли поведенческий эффект: память, перенос, результат задачи?
6. Размер эффекта большой или маленький?
7. Не превращают ли корреляцию в причинность?
8. Не используют ли нейротермин как маркетинговый ярлык?
9. Есть ли независимое повторение?
10. Какие ограничения сами авторы указали?

## Правило "механизм не равен методу"

Пример:

```text
В исследовании изучали hippocampal ripples и replay.
```

Из этого не следует автоматически:

```text
ровно 5 минут тишины после каждой темы гарантируют запоминание.
```

Осторожный практический вывод может быть таким:

```text
после сложного блока полезно закрыть источник, восстановить 3 идеи и не сразу забивать голову новым потоком.
```

## Булшит-фильтр для учебных методов

Используй его для курсов, марафонов, приложений, книг, блогеров, методик и "секретных систем".

## Красные флаги

| Красный флаг | Почему опасно |
|---|---|
| "Работает для всех" | В обучении почти всегда есть условия и ограничения |
| "Без усилий" | Устойчивое обучение требует попытки, ошибки и feedback |
| "Научно доказано", но без источников | Научность используется как ярлык |
| "Активирует мозг на 300%" | Псевдоколичественное обещание |
| "Секрет, о котором молчат университеты" | Маркетинг исключительности |
| Только отзывы и кейсы | Выборка успешных историй не доказывает метод |
| Давление дедлайном | FOMO вместо анализа |
| Автор продаёт тревогу | Сначала создаётся боль, потом продаётся решение |
| Нет конкретного навыка | Непонятно, чему именно учат |
| Много нейротерминов | Может маскировать отсутствие проверки |

## Что искать вместо обещаний

Хороший образовательный продукт показывает:

- чему конкретно учат;
- какие задачи ученик сможет делать;
- какой стартовый уровень нужен;
- сколько времени потребуется;
- как даётся feedback;
- как проверяется результат;
- какие ограничения у метода;
- кому курс не подойдёт;
- какие есть примеры заданий;
- кто преподаватель и в чём его компетенция.

## Протокол перед покупкой курса

1. Не покупать на пике эмоций.
2. Убрать рекламные фразы и отзывы.
3. Выписать только конкретные навыки.
4. Проверить, нужны ли они сейчас.
5. Проверить, есть ли в расписании время.
6. Найти независимые отзывы.
7. Посмотреть бесплатный фрагмент.
8. Проверить квалификацию автора.
9. Спросить: что будет, если я не куплю?
10. Принять решение через паузу.

Шаблон:

```text
Курс обещает:

Конкретные навыки:

Как проверяется результат:

Что вызывает эмоцию:

Какие источники доверия есть:

Какие красные флаги:

Есть ли у меня время:

Решение: купить / не покупать / отложить / найти альтернативу.
```

## 7-дневные эксперименты

## Для чего они нужны

7-дневный эксперимент не доказывает научную истину.

Он отвечает на более скромный вопрос:

```text
В моём текущем контексте этот метод снижает трение и помогает закрывать учебные циклы?
```

Это фильтр против накопления методов.

## Что можно тестировать

- AI только после попытки;
- карточки только из ошибок;
- один цвет в конспекте;
- 5 минут recall после чтения;
- пауза без контента после блока;
- план на следующий день вечером;
- запрет коротких видео перед учёбой;
- недельный обзор;
- чередование задач;
- ограничение новых карточек.

## Как ставить эксперимент

### 1. Одна гипотеза

Плохо:

```text
На этой неделе я улучшу обучение.
```

Хорошо:

```text
Если я 7 дней буду делать 5 минут recall после каждого учебного блока,
то через сутки буду лучше восстанавливать главную мысль без источника.
```

### 2. Один фактор

Не меняй сразу:

- сон;
- время занятий;
- Anki;
- ИИ;
- конспекты;
- спорт;
- питание;
- учебник.

Иначе непонятно, что повлияло.

### 3. Простые метрики

Выбирай 2-4 метрики:

- был ли учебный блок;
- была ли самостоятельная попытка;
- был ли feedback;
- была ли повторная задача;
- сколько раз переключался;
- смог ли вспомнить через сутки;
- сколько карточек добавил;
- сколько карточек реально помогли;
- сколько времени ушло на ритуал;
- стало ли меньше трения перед началом.

### 4. Критерий заранее

До начала эксперимента напиши:

```text
Оставлю метод, если...
Упрощу метод, если...
Уберу метод, если...
```

Иначе после недели легко подогнать вывод под настроение.

### 5. Итоговое решение

Через 7 дней не спрашивай:

```text
Мне понравилось?
```

Спроси:

```text
Метод помог закрывать проверяемые циклы?
```

Метрики:

## Проверка методов без самообмана

## Главные ловушки

## 1. Эффект новизны

Новый метод часто кажется сильным просто потому, что он новый.

Проверка:

```text
Сохранился ли эффект через 7-14 дней?
```

## 2. Подмена цели

Метод может улучшать не обучение, а ощущение порядка.

Пример:

- красивая система заметок;
- много тегов;
- настроенный Anki;
- длинный учебный план;
- аккуратный шаблон;
- красивый AI-prompt.

Проверка:

```text
Что я теперь могу сделать без подсказки?
```

## 3. Выбор удачных случаев

Если записывать только успехи, метод всегда будет выглядеть лучше.

Проверка:

```text
Записывать провалы, пропуски и дни, когда метод мешал.
```

## 4. Смена критерия после результата

До эксперимента цель была "лучше помнить", а после неудачи человек говорит:

```text
Зато я лучше структурировал.
```

Структура может быть полезна, но это уже другой результат.

Проверка:

```text
Критерий успеха задавать заранее.
```

## 5. Смешение факторов

Если одновременно начать спать лучше, делать Anki, меньше сидеть в ленте и использовать ИИ после попытки, нельзя понять, что помогло.

Проверка:

```text
Один основной фактор на 7 дней.
```

## 6. Ошибка "мне помогло"

Личный опыт важен для личного решения, но не доказывает универсальность.

Правильная формулировка:

```text
В моём контексте метод выглядит полезным.
```

Неправильная:

```text
Метод доказан и должен работать для всех.
```

## 7. Performance-learning gap

ИИ, подсказки, готовые конспекты и разобранные примеры могут улучшить текущий результат, но не оставить самостоятельного навыка.

Проверка:

```text
Через сутки решить похожую задачу без помощи.
```

## Ограничения исследований

Когда читаешь "исследование показало", проверь не только результат, но и дизайн.

## Вопросы к исследованию

1. Кто участвовал?
2. Сколько людей было в выборке?
3. Какая была задача?
4. Что именно измеряли?
5. Был ли контроль?
6. Было ли случайное распределение?
7. Был ли blind assessment?
8. Какой размер эффекта?
9. Был ли перенос?
10. Как долго длилось наблюдение?
11. Есть ли независимые повторения?
12. Кто финансировал исследование?
13. Не является ли это preprint?
14. Что сами авторы называют ограничениями?

## Типичные ограничения

| Ограничение | Что значит |
|---|---|
| Маленькая выборка | Результат нестабилен |
| Особая группа участников | Не ясно, переносится ли на всех |
| Короткий срок | Не ясно, есть ли долговременный эффект |
| Только immediate test | Проверили текущее выполнение, а не обучение |
| Нет transfer test | Не ясно, работает ли знание в новой ситуации |
| Корреляция | Нельзя уверенно говорить о причине |
| Нет контроля | Не ясно, с чем сравнивали |
| Самоотчёт | Люди могут ошибаться в оценке себя |
| Publication bias | Успешные результаты публикуются чаще |
| Конфликт интересов | Исследователь или компания может быть заинтересована |

## Как переводить исследование в практику

Не так:

```text
Исследование про X доказало, что мне нужно делать Y каждый день.
```

Лучше:

```text
Исследование поддерживает принцип X в условиях A.
Мой контекст похож / не похож.
Я проверю упрощённую версию через 7-дневный эксперимент.
```

## Неуниверсальность учебных техник

## Почему метод не может быть универсальным

Учебная техника зависит от:

- цели;
- типа материала;
- уровня ученика;
- времени до проверки;
- требуемого переноса;
- усталости;
- доступного feedback;
- мотивации;
- сложности задачи;
- среды;
- предметной области.

## Примеры

| Метод | Где полезен | Где может мешать |
|---|---|---|
| Anki | Термины, различия, ошибки, правила | Длинные рассуждения, проекты, всё подряд |
| Конспект | Структура, навигация, переработка | Дословное копирование и красивое оформление |
| ИИ | Feedback, вопросы, план, проверка | Готовые ответы до попытки |
| Зубрёжка | Дословный текст, краткий факт | Понимание, перенос, решение задач |
| Мнемотехника | Списки и последовательности | Глубокое понимание |
| Видео | Демонстрация и вход | Основной способ учить сложную тему |
| Interleaving | Выбор метода и перенос | Слишком рано, когда нет базовой модели |
| Сложные проекты | Практический синтез | Старт без базы и feedback |

## Правило применимости

Перед методом спроси:

```text
Какую задачу он решает?
Какую задачу он не решает?
Когда он становится вредным?
Как я это проверю?
```

## Когда карточки начинают мешать

Карточки мешают, если:

- добавляются "на всякий случай";
- копируют абзацы;
- проверяют узнавание;
- не связаны с ошибками и задачами;
- становятся главным занятием;
- создают долг;
- заставляют повторять лёгкое вместо практики;
- заменяют понимание;
- делаются ИИ без самогенерации;
- не ведут обратно к действию.

Хорошая карточка отвечает:

```text
Какую ошибку она предотвратит?
Какое различие поможет вспомнить?
Какую задачу поможет решить?
```

Плохая карточка отвечает:

```text
Как сохранить кусок текста?
```

### Проверка карточки

```text
1. Одна мысль?
2. Короткий вопрос?
3. Ответ проверяемый?
4. Нужна попытка вспомнить?
5. Есть связь с задачей?
6. Есть риск двусмысленности?
7. Не лучше ли оставить это в конспекте?
```

## Когда ИИ начинает мешать

ИИ мешает, если:

- даёт готовый ответ до попытки;
- улучшает работу, но не навык;
- заменяет самопроверку;
- делает карточки полностью сам;
- пишет эссе, которое пользователь не может защитить;
- решает код вместо ученика;
- подменяет источник гладким пересказом;
- усиливает уверенность без проверки;
- уменьшает потребность думать;
- создаёт зависимость от подсказки.

ИИ полезнее, если:

- проверяет твою попытку;
- задаёт вопросы;
- даёт минимальную подсказку;
- ищет ошибку;
- просит повторить без него;
- помогает аудитить источники;
- не пишет финальный ответ раньше времени.

### Проверка ИИ-сессии

После ИИ спроси:

```text
Что я могу сделать без ИИ прямо сейчас?
```

Если ответ:

```text
ничего, но я прочитал хорошее объяснение
```

значит, нужен ещё один цикл:

```text
закрыть чат -> попытка -> feedback -> повтор.
```

## Когда конспекты начинают мешать

Конспекты мешают, если:

- превращаются в стенографию;
- копируют чужие слова;
- слишком красиво оформляются;
- не используются для recall;
- не ведут к вопросам;
- не связаны с задачами;
- дублируют источник;
- создают ощущение завершённости без проверки;
- разрастаются быстрее, чем понимание.

Конспект полезен, если:

- написан своими словами;
- помогает быстро восстановить структуру;
- содержит вопросы;
- содержит примеры;
- содержит ошибки и ограничения;
- ведёт к карточкам только по важному;
- помогает вернуться к источнику;
- заканчивается проверяемым действием.

### Проверка конспекта

```text
1. Я могу по нему восстановить тему без источника?
2. Есть ли в нём вопросы для проверки?
3. Есть ли примеры и ограничения?
4. Нет ли дословного копирования?
5. Не потратил ли я на оформление больше, чем на понимание?
6. Что я сделаю после конспекта?
```

## Когда зубрёжка начинает мешать

Зубрёжка мешает, если:

- используется вместо понимания;
- маскирует тревогу;
- создаёт знакомость без воспроизведения;
- работает только до экзамена;
- не даёт примеров;
- не даёт переноса;
- не связывает материал с задачей.

Зубрёжка допустима, если:

- нужно дословно выучить короткий текст;
- нужно временно удержать факты;
- экзамен реально проверяет формулировки;
- после неё есть осмысление, применение и интервальный recall.

Правило:

```text
Если нужно знать форму - зубрёжка может помочь.
Если нужно владеть смыслом - нужна переработка и практика.
```

## Проверка образовательных советов из интернета

## Три главных вопроса

Из источников по критическому мышлению полезна короткая тройка:

```text
1. Достоверность: это правда?
2. Контекст: это вся правда?
3. Намерение: чего от меня хотят?
```

### Достоверность

Спроси:

- как это проверить;
- есть ли источник;
- есть ли независимая проверка;
- есть ли альтернативные данные;
- не является ли это личным опытом.

### Контекст

Спроси:

- для кого это верно;
- когда это верно;
- в какой задаче это проверяли;
- какой уровень ученика;
- какой срок;
- какой результат измеряли.

### Намерение

Спроси:

- что мне продают;
- к какому действию меня подталкивают;
- почему информация подана так эмоционально;
- кому выгодна эта рамка;
- что осталось за кадром.

## Шесть блоков вопросов

Для сложного решения можно использовать рамку:

| Блок | Вопросы |
|---|---|
| Кто | Кто выигрывает, кто проигрывает, кто автор, кто проверял? |
| Что | Что именно утверждается, что альтернативно, что будет без этого? |
| Где | Где это работало, где не работало, где найти больше данных? |
| Когда | Когда метод уместен, когда опасен, когда проверять результат? |
| Почему | Почему это важно, почему сейчас, почему я хочу поверить? |
| Как | Как проверить, как снизить риск, как понять, что метод работает? |

## Уровни доказательности

| Уровень | Как относиться |
|---|---|
| Мета-анализ / систематический обзор | Сильнее, но всё равно смотреть качество и ограничения |
| RCT / эксперимент | Хорошо для причинности, но зависит от дизайна и выборки |
| Квазиэксперимент | Полезно, но причинность слабее |
| Наблюдательное исследование | Показывает связи, не всегда причины |
| Лабораторный механизм | Объясняет процесс, но не всегда даёт бытовой протокол |
| Preprint | Интересный сигнал, но осторожно |
| Экспертное мнение | Полезно как ориентир, не как доказательство |
| Личный опыт | Полезен для гипотезы, не для универсального вывода |
| Отзывы | Слабый источник, часто отобраны |
| Маркетинговый текст | Проверять особенно строго |

## Решение по методу

| Доказательность | Цена | Решение |
|---|---|---|
| Высокая | Низкая | Внедрять в ядро |
| Высокая | Высокая | Внедрять дозированно |
| Умеренная | Низкая | Тестировать 7 дней |
| Умеренная | Высокая | Тестировать осторожно |
| Низкая | Низкая | Можно попробовать как гипотезу |
| Низкая | Высокая | Не внедрять без причины |
| Непроверено + громкие обещания | Любая | Сначала булшит-фильтр |

## Практические сценарии

## Если нашёл новый метод обучения

1. Выпиши точное утверждение.
2. Найди, что метод обещает улучшить.
3. Проверь источник.
4. Отдели механизм от результата.
5. Найди ограничения.
6. Сформулируй нулевую гипотезу.
7. Запусти 7-дневную проверку.
8. Реши: оставить, упростить, ограничить или убрать.

## Если метод "научно доказан"

Спроси:

- что именно доказано;
- в какой задаче;
- на какой выборке;
- как измеряли результат;
- был ли transfer;
- есть ли долгосрочная проверка;
- не продают ли конкретный продукт под видом общего принципа.

## Если метод помог знакомому

Прими это как гипотезу, не как доказательство.

Спроси:

- что именно ему помогло;
- какая была цель;
- сколько времени он вложил;
- что ещё изменилось;
- как он проверял результат;
- подойдет ли это моему контексту.

## Если хочется купить курс

Пауза 24 часа.

Затем:

- убрать эмоции;
- выписать навыки;
- проверить автора;
- проверить программу;
- проверить время;
- найти альтернативы;
- спросить, что будет без покупки;
- выбрать маленький тест до покупки, если возможно.

## Если хочется внедрить Anki

Не начинай с большого импорта.

Начни с эксперимента:

```text
7 дней: максимум 5 новых карточек в день, только из ошибок и важных различий.
```

Метрики:

- помогла ли карточка решить задачу;
- сколько карточек стали мусором;
- не вырос ли долг;
- стало ли меньше повторяющихся ошибок.

## Если хочется внедрить ИИ

Не начинай с:

```text
ИИ решает за меня.
```

Начни с:

```text
7 дней: ИИ только после моей попытки, только feedback и подсказки.
```

Метрики:

- была ли своя попытка;
- смог ли повторить без ИИ;
- улучшилось ли понимание ошибки;
- не выросла ли зависимость от подсказок.

## Если хочется переписывать конспекты

Не проверяй "переписывание вообще".

Проверь:

```text
7 дней: после лекции я перерабатываю конспект в вопросы, схему и 3 тезиса по памяти.
```

Метрики:

- смог ли восстановить тему через сутки;
- появились ли вопросы;
- стало ли меньше перечитывания;
- не ушло ли слишком много времени на оформление.

## Если хочется отказаться от метода

Проверь, не слишком ли рано.

Метод мог не сработать из-за:

- плохой реализации;
- слишком высокой сложности;
- усталости;
- недостатка времени;
- неправильной задачи;
- слишком короткой проверки;
- отсутствия feedback.

Сначала упростить, потом убрать.

## 7-дневный старт

## День 1. Ввести разделение фактов и гипотез

В любой учебной заметке добавь блок:

```text
Факты:
Гипотезы:
Преувеличения:
Что проверить:
```

Цель дня: перестать смешивать источник, вывод и личную догадку.

## День 2. Проверить один учебный миф

Выбери один миф:

- стили обучения;
- зубрёжка;
- идеальные условия;
- "ИИ всё ускоряет";
- "Anki решит память";
- "конспект = знание".

Разбери по схеме:

```text
Что утверждается?
Что подтверждено?
Где ограничение?
Какой осторожный вывод?
```

## День 3. Запустить один 7-дневный эксперимент

Выбери один фактор.

Примеры:

- 5 минут recall после блока;
- ИИ только после попытки;
- карточки только из ошибок;
- один цвет в конспекте;
- короткая пауза без контента после учёбы.

Запиши гипотезу и критерии заранее.

## День 4. Провести булшит-фильтр курса или метода

Возьми один курс, статью, ролик или приложение.

Проверь:

- обещание;
- доказательства;
- красные флаги;
- конкретный навык;
- цена;
- время;
- альтернативы.

## День 5. Проверить карточки, ИИ или конспекты

Выбери один инструмент, который уже используешь.

Спроси:

```text
Он ведёт к самостоятельной попытке или заменяет её?
```

Убери один лишний элемент.

## День 6. Найти ограничения исследования

Возьми один научно звучащий тезис и найди:

- выборку;
- задачу;
- результат;
- срок;
- ограничения;
- что нельзя выводить.

## День 7. Принять одно решение

По итогам недели:

```text
Оставляю:
Упрощаю:
Ограничиваю:
Убираю:
Проверяю позже:
```

Цель дня: не накопить ещё одну методику, а очистить систему.

## Уровень доказательности по темам

| Тема | Оценка |
|---|---|
| Retrieval practice полезна для долговременного запоминания | высокая |
| Spaced repetition полезна для удержания | высокая |
| Feedback помогает обучению, если ведёт к исправлению | умеренно высокая |
| Ошибки полезны сами по себе | неверно; нужна обратная связь и повтор |
| Перечитывание как основной метод эффективно | слабая поддержка; часто создаёт иллюзию знания |
| Зубрёжка полезна для краткосрочного дословного воспроизведения | ограниченно верно |
| Зубрёжка как универсальное обучение | неверно |
| Стили обучения "визуал/аудиал/кинестетик" как строгая стратегия | слабая поддержка |
| Мнемотехники полезны для списков и фактов | умеренно / контекстно |
| Мнемотехники заменяют понимание | неверно |
| Anki полезен как spaced retrieval | умеренно высокая |
| Любая карточка полезна | неверно |
| ИИ полезен как feedback после попытки | умеренно, зависит от дизайна |
| ИИ всегда улучшает learning | неверно |
| Конспект полезен, если перерабатывает материал | практично и согласуется с learning science |
| Красивый конспект сам по себе улучшает обучение | слабая поддержка |
| 7-дневный эксперимент доказывает метод | неверно |
| 7-дневный эксперимент помогает личной настройке | разумная практическая гипотеза |
| Нейромифы про 10% мозга и жёсткие полушарные типы | не подтверждаются |
| Нейромеханизм автоматически даёт бытовой протокол | неверно |

## Минимальная система критической проверки

Если не хочется вести большую систему, оставь 5 вопросов:

```text
1. Что именно утверждается?
2. Какие данные это поддерживают?
3. Какие есть ограничения?
4. Что может быть альтернативным объяснением?
5. Как я проверю это маленьким действием?
```

Для обучения добавь шестой:

```text
6. Что я смогу сделать без подсказки?
```

## Итоговый пользовательский протокол

1. Не принимай учебный совет целиком.
2. Выпиши точное утверждение.
3. Раздели факт, гипотезу, механизм и маркетинг.
4. Спроси про нулевую гипотезу.
5. Найди ограничения исследования или источника.
6. Проверь применимость к своей задаче.
7. Запусти маленький 7-дневный эксперимент, если метод низкорисковый.
8. Смотри не на ощущение продуктивности, а на закрытые учебные циклы.
9. Оставляй только то, что помогает: попытка, feedback, исправление, повтор, перенос.
10. Убирай методы, которые стали ритуалом, долгом или заменой мышления.

Короткая версия:

```text
метод -> утверждение -> доказательства -> ограничения -> мой эксперимент -> решение
```

Главный критерий:

```text
после метода я стал лучше думать, вспоминать и действовать без подсказки?
```
