# Active recall: инструкция пользователя

## О чём эта инструкция

Active recall (активное воспроизведение, активное припоминание, активное извлечение) - это способ учиться через попытку достать знание из памяти без подсказки.

Главная идея:

```text
не перечитать ещё раз,
а закрыть источник и попробовать вспомнить, объяснить, решить или применить.
```

Active recall близок к терминам:

- retrieval practice - практика извлечения из памяти;
- practice testing - учебное самотестирование;
- testing effect - эффект тестирования;
- test-enhanced learning - обучение, усиленное проверкой;
- free recall - свободное воспроизведение;
- cued recall - воспроизведение по вопросу или подсказке;
- spaced retrieval - извлечение через интервалы.

Эта инструкция не про один трюк и не только про карточки. Active recall - это семейство практик:

- вопросы к себе;
- пересказ без конспекта;
- чистый лист;
- карточки и Anki;
- тесты;
- задачи без подсказки;
- объяснение своими словами;
- восстановление схемы;
- работа с ошибками;
- pre-testing до материала;
- повторение через интервалы;
- interleaving похожих тем;
- feedback после попытки;
- использование ИИ только после собственной попытки.

Короткая формула:

```text
изучил -> закрыл источник -> вспомнил -> проверил -> исправил -> повторил позже -> применил
```

## Главная мысль

Повторное чтение часто создаёт ощущение знакомости:

```text
я это видел,
значит, я это знаю.
```

Но знакомость не равна владению. Узнать фразу в конспекте проще, чем самостоятельно восстановить идею, объяснить её, применить к задаче и не спутать с соседней темой.

Active recall проверяет более строгий вопрос:

```text
Что я могу достать из памяти без источника?
```

Если ответ не вспоминается, это не провал. Это диагностический сигнал: где пробел, что нужно перечитать точечно, какую ошибку исправить, что повторить позже.

## Чем active recall отличается от похожих вещей

| Подход | Что происходит | Риск |
|---|---|---|
| Перечитывание | Материал снова перед глазами | Возникает узнавание без владения |
| Подсветка текста | Выделяются фрагменты | Можно выделить, но не понять и не вспомнить |
| Конспектирование | Материал записывается | Можно переписать без обработки |
| Active recall | Источник закрыт, идёт попытка вспомнить | Труднее и неприятнее, зато честнее |
| Practice testing | Проверка становится частью обучения | Плохой тест может проверять узнавание |
| Spaced repetition | Повторения разнесены во времени | Без извлечения превращается в просмотр |
| Anki | Карточки показываются по расписанию | Плохие карточки закрепляют мусор |

Правильная связка:

```text
понимание -> active recall -> feedback -> интервал -> повторный recall -> применение
```

## Что считается active recall

Active recall есть там, где сначала идёт самостоятельная попытка.

Подходит:

- закрыть книгу и написать главную мысль;
- ответить на вопросы без конспекта;
- пересказать тему своими словами;
- нарисовать схему по памяти;
- восстановить ход решения;
- решить задачу без подсказки;
- объяснить тему воображаемому ученику;
- вспомнить определение и привести пример;
- сравнить два похожих понятия;
- найти ошибку в примере;
- заполнить пропуск в формуле или предложении;
- пройти карточки в Anki;
- перед чтением попробовать ответить на предварительные вопросы;
- после ошибки объяснить, какой принцип был нарушен;
- попросить ИИ проверить уже готовую попытку.

Не считается полноценным active recall:

- читать ответ и думать "я бы так сказал";
- подсвечивать текст;
- смотреть конспект и узнавать знакомые формулировки;
- сразу спрашивать ИИ и читать готовое объяснение;
- делать карточки с огромными абзацами;
- проходить тест без разбора ошибок;
- повторять лёгкую карточку десять раз подряд ради спокойствия;
- пересказывать без проверки по источнику.

## Базовый протокол после любого учебного блока

Используй после статьи, видео, лекции, главы учебника, практической задачи, занятия с преподавателем или разговора с ИИ.

### Шаг 1. Ограничь блок

Не пытайся вспоминать сразу весь курс.

Хороший блок:

- 15-40 минут чтения или видео;
- один раздел;
- одна идея;
- один тип задачи;
- одна ошибка;
- один небольшой навык;
- одна лекция или её фрагмент.

Если блок слишком большой, recall превращается в хаотичный пересказ. Если слишком маленький, проверка становится формальной.

### Шаг 2. Закрой источник

Пока источник открыт, мозг получает подсказки.

Закрыть источник значит:

- убрать текст;
- не смотреть в конспект;
- не держать перед глазами ответ;
- не открывать ИИ;
- не листать "только на секунду";
- сначала сделать честную попытку.

Даже 2-5 минут достаточно, если попытка настоящая.

### Шаг 3. Восстанови главное

Минимальный набор:

```text
1. О чём был материал?
2. Какая главная идея?
3. Какие 3-5 опорных тезисов?
4. Какой пример важен?
5. Где ограничение?
6. Что я могу применить?
7. Что осталось мутным?
```

Для сложной темы добавь:

```text
Почему это работает?
Чем это отличается от похожей идеи?
Где типичная ошибка?
Как проверить, что я понял?
Какую маленькую задачу я могу сделать без подсказки?
```

### Шаг 4. Оцени качество ответа

До проверки отметь:

- что помню уверенно;
- что помню смутно;
- где угадываю;
- где не могу привести пример;
- где могу пересказать, но не могу применить;
- где путаю похожие понятия.

Это metacognitive monitoring: ты проверяешь не только ответ, но и точность своей уверенности.

### Шаг 5. Проверь по источнику

Проверка должна быть конкретной:

- что пропущено;
- что искажено;
- где перепутана причина и следствие;
- где термин использован неточно;
- где пример не подходит;
- где ответ слишком общий;
- где нет условия применения;
- где нет ограничения.

### Шаг 6. Исправь одну ошибку

Не оставляй ошибку в виде "понятно".

Сделай одно действие:

- перепиши формулировку;
- добавь один пример;
- составь один вопрос;
- сделай карточку;
- реши похожую задачу;
- объясни различие с похожей идеей;
- запланируй повтор.

Хорошее исправление заканчивается новой проверяемой попыткой.

### Шаг 7. Вернись позже

Active recall особенно силён вместе с интервалами.

Простой график:

```text
сразу после изучения
через 1 день
через 3-4 дня
через 7 дней
через 2-4 недели
```

Не нужно считать интервалы священными. Важно не делать всё одним плотным блоком и не заменять повторение пассивным перечитыванием.

## Вариации active recall

### 1. Free recall: свободное воспроизведение

Формат:

```text
Закрыть источник и написать всё, что помнишь по теме.
```

Подходит для:

- статей;
- лекций;
- видео;
- глав учебника;
- сложных концептов;
- подготовки к обсуждению;
- недельного обзора.

Как делать:

1. Поставить таймер на 3-7 минут.
2. Писать без редактирования.
3. Если структура не вспоминается, писать отдельные фрагменты.
4. После первой волны добавить схему.
5. Свериться с источником.
6. Отметить пропуски и искажения.

Плюс: хорошо показывает, что реально доступно из памяти.

Минус: может быть слишком расплывчатым, если тема новая или плохо структурирована.

### 2. Cued recall: вопросы к себе

Формат:

```text
Ответить на вопросы без подсказки.
```

Базовые вопросы:

- Что это?
- Зачем это нужно?
- Как работает?
- Почему это важно?
- Где пример?
- Где ограничение?
- Чем отличается от похожего?
- Где типичная ошибка?
- Как применить?
- Как проверить?

Плохой вопрос:

```text
Active recall полезен?
```

Лучше:

```text
Почему active recall обычно полезнее повторного чтения для долговременного запоминания?
```

Ещё лучше:

```text
После просмотра лекции я хочу закрепить материал. Почему сначала стоит ответить на вопросы без конспекта, а не пересмотреть лекцию?
```

### 3. Blank page recall: чистый лист

Формат:

```text
Взять пустой лист и восстановить тему: понятия, связи, схема, примеры, ошибки.
```

Шаблон:

```text
Тема:
Главная идея:
Ключевые элементы:
Связи:
Пример:
Контрпример:
Типичная ошибка:
Ограничение:
Что проверить:
```

Подходит для:

- системных тем;
- программирования;
- права;
- психологии;
- научных концептов;
- подготовки к экзамену;
- Zettelkasten-заметок.

Главная польза: чистый лист показывает не только отдельные факты, но и структуру знания.

### 4. Closed-book summary: закрытый пересказ

Формат:

```text
Закрыть источник и написать короткое summary своими словами.
```

Правило:

```text
Не больше 5-7 предложений.
```

Хорошее summary отвечает:

- о чём материал;
- какой главный вывод;
- почему это важно;
- что я применю;
- где ограничение.

Плохой вариант - переписывать формулировки автора после открытия текста. Это уже не recall, а конспектирование.

### 5. Practice testing: учебное самотестирование

Формат:

```text
Сделать мини-тест не ради оценки, а ради обучения.
```

Варианты:

- 5 вопросов после раздела;
- 10 вопросов перед экзаменом;
- пробный билет;
- тест с открытым ответом;
- задача с проверкой решения;
- устный ответ по таймеру.

Важно:

- сначала ответить самому;
- затем проверить;
- исправить ошибку;
- повторить позже.

Тест без разбора ошибок превращается в измерение, а не в обучение.

### 6. Flashcards и Anki

Карточки - частный случай active recall.

Карточка полезна, если проверяет:

- определение;
- различие;
- правило;
- команду;
- формулу;
- типичную ошибку;
- мини-кейс;
- связь "условие -> действие";
- вопрос, который часто забывается.

Плохая карточка:

```text
Q: Active recall
A: Метод активного вспоминания, retrieval practice, testing effect...
```

Проблема: вопрос слишком широкий, ответ легко перечитывать.

Хорошая карточка:

```text
Q: Чем active recall отличается от перечитывания?
A: При active recall я пытаюсь восстановить знание без подсказки; при перечитывании в основном узнаю знакомый текст.
```

Хорошая карточка на применение:

```text
Q: Что сделать после учебного видео, чтобы не ограничиться иллюзией понимания?
A: Закрыть видео, восстановить 3-5 тезисов, проверить по источнику и сделать одну задачу или вопрос для повтора.
```

Правила карточек:

- одна карточка - одна мысль;
- короткий проверяемый ответ;
- не копировать абзацы;
- не делать карточки на всё подряд;
- добавлять карточки из ошибок, различий и рабочих правил;
- для навыков после карточки делать задачу.

### 7. Cloze recall: восстановление пропусков

Формат:

```text
В предложении скрыт важный элемент, который нужно восстановить.
```

Пример:

```text
Active recall полезен, потому что тренирует не узнавание, а {{c1::извлечение знания из памяти}}.
```

Когда полезно:

- формулы;
- термины в контексте;
- иностранные фразы;
- короткие правила;
- последовательности.

Риск: закрывать случайные слова. Cloze должен проверять смысловой элемент, а не то, что легко угадать по грамматике.

### 8. Active recall через задачи

Для навыков пересказа недостаточно.

Формат:

```text
Закрыть источник и сделать маленькое действие.
```

Примеры:

- написать функцию без подсказки;
- решить задачу;
- разобрать кейс;
- составить аргумент;
- перевести фразу;
- записать голос;
- построить схему;
- найти ошибку в примере;
- объяснить вывод программы.

Критерий:

```text
я не просто рассказал, а сделал проверяемую попытку.
```

### 9. Active recall через объяснение

Формат:

```text
Объяснить тему своими словами без источника.
```

Варианты:

- вслух самому себе;
- письменно;
- другу;
- воображаемому ученику;
- в формате метода Фейнмана;
- в виде короткого урока.

Шаблон:

```text
Если объяснять новичку:
1. Что это?
2. Зачем нужно?
3. Как работает?
4. Пример.
5. Типичная ошибка.
6. Ограничение.
```

Если объяснение разваливается, это не повод ругать себя. Это сигнал: модель ещё не собрана.

### 10. Self-explanation: объяснение собственных шагов

Self-explanation - это не просто пересказ темы. Это объяснение того, почему конкретный шаг работает.

После примера:

```text
1. Что здесь происходит?
2. Почему этот шаг правильный?
3. Какой принцип используется?
4. Где здесь можно ошибиться?
```

После ошибки:

```text
1. Какой шаг был неверным?
2. Какой принцип я нарушил?
3. Какой похожий пример проверит исправление?
```

Главное: self-explanation должно быть коротким и проверяемым. Длинное рассуждение без проверки может снова стать иллюзией понимания.

### 11. Generation effect: самостоятельная генерация

Формат:

```text
Не только вспомнить готовый ответ, а самостоятельно создать пример, правило, вопрос или схему.
```

Варианты:

- придумать пример;
- придумать контрпример;
- вывести правило из случаев;
- заполнить пропущенный шаг;
- сформулировать вопрос;
- объяснить, почему неверный вариант неверен;
- создать мини-задачу.

Связь с active recall:

- active recall чаще восстанавливает уже изученное;
- generation шире: человек создаёт ответ или пример;
- оба работают лучше пассивного просмотра, если трудность посильная и есть проверка.

### 12. Pre-testing: вопросы до изучения

Формат:

```text
Перед материалом попробовать ответить на вопросы, даже если пока не знаешь ответ.
```

Протокол:

```text
1. Перед чтением записать 3-5 вопросов.
2. Попробовать ответить без подсказки.
3. Отметить неуверенность.
4. Читать или смотреть источник, специально ища ответы.
5. После материала ответить повторно.
```

Польза: предварительная попытка направляет внимание.

Ограничение: ошибочный ответ полезен не сам по себе. Нужна последующая корректировка.

### 13. Error-based recall: воспроизведение через ошибки

Формат:

```text
Сначала попытка, потом конкретная ошибка, потом исправление и похожая повторная задача.
```

Шаблон:

```text
Мой прогноз:
Моя попытка:
Факт:
Ошибка:
Почему возникла:
Правильный принцип:
Похожая задача:
```

Ошибка полезна, если она:

- замечена;
- конкретна;
- связана с действием;
- имеет корректировку;
- проверяется в повторной попытке.

Плохая формулировка:

```text
Я туплю.
```

Хорошая формулировка:

```text
Я перепутал условие задачи и применил правило не к тому случаю.
```

### 14. Spaced active recall: активное вспоминание через интервалы

Формат:

```text
Возвращаться к вопросу не подряд, а через паузы.
```

Пример:

```text
День 0: изучил и вспомнил сразу.
День 1: ответил без источника.
День 4: решил задачу или объяснил тему.
День 7: восстановил схему.
День 21: применил к новому примеру.
```

Почему это важно: если повторять слишком быстро, материал кажется лёгким. Интервал делает извлечение труднее и честнее.

### 15. Successive relearning: последовательное переучивание

Формат:

```text
Через интервалы снова воспроизводить материал до заданного критерия.
```

Пример критерия:

- 3 правильных ответа подряд;
- 90-100% карточек;
- объяснить тему за 3 минуты;
- решить задачу без подсказки;
- написать пример без ошибки.

Протокол:

```text
День 1: изучить и воспроизвести до критерия.
День 3: снова воспроизвести до критерия.
День 7: повторить без подсказки.
День 14: контрольное воспроизведение.
День 30: проверка удержания.
```

Не путать с оверлернингом. Successive relearning возвращает материал через интервалы; оверлернинг часто гоняет лёгкое подряд в одной сессии.

### 16. Variable recall: вариативное воспроизведение

Формат:

```text
Одну идею проверять разными способами.
```

Например, active recall можно проверить так:

- дать определение;
- объяснить механизм;
- сравнить с перечитыванием;
- привести пример после видео;
- придумать плохую карточку и улучшить её;
- найти ошибку в учебном плане;
- применить к подготовке к экзамену;
- назвать ограничение.

Польза: знание становится гибче и лучше переносится.

Риск: слишком ранняя вариативность перегружает. Сначала нужна минимальная модель.

### 17. Interleaved recall: чередование похожих тем

Формат:

```text
Перемешивать вопросы по похожим темам, чтобы учиться различать.
```

Не так:

```text
AAAA BBBB CCCC
```

А так:

```text
A B C A C B A
```

Полезно, когда темы похожи:

- active recall / spaced repetition / interleaving;
- fake / mock / stub;
- value receiver / pointer receiver;
- простое / сокращение / изменение условий;
- тревога / страх / стресс;
- времена в иностранном языке.

Главный вопрос interleaving:

```text
Какой тип знания нужен именно здесь?
```

Ограничение: на самом первом этапе лучше коротко разобрать каждый элемент отдельно, а потом смешивать.

### 18. Recall + monitoring + feedback

Формат:

```text
Вспомнить -> оценить уверенность -> проверить -> исправить.
```

Шаблон:

```text
Ответ:
Уверенность 0-10:
Что точно:
Что смутно:
Проверка:
Исправление:
Следующая попытка:
```

Это защищает от двух ошибок:

- уверенно повторять неверный ответ;
- считать смутное узнавание настоящим знанием.

### 19. Active recall с ИИ

ИИ полезен после собственной попытки, а не вместо неё.

Плохой сценарий:

```text
Я не пробовал -> ИИ объяснил -> я прочитал -> кажется, что понял.
```

Хороший сценарий:

```text
мой прогноз -> моя попытка -> факт/ошибка -> вопрос к ИИ -> feedback -> повторная задача
```

Шаблон запроса:

```text
Я изучаю тему:
Мой ответ без подсказки:
Где я не уверен:
Проверь точность.
Сначала укажи искажения и пробелы.
Не решай всё за меня.
Дай один следующий вопрос или одну похожую задачу.
```

После ответа ИИ обязательно сделать повтор:

```text
закрыть ответ -> восстановить исправленную идею -> решить похожую задачу -> проверить
```

### 20. Morning recall: утренняя реактивация

Формат:

```text
Утром восстановить вчерашний материал без конспекта.
```

Мини-протокол:

```text
Вчера я понял:
1.
2.
3.

Пример:

Я путаю:

Сегодня продолжу с:
```

Важно: это не магический утренний ритуал. Польза появляется, когда есть активная попытка и проверка. Просмотр заголовков - это узнавание, а не recall.

### 21. Pause + recall: пауза и возвращение

Пауза после обучения полезна не сама по себе, а в цикле:

```text
изучил -> пауза -> вспомнил -> проверил -> исправил
```

После паузы не начинай с перечитывания. Сначала спроси:

```text
Что осталось доступно из памяти?
Что исчезло?
Что исказилось?
```

### 22. Recall после коротких видео

Короткое видео может быть полезным входом или демонстрацией, но сложную тему опасно собирать только из разрозненных фрагментов.

Протокол:

```text
1. Перед видео сформулировать 1-3 вопроса.
2. Смотреть один связный фрагмент.
3. Не переходить сразу к следующему ролику.
4. Закрыть видео.
5. Восстановить главную идею, пример и действие.
6. Сделать маленькую задачу или объяснение.
7. Вернуться только к непонятному месту.
```

### 23. Active recall для речи и языка

Для языка active recall должен проверять не только перевод, но и использование.

Варианты:

- слово -> фраза;
- фраза -> значение;
- значение -> собственная фраза;
- аудио -> записать услышанное;
- увидеть слово -> произнести;
- минимальные пары;
- типичная ошибка употребления.

Для произношения полезна петля:

```text
образец -> попытка -> запись себя -> сравнение -> исправление одного параметра
```

Параметры:

- звук;
- ударение;
- ритм;
- паузы;
- интонация.

### 24. Active recall для письма

Формат:

```text
Не перечитывать советы по письму, а восстановить структуру и написать фрагмент.
```

Протокол:

```text
1. Закрыть источник.
2. Вспомнить структуру текста.
3. Написать тезис.
4. Написать один абзац.
5. Проверить по критериям.
6. Исправить одну ошибку.
```

### 25. Active recall для экзамена

Формат:

```text
Ответ без конспекта в формате будущей проверки.
```

Нужны:

- пробные билеты;
- устные ответы;
- письменные ответы;
- задачи по времени;
- смешанные темы;
- разбор ошибок;
- повтор через интервалы.

Чего не делать:

- только перечитывать;
- учить всю ночь;
- откладывать самопроверку до последнего дня;
- тренировать только лёгкие вопросы.

## Как применять по типам материала

### После статьи

```text
1. Закрыть статью.
2. Написать главную мысль.
3. Восстановить 5-7 тезисов.
4. Назвать аргументы.
5. Назвать ограничения.
6. Сформулировать практический вывод.
7. Проверить по источнику.
8. Сделать 1-3 вопроса для повтора.
```

### После видео или лекции

```text
1. Не пересматривать сразу.
2. Восстановить структуру видео.
3. Выписать 3-5 главных идей.
4. Объяснить один пример.
5. Записать вопросы, которые остались мутными.
6. Вернуться к источнику точечно.
```

### После учебника

```text
1. Перед главой задать 3 вопроса.
2. После раздела закрыть книгу.
3. Восстановить определения, связи и примеры.
4. Решить 1-3 задачи.
5. Свериться с ответами.
6. Ошибки превратить в вопросы для повторения.
```

### После конспекта

Не перечитывать конспект первым делом.

Сначала:

```text
1. Закрыть конспект.
2. Восстановить структуру.
3. Ответить на вопросы с полей, если они есть.
4. Проверить пробелы.
5. Перечитать только слабые места.
```

### После практической задачи

```text
1. Закрыть решение.
2. Восстановить ход рассуждения.
3. Назвать принцип.
4. Назвать ошибку или риск.
5. Решить похожую задачу.
6. Сделать карточку только по важному различию или ошибке.
```

### При изучении программирования

Примеры active recall:

- написать код без подсказки;
- объяснить вывод программы;
- восстановить API;
- предсказать ошибку компиляции;
- написать тест;
- сравнить два похожих подхода;
- найти баг;
- объяснить, почему решение работает.

Шаблон:

```text
Тема:
Что это делает:
Мини-пример без подсказки:
Где ошибка возможна:
Как проверить:
Похожая задача:
```

### При изучении права или сложных правил

Примеры:

- назвать норму;
- восстановить условия применения;
- перечислить юридически значимые факты;
- отличить факт от вывода;
- сформулировать аргумент;
- найти слабое место позиции;
- сравнить похожие процедуры.

Шаблон:

```text
Ситуация:
Какие факты важны:
Какая норма:
Что нужно доказать:
Какие документы:
Где спор:
Какой вывод:
```

### При изучении психологии, памяти и обучения

Примеры:

- дать определение;
- объяснить механизм;
- назвать уровень доказательности;
- указать ограничение;
- сравнить с похожим эффектом;
- придумать практический протокол;
- назвать риск неправильного вывода.

## Уровни вопросов

Чтобы active recall не застрял на определениях, используй разные уровни.

### Уровень 1. Факт

```text
Что такое active recall?
Как называется эффект тестирования?
Что такое spaced repetition?
```

### Уровень 2. Понимание

```text
Почему перечитывание создаёт иллюзию знания?
Почему попытка вспомнить укрепляет доступ к знанию?
Зачем нужна обратная связь после ответа?
```

### Уровень 3. Различение

```text
Чем active recall отличается от retrieval practice?
Чем active recall отличается от spaced repetition?
Чем interleaving отличается от spacing?
```

### Уровень 4. Применение

```text
Как применить active recall после видео?
Как превратить ошибку в карточку?
Как построить повторение темы на неделю?
```

### Уровень 5. Перенос

```text
Как применить этот принцип к новой задаче?
Какой пример отличается от изученного, но требует того же правила?
Как использовать active recall в реальной работе?
```

### Уровень 6. Критика

```text
Когда active recall может быть бесполезен?
Какие риски у карточек?
Почему любой тест не является хорошим обучением?
Как понять, что вопрос слишком лёгкий или слишком трудный?
```

## Как выбрать сложность

Хороший active recall должен быть трудным, но посильным.

Слишком легко:

- ответ вспоминается автоматически;
- вопрос проверяет узнавание;
- нет риска ошибки;
- не нужен пример;
- не нужно применять.

Слишком трудно:

- невозможно начать;
- базового понимания нет;
- человек угадывает;
- нет обратной связи;
- слишком много новых элементов;
- задание вызывает только фрустрацию.

Рабочая зона:

- нужно подумать;
- ответ частично доступен;
- есть шанс ошибки;
- можно проверить;
- ошибка ведёт к следующей попытке;
- задача связана с будущим применением.

## Режим дня для active recall

### Утро

Подходит для:

- короткой реактивации вчерашнего материала;
- сложных вопросов, если это твоё окно ясности;
- Anki review;
- планирования повторов;
- первой самостоятельной задачи.

Мини-протокол:

```text
1. Вспомнить вчерашнюю тему без конспекта.
2. Проверить 1-2 пробела.
3. Сделать due cards или 3 вопроса.
4. Выбрать одну задачу на сегодня.
```

Если утро слабое, не делай из этого проблему. Оставь утром лёгкий recall, а сложные задачи перенеси в устойчивое окно ясности.

### День

Подходит для:

- основных учебных блоков;
- задач;
- feedback;
- смешанных вопросов;
- работы с ошибками.

Протокол:

```text
25-50 минут обучение или задача
3-7 минут active recall
5-10 минут проверка и исправление
пауза
```

### Вечер

Подходит для:

- лёгкого review;
- Anki;
- записи ошибок;
- подготовки первого шага на завтра;
- короткого recall без перегруза.

Не лучший момент для:

- многочасовой зубрёжки;
- тяжёлой новой темы после истощения;
- бесконечного добивания лёгких карточек.

### После недосыпа

Если сон был плохой:

- уменьши новые карточки;
- делай короткий review;
- не ставь самый сложный тест первым;
- оценивай ошибки честно;
- кофеин не считай заменой сна;
- восстановление поставь в план.

## Типичные ошибки

### Ошибка 1. Считать перечитывание самопроверкой

Если ответ открыт, память не делает нужную работу. Перечитывание полезно для уточнения, но не должно быть первым способом закрепления.

### Ошибка 2. Проверять только определения

Определение важно, но для владения нужны:

- пример;
- различие;
- условие применения;
- ошибка;
- задача;
- ограничение.

### Ошибка 3. Делать карточки на всё подряд

Карточка нужна не для каждого предложения, а для того, что:

- часто забывается;
- важно применять;
- путается с другим понятием;
- уже привело к ошибке;
- нужно быстро достать из памяти.

### Ошибка 4. Использовать active recall до понимания

Если материал вообще не понят, recall превращается в угадывание. Сначала нужен минимальный пример, схема или объяснение. Потом - попытка.

### Ошибка 5. Не проверять ответ

Active recall без feedback может закрепить уверенную ошибку. Особенно это важно в программировании, праве, медицине, языках и технических темах.

### Ошибка 6. Использовать тест как наказание

Active recall должен быть регулярной низкострессовой частью обучения, а не угрозой. Иначе человек начинает избегать проверки.

### Ошибка 7. Просить ИИ до собственной попытки

ИИ может быстро создать ощущение понимания. Сначала прогноз, попытка и ошибка; потом ИИ как feedback.

### Ошибка 8. Гонять лёгкое ради уверенности

Если ответ уже легко вспоминается, дополнительные повторы подряд часто дают меньше пользы, чем повтор позже. Лучше добавить интервал, пример или применение.

### Ошибка 9. Делать паузу без последующего recall

Пауза полезна, если после неё есть попытка вспомнить. Иначе это просто отдых, который может быть нужен, но не является повторением.

### Ошибка 10. Учиться цепочкой коротких видео без сборки

Разрозненные видео могут разрушать связность. После каждого фрагмента нужен выход: идея по памяти, пример, вопрос, действие.

### Ошибка 11. Подменять применение пересказом

Для навыков пересказа мало. Нужно делать, проверять, ошибаться и исправлять.

### Ошибка 12. Мерить прогресс количеством часов

Лучше считать:

- сколько было самостоятельных попыток;
- сколько ошибок разобрано;
- что повторено через интервал;
- что теперь можно сделать без подсказки.

## 7-дневный старт

### День 1. Базовый цикл

После каждого учебного блока:

```text
закрыл источник -> 5 минут вспомнил -> проверил -> записал 1 пробел
```

### День 2. Вопросы

После материала формулировать 3-5 вопросов:

```text
что это?
как работает?
где пример?
где ограничение?
как применить?
```

### День 3. Интервалы

Вернуться к вопросам предыдущего дня без источника.

### День 4. Чистый лист

Одну тему восстановить на пустом листе:

```text
главная идея -> элементы -> связи -> пример -> ошибка -> ограничение
```

### День 5. Вариативность

Одну тему проверить в разных форматах:

```text
определение -> пример -> схема -> задача -> ошибка
```

### День 6. Interleaving

Взять 2-4 похожие темы и перемешать вопросы.

### День 7. Проверка системы

Ответить:

- какие темы вспоминаются без источника;
- какие только узнаются при чтении;
- какие карточки полезны;
- где нужен не конспект, а задача;
- где нужен feedback;
- что повторить на следующей неделе;
- что убрать.

## Минимальная система

Если нужна самая простая версия:

```text
1. Одна тема на блок.
2. После блока закрыть источник.
3. За 3-5 минут восстановить главное.
4. Ответить на 3 вопроса.
5. Проверить по источнику.
6. Одну ошибку исправить.
7. Одну важную идею повторить завтра.
8. Для навыка сделать маленькую задачу.
```

## Чек-лист хорошего active recall

Перед завершением блока проверь:

```text

3. Вопрос требовал вспоминания, а не узнавания?
4. Я проверил ответ?

6. Есть дата или условие повтора?
7. Есть связь с будущей задачей?

```

## Уровень доказательности

| Утверждение | Оценка |
|---|---|
| Active recall / retrieval practice улучшает долговременное запоминание по сравнению с пассивным перечитыванием | высокая |
| Practice testing полезен как способ обучения, а не только оценки | высокая |
| Feedback после попытки повышает качество обучения | высокая / умеренно высокая |
| Active recall помогает сложным концептам и переносу | умеренно высокая, зависит от задач |
| Spaced active recall лучше массированной зубрёжки для долгой памяти | высокая |
| Variable recall и interleaving могут помогать переносу | умеренно высокая |
| Self-explanation полезно после примеров и ошибок | умеренно высокая |
| Pre-testing всегда полезен независимо от условий | преувеличено |
| Любая карточка полезна | неверно |
| Active recall заменяет понимание, сон, задачи и feedback | неверно |

## Ограничения

Active recall - сильная техника, но не универсальная замена обучению.

Он не заменяет:

- первичное понимание;
- хорошие объяснения;
- примеры;
- практику задач;
- feedback;
- проекты;
- сон и восстановление;
- работу с тревогой;
- нормальную организацию времени;
- мотивационный смысл обучения.

Он может работать плохо, если:

- материал ещё совсем непонятен;
- вопросы слишком лёгкие;
- вопросы слишком трудные;
- нет проверки;
- ошибки не исправляются;
- всё сводится к карточкам;
- слишком много новых карточек;
- человек пытается вспомнить в состоянии сильного истощения;
- ИИ даёт готовый ответ до попытки;
- повторение превращается в ритуал без применения.

## Итоговый пользовательский протокол

Если внедрять только одну версию, используй эту:

```text
1. Изучи небольшой блок.
2. Закрой источник.
3. За 5 минут восстанови главную мысль, тезисы, пример и ограничение.
4. Ответь на 3-5 вопросов без подсказки.
5. Проверь по источнику, задаче, преподавателю или ИИ.
6. Одну ошибку преврати в вопрос, карточку или мини-задачу.
7. Повтори через 1 день, 3-4 дня и неделю.
8. Для навыков добавь задачу без подсказки.
9. Для похожих тем добавь interleaving.
10. Для ИИ сначала сделай свою попытку, потом проси feedback.
11. Лёгкое не добивай много раз подряд; вернись позже.
12. Если устал или не спал, уменьши сложность, но сохрани честную попытку.
```

Короткая рабочая формула:

```text
Сначала достань из памяти.
Потом проверь.
Потом исправь.
Потом вернись позже.
```
