# Anki: инструкция пользователя

## О чём эта инструкция

Anki - это программа для интервального повторения. Её главная польза не в том, что она хранит карточки, а в том, что она заставляет регулярно извлекать знания из памяти через увеличивающиеся интервалы.

Правильная роль Anki:

```text
не архив знаний,
а тренажёр spaced retrieval.
```

`Spaced retrieval` - это сочетание двух принципов:

- spaced repetition (интервальное повторение): возвращаться к материалу не подряд, а через интервалы;
- retrieval practice (практика извлечения): сначала пытаться вспомнить ответ без подсказки, потом проверять.

Если упростить:

```text
карточка -> попытка вспомнить -> проверка -> честная оценка -> повтор позже
```

Anki хорошо подходит для долговременного удержания фактов, терминов, различий, правил, примеров, ошибок и коротких принципов. Но Anki плохо работает, если использовать его как свалку конспектов, длинных цитат и готовых ответов, которые никогда не возвращают к задаче.

## Обновление 2026-06-01: уточнения по новым материалам из `4sort`

Новые материалы из `4sort` уточняют роль Anki так:

- Anki - это реализация spaced retrieval, а не способ добивать одну и ту же карточку до ощущения полной уверенности;
- лёгкие карточки не нужно повторять много раз подряд: если ответ быстрый и точный, следующий полезный повтор чаще всего должен быть позже;
- пауза, сон и интервалы важны не как ритуалы, а потому что после них проверяется доступность знания без подсказки;
- утреннюю сессию Anki можно использовать как короткую реактивацию вчерашнего материала, но не как замену полноценной практики;
- карточки после коротких видео стоит делать только после восстановления главной мысли, иначе Anki закрепляет фрагменты без связности;
- для языков и произношения аудиокарточки полезнее, если есть петля "образец -> попытка -> запись себя -> сравнение";
- после недосыпа Anki лучше делать мягче: меньше новых карточек, больше честного review, без попытки компенсировать сон кофеином.

## Главная мысль

Карточка нужна не для того, чтобы "сохранить информацию". Для этого есть конспект, заметка, документ, закладка или Obsidian.

Карточка нужна для другого:

```text
быстро и точно достать нужное знание из памяти,
когда источника перед глазами нет.
```

Хорошая карточка помогает:

- вспомнить термин;
- отличить похожие понятия;
- не повторить типовую ошибку;
- удержать правило;
- восстановить шаг алгоритма;
- применить принцип к задаче;
- проверить, что знание действительно доступно без подсказки.

Плохая карточка создаёт иллюзию обучения:

- содержит слишком много текста;
- копирует абзац из источника;
- проверяет сразу несколько идей;
- даёт расплывчатый вопрос;
- требует узнавания, а не вспоминания;
- не связана с применением;
- повторяется механически, без проверки понимания.

## Что Anki делает хорошо

Anki особенно полезен для материала, который должен быть доступен быстро и надолго.

Подходящие типы материала:

- термины и определения;
- иностранные слова и устойчивые выражения;
- формулы;
- команды;
- синтаксис;
- короткие правила;
- даты, имена, нормы, классификации;
- различия между похожими понятиями;
- типовые ошибки;
- причинно-следственные связи;
- диагностические признаки;
- минимальные примеры;
- вопросы для экзамена;
- короткие принципы из практики.

Примеры хороших задач для Anki:

```text
Вспомнить, чем A отличается от B.
Вспомнить, когда применяется правило.
Вспомнить типичную ошибку.
Вспомнить один пример.
Вспомнить первый шаг алгоритма.
Вспомнить, какой вопрос задать в похожей ситуации.
```

## Что Anki делает плохо

Anki не должен заменять:

- первичное понимание;
- чтение источника;
- нормальный конспект;
- решение задач;
- проекты;
- практику речи или письма;
- обратную связь;
- разбор ошибок;
- сон и восстановление;
- работу с мотивацией и вниманием.

Если человек только повторяет карточки, но не решает задачи, не пишет, не объясняет, не применяет и не получает feedback, обучение становится узким. Карточки помогают помнить элементы, но не всегда учат связывать их в действие.

Короткое правило:

```text
Anki поддерживает навык, но не заменяет практику навыка.
```

## Базовый рабочий цикл

### Шаг 1. Сначала понять материал

Не стоит сразу превращать непонятный материал в карточки.

Правильная последовательность:

```text
прочитать / посмотреть -> понять общий смысл -> закрыть источник -> попытаться вспомнить -> проверить -> сделать карточки только по важному
```

Если карточка создаётся до понимания, она часто превращается в механическую зубрёжку.

### Шаг 2. Выбрать, что действительно нужно помнить

В Anki отправлять не всё подряд, а только то, что:

- часто забывается;
- нужно применять без источника;
- является опорным понятием;
- путается с соседней идеей;
- уже привело к ошибке;
- нужно быстро достать в работе, учёбе или экзамене;
- помогает сделать следующую задачу.

Фильтр:

```text
Если я вспомню это через месяц, что мне это позволит сделать?
```

Если ответ неясен, карточку лучше не добавлять.

### Шаг 3. Сформулировать вопрос

Хорошая карточка начинается с вопроса. Вопрос должен вызывать попытку вспомнить, а не узнавание.

Плохо:

```text
Retrieval practice
```

Лучше:

```text
Почему retrieval practice обычно полезнее повторного чтения для долговременного запоминания?
```

Ещё лучше:

```text
После чтения главы я хочу закрепить материал. Почему сначала стоит ответить на вопросы без источника, а не перечитать главу?
```

### Шаг 4. Ответить коротко

Ответ должен быть достаточно коротким, чтобы его можно было проверить.

Хороший ответ:

- конкретный;
- своими словами;
- без длинного абзаца;
- содержит один главный смысл;
- при необходимости даёт мини-пример;
- отделяет правило от исключения.

Плохой ответ:

- длинный кусок текста;
- цитата без переработки;
- несколько разных идей;
- ответ "расскажи всё";
- формулировка, которую трудно оценить.

### Шаг 5. Повторять честно

Во время повторения порядок такой:

```text
прочитал вопрос -> закрыл доступ к ответу -> попытался вспомнить -> сформулировал ответ полностью -> открыл ответ -> сравнил -> поставил оценку
```

Нельзя сразу переворачивать карточку с мыслью "я примерно помню". Нужно сначала реально ответить.

### Шаг 6. Исправлять карточки

Если карточка регулярно не работает, проблема может быть не в памяти, а в карточке.

Карточку нужно переписать, если она:

- слишком широкая;
- содержит несколько вопросов;
- требует длинного ответа;
- сформулирована чужими словами;
- допускает несколько правильных ответов;
- проверяет не то, что нужно;
- повторяется, но не помогает действовать.

## Типы карточек

### 1. Basic: обычная карточка вопрос-ответ

Формат:

```text
Вопрос: Что такое spaced repetition?
Ответ: Повторение материала через распределённые интервалы, а не одним длинным блоком.
```

Подходит для:

- определений;
- коротких правил;
- фактов;
- принципов;
- вопросов "почему";
- вопросов "когда применять".

Хороший вариант:

```text
Вопрос: Почему интервальное повторение обычно лучше зубрёжки за один вечер?
Ответ: Интервалы заставляют снова извлекать материал из памяти и снижают иллюзию знакомости, поэтому знание лучше удерживается надолго.
```

### 2. Reversed card: двусторонняя карточка

Формат:

```text
Сторона A -> сторона B
Сторона B -> сторона A
```

Подходит для:

- слов и переводов;
- терминов и определений;
- символов и значений;
- дат и событий, если нужны оба направления.

Пример:

```text
Q: retrieval practice
A: практика извлечения из памяти
```

И обратная:

```text
Q: практика извлечения из памяти
A: retrieval practice
```

Ограничение: двусторонние карточки нужны не всегда. Если нужно только узнавать термин и применять идею, обратная карточка может быть лишней.

### 3. Cloze: карточка с пропуском

Формат:

```text
Интервальное повторение работает лучше, когда соединяется с {{c1::активным извлечением}}, а не только с перечитыванием.
```

Подходит для:

- формул;
- коротких определений;
- списков;
- терминов в контексте;
- фраз на иностранном языке;
- последовательностей шагов.

Хороший cloze проверяет важный элемент, а не случайное слово.

Плохо:

```text
{{c1::Anki}} - это программа для интервального повторения.
```

Слишком легко и почти не проверяет понимание.

Лучше:

```text
Anki полезен не как архив, а как инструмент {{c1::spaced retrieval}}.
```

### 4. Multiple cloze: несколько пропусков

Формат:

```text
Хороший цикл карточки: {{c1::вопрос}} -> {{c2::попытка вспомнить}} -> {{c3::проверка}} -> {{c4::повтор позже}}.
```

Использовать осторожно. Если в одной карточке слишком много пропусков, она становится перегруженной.

Правило:

```text
Одна карточка должна проверять одну мысль или один небольшой связанный набор.
```

### 5. Type-in: ввод ответа

Формат:

```text
Пользователь должен напечатать ответ.
```

Подходит для:

- правописания;
- иностранных слов;
- команд;
- синтаксиса;
- коротких формул;
- точных терминов.

Ограничение: ввод ответа полезен, когда важна точность формы. Для сложных концептов он может замедлять повторение без большой пользы.

### 6. Image occlusion: закрытие частей изображения

Формат:

```text
На схеме закрывается фрагмент, который нужно вспомнить.
```

Подходит для:

- анатомии;
- географии;
- схем;
- диаграмм;
- интерфейсов;
- архитектурных карт;
- таблиц;
- визуальных классификаций.

Риск: легко превратить в угадывание по расположению. Чтобы этого избежать, добавляй вопрос:

```text
Что закрыто и какую функцию выполняет?
```

А не только:

```text
Что находится в этом месте картинки?
```

### 7. Audio cards: аудиокарточки

Подходят для:

- языков;
- произношения;
- аудирования;
- музыки;
- распознавания звуков;
- диктовки.

Варианты:

- услышать слово -> вспомнить перевод;
- увидеть слово -> произнести;
- услышать фразу -> записать;
- услышать пример -> объяснить значение.

### 8. Example cards: карточки с примером

Формат:

```text
Вопрос: Какой принцип показывает этот пример?
Ответ: ...
```

Подходит для:

- права;
- программирования;
- медицины;
- психологии;
- математики;
- распознавания паттернов.

Пример:

```text
Вопрос: Ученик перечитывает конспект и чувствует, что всё знает, но не может ответить без подсказки. Какая проблема?
Ответ: Иллюзия знания: узнавание текста не равно извлечению из памяти.
```

### 9. Difference cards: карточки на различие

Это один из самых полезных типов.

Формат:

```text
Чем A отличается от B?
Когда использовать A, а когда B?
Какой признак помогает не спутать?
```

Примеры:

```text
Чем spaced repetition отличается от retrieval practice?
Чем перечитывание отличается от активного извлечения?
Чем пример отличается от задачи?
Чем понимание при чтении отличается от способности применить?
```

Карточки на различие особенно полезны там, где ошибки возникают из-за путаницы похожих понятий.

### 10. Error cards: карточки из ошибок

Формат:

```text
Вопрос: В какой ситуации я обычно ошибаюсь?
Ответ: Правильный принцип + короткий пример.
```

Шаблон:

```text
Ошибка:
Почему возникла:
Правило:
Как проверить в следующий раз:
```

Карточки из ошибок часто полезнее карточек из красивых фраз, потому что они связаны с реальной проблемой.

### 11. Scenario cards: карточки-кейсы

Формат:

```text
Дана ситуация. Нужно выбрать принцип, действие или объяснение.
```

Пример:

```text
Вопрос: После учебного видео хочется сразу сделать 20 карточек по всем тезисам. Что лучше сделать сначала?
Ответ: Закрыть источник, восстановить главные идеи, проверить пробелы и сделать карточки только по важным ошибкам, различиям и принципам.
```

Кейсы помогают переносить знание за пределы определения.

### 12. Process cards: карточки на шаги

Подходят для алгоритмов и процедур.

Плохо:

```text
Расскажи весь процесс создания карточек.
```

Лучше разбить:

```text
Какой первый шаг перед созданием карточек?
Что проверить перед добавлением карточки в Anki?
Что делать с карточкой, которая регулярно проваливается?
```

Для длинных процессов лучше использовать несколько карточек и отдельную схему в заметке.

### 13. Prompt cards: карточки с вопросом-подсказкой

Подходят для привычек мышления.

Примеры:

```text
Перед добавлением карточки в Anki какой вопрос стоит задать?
```

Ответ:

```text
Если я вспомню это через месяц, что мне это позволит сделать?
```

Такие карточки помогают запускать правильное действие, а не просто помнить факт.

## Принципы хорошей карточки

### Правило 1. Одна карточка - одна мысль

Плохо:

```text
Что такое Anki, spaced repetition, retrieval practice и FSRS?
```

Лучше:

```text
Что делает Anki?
Чем spaced repetition отличается от retrieval practice?
Что планирует FSRS?
Почему карточка должна проверять одну мысль?
```

### Правило 2. Вопрос должен быть точным

Плохо:

```text
Про Anki.
```

Лучше:

```text
Почему Anki не должен превращаться в архив прочитанного?
```

### Правило 3. Ответ должен быть проверяемым

Плохо:

```text
Ответь подробно и красиво.
```

Лучше:

```text
Назови 3 признака плохой карточки.
```

### Правило 4. Карточка должна требовать вспоминания

Если ответ можно узнать по форме вопроса, эффект слабее.

Плохо:

```text
Интервальное повторение - это повторение через интервалы?
```

Лучше:

```text
Почему повторение через интервалы обычно полезнее массовой зубрёжки?
```

### Правило 5. Карточка должна быть связана с будущим действием

Перед добавлением спроси:

```text
Где я буду использовать это знание?
Какую ошибку оно поможет предотвратить?
Какую задачу поможет сделать?
Какое различие поможет удержать?
```

Если ответа нет, возможно, карточка не нужна.

### Правило 6. Не копировать конспект

Anki не место для длинных фрагментов текста.

Конспект отвечает на вопрос:

```text
Где хранится объяснение?
```

Карточка отвечает на вопрос:

```text
Что я должен суметь вспомнить без объяснения перед глазами?
```

### Правило 7. Переписывать слабые карточки

Если карточка постоянно проваливается, не всегда нужно просто нажимать Again.

Возможные причины:

- вопрос слишком широкий;
- ответ слишком длинный;
- тема ещё не понята;
- не хватает примера;
- смешаны два похожих понятия;
- карточка проверяет не то, что важно;
- нет связи с задачей.

Иногда лучший ход - удалить карточку или заменить её двумя более точными.

## Что добавлять в Anki

### Добавлять

- повторяющиеся ошибки;
- различия между похожими понятиями;
- короткие правила;
- важные термины;
- минимальные примеры;
- формулы;
- команды;
- вопросы для экзамена;
- диагностические признаки;
- принципы, которые нужны в задачах;
- выводы, которые нужно вспоминать без источника.

### Не добавлять

- всё подряд из книги;
- длинные цитаты;
- очевидные факты;
- материал, который не нужен через месяц;
- идеи, которые пока не поняты;
- карточки "на всякий случай";
- большие концепции без разбивки;
- фразы, которые не ведут к действию;
- чужие готовые карточки без переработки.

## Как оценивать ответы

Смысл кнопок не в эмоции, а в качестве извлечения.

### Again

Используй, если:

- не вспомнил;
- вспомнил неверно;
- угадал;
- перепутал с похожим понятием;
- увидел ответ и понял, что сам бы не достал;
- ответ был правильным только частично, но смысл исказился.

### Hard

Используй, если:

- ответ правильный, но вспоминался тяжело;
- ответ был медленный;
- была неуверенность;
- пришлось долго вытаскивать;
- есть риск спутать с похожей темой;
- формулировка карточки заставила зависнуть.

Важное правило из локальных материалов:

```text
Верно, но медленно или неуверенно -> Hard.
```

Но не превращай это в микроменеджмент секунд. Время ответа зависит от усталости, шума, формулировки и состояния.

### Good

Используй, если:

- ответ вспомнился правильно;
- смысл точный;
- усилие было нормальным;
- пример или объяснение доступны;
- нет заметной путаницы.

### Easy

Используй, если:

- ответ очень лёгкий;
- вспоминается быстро и уверенно;
- карточка уже кажется чрезмерно частой;
- знание устойчиво.

Осторожность: не нажимай Easy только потому, что ответ знаком после просмотра. Easy уместно, когда ответ был доступен до проверки.

Если карточка стабильно `Easy`, не возвращай её вручную в текущую сессию ради спокойствия. Это типичный путь к оверлернингу: субъективная уверенность растёт быстрее, чем долгосрочная польза. Лучше доверить следующий контакт интервалу.

## FSRS и алгоритмы

FSRS - это современный планировщик интервального повторения. В общих чертах он оценивает состояние карточки через параметры вроде сложности, устойчивости памяти и вероятности вспомнить ответ в данный момент.

Практический смысл:

```text
алгоритм выбирает дату повтора,
но качество обучения всё равно зависит от качества карточки и честной оценки.
```

Что важно:

- FSRS не исправит плохой вопрос;
- FSRS не заменит понимание;
- FSRS зависит от честных нажатий Again, Hard, Good, Easy;
- настройки не должны становиться главным объектом внимания;
- лучше иметь 200 хороших карточек, чем 2000 слабых;
- если карточка бесполезна, алгоритм будет эффективно повторять бесполезность.

Практическое правило:

```text
Доверять планировщику, но регулярно чистить качество карточек.
```

## Скорость ответа и уверенность

Правильный ответ бывает разным.

### Быстро и уверенно

Обычно это `Good` или `Easy`, если ответ действительно точный.

### Правильно, но медленно

Обычно это `Hard`.

Причины медленного ответа:

- слабое извлечение;
- слишком широкий вопрос;
- плохая формулировка;
- усталость;
- путаница похожих понятий;
- нет примера;
- знание не связано с практикой.

Что делать:

```text
1. Нажать Hard.
2. Если карточка повторно медленная, переписать её.
3. Если есть путаница, сделать карточку на различие.
4. Если нет примера, добавить мини-кейс.
```

### Правильно, но через угадывание

Это не `Good`. Обычно это `Again` или повод переписать карточку.

Если ответ появился только после просмотра, значит, до проверки его не было.

## Anki и оверлернинг

Оверлернинг в Anki - это ситуация, когда карточка уже легко вспоминается, но человек продолжает гонять её в тот же день, чтобы почувствовать "теперь точно знаю".

Для фактов, терминов, определений и типовых различий это часто плохой обмен времени на пользу. После лёгкого правильного ответа лучше не добавлять дополнительные ручные повторы, а дать карточке уйти в следующий интервал.

Рабочее правило:

```text
легко вспомнил -> проверил точность -> Good/Easy -> не трогать до следующего показа
```

Когда не стоит убирать слишком быстро:

- ответ правильный, но медленный;
- карточка похожа на другую и легко спутать;
- есть экзамен завтра;
- карточка нужна для навыка, где требуется автоматизация;
- ответ вспоминается, но не применяется в задаче.

В этих случаях не надо просто повторять ту же карточку. Лучше сделать карточку на различие, добавить пример, решить мини-задачу или проверить применение.

## Вариации использования Anki

### 1. Anki для терминов

Подходит, если термин нужно быстро узнавать и объяснять.

Плохо:

```text
Q: Metacognition
A: длинный абзац из источника
```

Лучше:

```text
Q: Что такое metacognition?
A: Способность отслеживать и регулировать собственное мышление и обучение.
```

Ещё лучше:

```text
Q: Какой вопрос проверяет metacognition во время учёбы?
A: Что я действительно могу вспомнить и где я только чувствую знакомость?
```

### 2. Anki для различий

Это один из самых сильных сценариев.

Шаблон:

```text
Q: Чем A отличается от B?
A: A - ..., B - ..., главный признак различия - ...
```

Примеры:

```text
Чем spaced repetition отличается от retrieval practice?
Чем active recall отличается от rereading?
Чем карточка отличается от конспекта?
Чем ошибка памяти отличается от плохой формулировки карточки?
```

### 3. Anki для ошибок

Карточка из ошибки должна фиксировать не только правильный ответ, но и причину ошибки.

Шаблон:

```text
Q: В какой ситуации я путаю A и B?
A: Признак различия: ...
Типичная ошибка: ...
Проверка: ...
```

Хорошая карточка из ошибки помогает не повторить тот же сбой в следующей задаче.

### 4. Anki для экзамена

Подходит для:

- терминов;
- дат;
- формул;
- классификаций;
- вопросов билетов;
- коротких аргументов;
- типичных задач.

Но подготовка к экзамену не должна состоять только из карточек.

Нужны:

- пробные ответы вслух;
- письменные ответы;
- задачи в формате экзамена;
- смешанные темы;
- работа с ограничением времени;
- разбор ошибок.

### 5. Anki для языков

Варианты карточек:

- слово -> перевод;
- перевод -> слово;
- слово -> фраза;
- фраза -> значение;
- аудио -> записать;
- картинка -> слово;
- cloze в предложении;
- минимальные пары;
- типичная ошибка в употреблении.

Лучше учить не только отдельные слова, но и фразы.

Для произношения и речи добавляй не только "аудио -> значение", но и обратную связь:

```text
образец -> повторить вслух -> записать себя -> сравнить -> исправить один параметр
```

Параметры сравнения:

- звук;
- ударение;
- ритм;
- паузы;
- интонация.

Anki здесь хранит короткий стимул и расписание повтора, но само обучение происходит в попытке произнести, услышать себя и поправить конкретное отличие от образца.

Плохо:

```text
Q: run
A: бежать
```

Лучше:

```text
Q: to run out of time
A: не хватить времени
```

### 6. Anki для чтения сложных материалов

После статьи или главы не нужно делать карточку на каждый абзац.

Процесс:

```text
1. Прочитать блок.
2. Закрыть источник.
3. Восстановить главную мысль.
4. Найти 3-5 важных вопросов.
5. Сделать карточки только по тому, что нужно помнить или применять.
```

Типы карточек:

- главная идея;
- ключевое различие;
- термин;
- аргумент;
- ограничение;
- пример;
- типичная ошибка интерпретации.

### 7. Anki для практических навыков

Карточки должны вести обратно к действию.

Для навыка лучше сочетать:

- короткие карточки на правила;
- задачи без подсказки;
- разбор ошибок;
- проекты;
- повтор похожих задач.

Плохой сценарий:

```text
100 карточек по теме, но ни одной самостоятельной попытки применить.
```

Хороший сценарий:

```text
5 карточек по ключевым различиям + 3 задачи + разбор ошибки + повтор через неделю.
```

### 8. Anki для процедур

Процедуры лучше разбивать на небольшие вопросы.

Плохо:

```text
Опиши всю процедуру подготовки к экзамену.
```

Лучше:

```text
Какой первый шаг подготовки?
Когда делать пробный тест?
Что делать с ошибкой после теста?
Какой материал отправлять в Anki?
```

### 9. Anki для концептуальных тем

Для больших идей карточки должны проверять разные стороны:

- определение;
- механизм;
- пример;
- контрпример;
- ограничение;
- отличие от похожего;
- применение к задаче.

Например, по теме "интервальное повторение":

```text
Q1: Что такое spaced repetition?
Q2: Почему интервалы полезнее массового повторения?
Q3: Как spaced repetition связано с retrieval practice?
Q4: Когда карточки не заменяют практику?
Q5: Как понять, что карточка слишком широкая?
```

### 10. Anki с Obsidian

Obsidian и Anki решают разные задачи.

Obsidian:

- хранит объяснения;
- связывает идеи;
- помогает видеть контекст;
- содержит конспекты, схемы и источники.

Anki:

- проверяет извлечение;
- возвращает к важным вопросам;
- удерживает короткие элементы;
- показывает слабые места.

Рабочая связка:

```text
Obsidian: подробное объяснение и контекст.
Anki: короткий вопрос, который заставляет вспомнить ключевой элемент.
```

Если карточка требует длинного объяснения, лучше дать в ней короткий ответ и ссылку на заметку.

### 11. Anki с ИИ

ИИ может помогать, но не должен полностью заменять создание карточек.

Плохой процесс:

```text
Дать текст ИИ -> получить 50 карточек -> импортировать -> повторять без переработки.
```

Проблема: пропущена важная часть обучения - самостоятельный выбор, формулирование и сжатие смысла.

Лучший процесс:

```text
1. Дать ИИ текст.
2. Попросить выделить возможные темы для карточек.
3. Самостоятельно выбрать важные.
4. Самому написать вопрос и ответ.
5. Попросить ИИ проверить карточку на двусмысленность.
6. Исправить своими словами.
```

Шаблон запроса:

```text
Вот материал.
Выдели 10 возможных идей для карточек, но не пиши готовые карточки.
Для каждой идеи укажи, что она проверяет: термин, различие, ошибка, пример, применение.
```

Шаблон проверки:

```text
Проверь карточку:
Вопрос: ...
Ответ: ...
Есть ли двусмысленность?
Не слишком ли широкий вопрос?
Не проверяет ли она несколько идей сразу?
Как сделать её короче?
```

Важно: конкретное утверждение, что ИИ-карточки снижают удержание на определённый процент, в локальной заметке отмечено как непроверенное. Но общий принцип осторожности сильный: если ИИ делает всю работу за человека, снижается глубина переработки материала.

## Колоды, теги и структура

### Не дробить колоды слишком мелко

Слишком много узких колод создаёт блочную практику:

```text
сначала только тема A,
потом только тема B,
потом только тема C.
```

Это удобно, но может снижать способность различать похожие темы.

Лучше:

```text
широкая колода + теги
```

Например:

```text
Колода: Learning
Теги: retrieval, spacing, anki, errors, exams
```

Или:

```text
Колода: English
Теги: phrasal_verbs, pronunciation, grammar, writing
```

### Когда отдельная колода оправдана

Отдельная колода уместна, если:

- другой режим повторения;
- другой предмет;
- другой язык;
- другой экзамен;
- другая ежедневная квота;
- материал нельзя смешивать по практическим причинам.

Но если колода создана только потому, что "так аккуратнее", лучше использовать теги.

### Теги

Теги нужны для:

- поиска;
- фильтрации;
- недельной чистки;
- подготовки к экзамену;
- отбора слабых тем;
- временных проектов.

Не нужно превращать теги в сложную классификацию ради классификации.

Практичные теги:

```text
error
difference
definition
exam
hard
rewrite
project
source_article
```

## Ежедневный режим

### Минимальный режим

```text
10-15 минут в день.
Сначала review.
Потом новые карточки, если есть ресурс.
```

Главное правило:

```text
Сначала повторить старое, потом добавлять новое.
```

Если постоянно добавлять новые карточки и не закрывать повторы, Anki быстро становится источником долга.

### Утренний режим

Anki удобно использовать утром как короткую реактивацию, особенно если вчера был учебный блок.

Формат:

```text
1. 3-5 минут вспомнить вчерашнюю тему без конспекта.
2. Сделать due cards по этой теме.
3. Отметить 1-2 карточки, которые вскрыли реальный пробел.
4. Не добавлять новые карточки, пока не закрыты старые повторы.
```

Если утро - слабое окно по состоянию или хронотипу, оставь утром лёгкий review, а сложные новые карточки и разбор ошибок перенеси в своё окно ясности.

### Рабочий порядок сессии

```text
1. Открыть Anki.
2. Повторить due cards.
3. Отметить 1-3 проблемные карточки.
4. Переписать только самые важные.
5. Добавить новые карточки только после повторения.
6. Закрыть Anki до следующей сессии.
```

### Сколько новых карточек добавлять

Безопасное правило:

```text
добавлять меньше, чем кажется возможным.
```

Если Anki перегружает, почти всегда проблема в избытке новых карточек или плохом качестве старых.

Практический ориентир:

- на старте: 3-10 новых карточек в день;
- при высокой нагрузке: 0 новых, только повторы;
- перед экзаменом: больше, но с датой окончания;
- после перегруза: временно остановить новые карточки.

## Как делать карточки после материала

### После статьи

```text
1. Закрыть статью.
2. Восстановить главную мысль.
3. Записать 3-5 тезисов.
4. Проверить по источнику.
5. Выбрать 1-5 карточек:
   - термин;
   - различие;
   - ограничение;
   - пример;
   - ошибка.
```

### После видео

```text
1. Не делать карточки во время пассивного просмотра.
2. Смотреть один связный фрагмент, а не цепочку случайных роликов.
3. После видео закрыть источник.
4. Восстановить структуру, главную мысль и пример.
5. Ответить на 3-5 вопросов.
6. Вернуться к фрагментам, где были пробелы.
7. Сделать карточки только по важным выводам.
```

Если видео короткое, карточка должна фиксировать не сам фрагмент, а то, что из него нужно будет вспомнить и применить. Иначе Anki превращается в коллекцию разрозненных клипов.

### После ошибки

```text
1. Описать ошибку.
2. Найти правильный принцип.
3. Сформулировать вопрос, который предотвратит повтор.
4. Добавить мини-пример.
5. Решить похожую задачу без подсказки.
```

### После конспекта

Не надо превращать весь конспект в карточки.

Лучше:

```text
1. Выделить опорные понятия.
2. Найти различия и условия применения.
3. Сформулировать вопросы.
4. Убрать всё, что можно просто найти в заметке.
5. Оставить то, что нужно помнить без поиска.
```

## Как улучшать старые карточки

### Признаки слабой карточки

- проваливается много раз;
- вызывает раздражение;
- требует длинного ответа;
- непонятно, что считать правильным;
- содержит несколько вопросов;
- не связана с задачей;
- устарела;
- дублирует другую карточку;
- ответ выглядит как кусок конспекта.

### Что делать

Варианты:

- переписать вопрос;
- сократить ответ;
- разделить на 2-3 карточки;
- добавить пример;
- сделать карточку на различие;
- добавить контекст;
- удалить карточку;
- вернуть тему в конспект и сначала понять.

Шаблон переработки:

```text
Старая карточка:
Почему не работает:
Новая цель:
Новый вопрос:
Новый короткий ответ:
Нужен ли пример:
Нужна ли задача:
```

## Leech cards: карточки, которые постоянно проваливаются

Leech card - это карточка, которая много раз проваливается.

Не надо автоматически считать, что проблема в памяти. Возможные причины:

- тема не понята;
- карточка слишком широкая;
- ответ слишком длинный;
- нет контекста;
- смешаны похожие понятия;
- карточка не нужна;
- вопрос плохо сформулирован.

Порядок действий:

```text
1. Приостановить или пометить карточку.
2. Проверить исходную заметку.
3. Понять, что именно не вспоминается.
4. Разделить или переписать.
5. Если карточка не нужна, удалить.
```

Принцип:

```text
Постоянный провал карточки - повод чинить систему, а не просто давить на Again.
```

## Карточки и вариативность

Одна и та же идея не должна всегда проверяться в одной форме.

Например, тему "Anki как spaced retrieval" можно проверить так:

```text
Q1: Что такое spaced retrieval?
Q2: Почему Anki не должен быть архивом?
Q3: Какая ошибка возникает при длинных карточках?
Q4: Как исправить карточку, которая проверяет несколько идей?
Q5: Как Anki связан с retrieval practice?
Q6: Когда карточки не заменяют задачу?
```

Это variable retrieval - вариативное извлечение. Оно помогает не просто заучить формулировку, а доставать идею в разных контекстах.

Ограничение: не добавляй 10 карточек на одну идею, если она не важна. Вариативность нужна для опорных понятий и частых ошибок.

## Anki и interleaving

Anki частично создаёт interleaving, потому что карточки перемешиваются.

Но можно усилить эффект:

- не делать слишком узкие колоды;
- смешивать похожие темы;
- делать карточки на различия;
- использовать теги для временной подготовки;
- не повторять всегда в одном фиксированном порядке.

Interleaving особенно полезен, когда нужно различать похожие понятия.

Пример:

```text
retrieval practice
spaced repetition
interleaving
generation effect
self-explanation
```

Вместо отдельного блока по каждой теме можно смешивать вопросы, чтобы тренировать выбор нужного принципа.

## Anki и интервалы

Принцип интервалов:

```text
лучше несколько коротких возвратов во времени,
чем один длинный блок зубрёжки.
```

Примерная логика:

```text
сегодня -> завтра -> через 3-4 дня -> через неделю -> через 2-4 недели
```

Anki автоматизирует это расписание. Но автоматизация работает только если:

- карточки качественные;
- оценки честные;
- повторы выполняются регулярно;
- новые карточки не добавляются без контроля;
- сложные навыки дополнительно тренируются задачами.

Пауза между повторами полезна только если после неё есть попытка вспомнить. Поэтому не начинай с перечитывания ответа: сначала дай себе честную попытку, даже если карточка кажется забытой.

## Anki и сон

В локальных материалах подчёркивается: не нужно бесконечно повторять одну карточку в один день. Для долговременного удержания важны возвращения через время, а сон участвует в закреплении памяти.

Практический вывод:

```text
Лучше один честный повтор сегодня и возвращение завтра,
чем десять механических повторов подряд.
```

Но это не абсолютный запрет на повтор в тот же день. Если материал совсем новый, короткое первичное закрепление может быть полезно. Главное - не подменять интервальное повторение многочасовой зубрёжкой.

После плохой ночи:

```text
1. Сначала закрыть due cards.
2. Новые карточки уменьшить или поставить на 0.
3. Ошибки оценивать честно, а не списывать всё на усталость.
4. Сложные leech cards не чинить в состоянии сильной сонливости.
5. Кофеин использовать только как временную опору, не как замену сна.
```

Бодрость после кофеина не гарантирует качественное вспоминание. Если ответ вспоминается только через угадывание, это всё равно `Again` или повод переписать карточку.

## Anki и сложные навыки

Для сложных навыков Anki должен быть вспомогательным инструментом.

Рабочая связка:

```text
карточка -> задача -> feedback -> исправление -> новая похожая задача
```

Если карточка относится к навыку, добавь к ней вопрос:

```text
Какую маленькую задачу это знание помогает решить?
```

Пример:

```text
Q: Когда стоит сделать карточку из ошибки?
A: Когда ошибка повторяется, связана с важным принципом и её можно проверить коротким вопросом или мини-задачей.
```

И рядом задача:

```text
Найти одну ошибку за неделю и превратить её в карточку на различие.
```

## Частые ошибки

### Ошибка 1. Делать карточки на всё подряд

Это быстро создаёт долг. Карточки должны проходить фильтр полезности.

### Ошибка 2. Копировать абзацы

Длинный ответ не тренирует точное извлечение. Он провоцирует пассивное перечитывание.

### Ошибка 3. Делать слишком широкие вопросы

Плохо:

```text
Расскажи всё про интервальное повторение.
```

Лучше:

```text
Почему интервальное повторение полезнее массовой зубрёжки?
```

### Ошибка 4. Не отвечать до просмотра ответа

Если человек сразу смотрит ответ, Anki превращается в чтение карточек.

### Ошибка 5. Считать узнавание знанием

Фраза "я это видел" не равна "я могу вспомнить".

### Ошибка 6. Не исправлять карточки

Плохая карточка не становится хорошей от большого количества повторов.

### Ошибка 7. Добавлять слишком много новых карточек

Большой поток новых карточек создаёт очередь повторов и снижает качество ответов.

### Ошибка 8. Делать Anki главным занятием

Если Anki занимает всё учебное время, он начинает мешать практике.

### Ошибка 9. Чрезмерно настраивать алгоритм

Настройки важны, но обычно меньше, чем качество карточек и регулярность повторений.

### Ошибка 10. Полностью доверять ИИ-генерации

ИИ может помочь, но финальный вопрос и ответ лучше формулировать самому.

### Ошибка 11. Ручное добивание лёгких карточек

Если карточка уже легко вспомнилась, не нужно вручную повторять её ещё несколько раз подряд. Лучше доверить следующий показ интервалу.

### Ошибка 12. Делать карточки из коротких видео без сборки

Короткий ролик может дать полезный пример, но карточка должна появляться после восстановления структуры. Иначе повторяется фрагмент, а не знание.

### Ошибка 13. Использовать Anki как компенсацию недосыпа

После плохой ночи лучше снизить новые карточки и закрыть минимум review, чем пытаться "дожать" память кофеином и длинной сессией.

### Ошибка 14. Делать аудиокарточки без собственной попытки

Для языков недостаточно слушать образец и узнавать слово. Нужно произнести, записать себя или хотя бы вслух сравнить ответ с образцом.

## Практический протокол на 7 дней

### День 1. Навести фильтр добавления

Правило:

```text
Добавлять только карточки, которые помогают вспомнить важное различие, правило, ошибку или пример.
```

Задача:

```text
Не больше 5 новых карточек.
```

### День 2. Проверить качество вопросов

Для новых карточек спросить:

```text
Вопрос точный?
Ответ короткий?
Одна мысль?
Есть будущая польза?
```

### День 3. Оценивать медленные ответы как Hard

Правило:

```text
Правильно, но медленно или неуверенно -> Hard.
```

Отметить не больше 3 таких карточек.

### День 4. Переписать слабые карточки

Выбрать 1-3 проблемные карточки и:

- сузить вопрос;
- разделить;
- добавить пример;
- сделать карточку на различие;
- удалить, если не нужна.

### День 5. Сделать карточки из ошибок

Взять одну реальную ошибку и оформить:

```text
что я перепутал?
какой правильный принцип?
какой вопрос предотвратит повтор?
```

### День 6. Добавить вариативность

Для одной важной темы сделать не больше 2-3 разных карточек:

- определение;
- различие;
- пример;
- ограничение;
- применение.

### День 7. Проверить нагрузку

Оценить:

- сколько минут занимает Anki;
- сколько карточек реально полезны;
- какие карточки раздражают;
- какие не ведут к задачам;
- нужно ли уменьшить новые карточки;
- какие темы лучше вынести в практику, а не в Anki.

## Минимальная система

Если нужна самая простая рабочая версия:

```text
1. Каждый день 10-15 минут Anki.
2. Сначала повторы, потом новые.
3. Новые карточки только из важных ошибок, различий и правил.
4. Один вопрос - одна мысль.
5. Ответ короткий и своими словами.
6. Правильно, но тяжело - Hard.
7. Регулярные провалы - переписать или удалить.
8. Для навыков после карточки делать задачу.
```

## Шаблоны

### Шаблон обычной карточки

```text
Вопрос:
Короткий ответ:
Мини-пример:
Ограничение:
Источник / заметка:
```

### Шаблон карточки на различие

```text
Вопрос: Чем A отличается от B?
Ответ:
A - ...
B - ...
Главный признак различия - ...
Типичная ошибка - ...
```

### Шаблон карточки из ошибки

```text
Ошибка:
Почему возникла:
Правильный принцип:
Проверочный вопрос:
Похожая задача:
```

### Шаблон проверки карточки через ИИ

```text
Проверь карточку.

Вопрос:
Ответ:

Оцени:
1. Проверяет ли она одну мысль?
2. Нет ли двусмысленности?
3. Не слишком ли длинный ответ?
4. Требует ли она вспоминания, а не узнавания?
5. Как сделать её точнее?
```

### Шаблон разбора медленной карточки

```text
Карточка:
Ответ был правильный?
Было быстро / медленно / неуверенно / угадал:
Почему медленно:
Что изменить:
Нужна ли отдельная карточка на различие:
Нужна ли практическая задача:
```

## Чек-лист перед добавлением карточки

Перед добавлением спроси:

```text
1. Я понимаю материал?
2. Это нужно помнить без источника?
3. Вопрос проверяет одну мысль?
4. Ответ короткий?
5. Есть пример или контекст?
6. Это связано с задачей, ошибкой, экзаменом или будущим действием?
7. Я не копирую конспект?
8. Не лучше ли оставить это в Obsidian без карточки?
```

Если 3-4 ответа отрицательные, карточку лучше не добавлять.

## Чек-лист во время повторения

```text
1. Сначала ответил, потом открыл ответ.
2. Не засчитал узнавание как знание.
3. Нажал Again при ошибке или угадывании.
4. Нажал Hard при правильном, но тяжёлом ответе.
5. Отметил проблемные карточки.
6. Не стал исправлять всю базу сразу.
7. После Anki вернулся к реальной задаче.
```

## Чек-лист еженедельной чистки

Раз в неделю выбрать 10-20 проблемных карточек и проверить:

- удалить;
- переписать;
- разделить;
- добавить пример;
- сделать карточку на различие;
- связать с задачей;
- приостановить до понимания темы.

Не нужно чистить всю базу за один раз. Это быстро превращается в отдельный проект вместо обучения.

## Уровень доказательности

| Утверждение | Оценка |
|---|---|
| Интервальное повторение улучшает долговременное удержание | высокая |
| Retrieval practice полезна для запоминания | высокая |
| Anki может быть хорошей реализацией spaced retrieval | умеренно высокая |
| FSRS помогает планировать интервалы | умеренная / практическая, зависит от данных и использования |
| Любая карточка полезна | неверно |
| ИИ-карточки всегда хуже самостоятельных | слишком сильное утверждение |
| Самостоятельная формулировка карточек полезна для переработки материала | хорошо согласуется с принципами active recall и generation effect |
| Anki заменяет практические задачи | неверно |
| Избыточное повторение лёгких карточек подряд эффективно для долгой памяти | часто неверно или невыгодно по времени |
| Утренняя реактивация через Anki полезна сама по себе | предварительно; полезнее, когда это активное вспоминание с проверкой |
| Кофеин после недосыпа восстанавливает качество Anki-сессии | не подтверждено; сон не заменяется кофеином |

## Итоговый пользовательский протокол

```text
1. Сначала понять материал.
2. Закрыть источник и вспомнить главное.
3. Выбрать только важные элементы для долговременной памяти.
4. Сделать короткие карточки: одна мысль, точный вопрос, проверяемый ответ.
5. Повторять каждый день: сначала старые карточки, потом новые.
6. Оценивать честно: ошибка - Again, тяжёлое правильное вспоминание - Hard.
7. Переписывать карточки, которые регулярно не работают.
8. Использовать FSRS как планировщик, а не как замену мышления.
9. Использовать ИИ как редактора и проверяющего, а не как фабрику готовых карточек.
10. Для навыков связывать карточки с задачами, feedback и повторной попыткой.
11. Лёгкие карточки не добивать вручную; следующий полезный повтор обычно позже.
12. После недосыпа уменьшать новые карточки и не считать кофеин заменой восстановления.
```

Короткая формула:

```text
Anki нужен не для хранения знаний, а для регулярного честного вспоминания того, что должно работать без подсказки.
```
