# Конспекты и работа с материалом - инструкция пользователя

Режим: `ytb`

Тема: как читать, слушать, конспектировать, перерабатывать, проверять и возвращать материал в действие.

Основные источники: `Learning`, `Memory`, `4sort`.

---

## 0. Главная идея

Конспект - это не доказательство обучения.

Хороший конспект - это рабочий инструмент, который помогает:

- понять материал;
- выделить структуру;
- найти пробелы;
- сформулировать вопросы;
- вспомнить без подсказки;
- повторить через время;
- применить в задаче;
- вернуться к теме без повторного изучения с нуля.

Плохой конспект просто хранит текст.

Хороший конспект запускает учебное действие.

Короткая формула:

```text
материал -> вопросы -> обработка -> пересказ -> проверка -> повтор -> применение
```

Если после конспекта ты можешь только узнавать знакомые фразы, но не можешь объяснить, решить задачу или ответить на вопросы без подсказки, конспект пока не сделал свою работу.

---

## 1. Что считать качественной работой с материалом

Качественная работа с материалом включает пять слоев.

| Слой | Что происходит | Проверка |
| --- | --- | --- |
| Захват | Ты фиксируешь важное из лекции, книги, видео, статьи | Есть сырой материал, но не обязательно понимание |
| Структура | Ты отделяешь главное от деталей и связываешь идеи | Можно показать карту темы |
| Обработка | Ты пишешь своими словами, задаешь вопросы, делаешь схемы | Есть собственные формулировки |
| Воспроизведение | Ты закрываешь источник и вспоминаешь | Можно пересказать без подглядывания |
| Применение | Ты используешь идею в задаче, примере, решении | Есть проверяемый результат |

Главная ошибка:

```text
захватить материал и принять это за обучение
```

Правильнее:

```text
захватить материал -> переработать -> проверить -> применить
```

## 2. Карта методов

| Технология | Краткое описание | Когда использовать |
| --- | --- | --- |
| Cornell Notes / метод Корнелла | Страница с полем для заметок, вопросов и итогового резюме | Лекции, учебники, темы для повторения |
| SQ3R | Survey, Question, Read, Recite, Review: обзор, вопросы, чтение, пересказ, повтор | Учебники, главы, сложные тексты |
| K-W-L | Know, Want to know, Learned: что знаю, что хочу узнать, что узнал | Самообразование, новая тема, борьба с хаосом |
| Метод Фейнмана | Объяснить идею простыми словами, найти пробелы, уточнить | Проверка понимания |
| Метод Сократа | Продвигаться через точные вопросы, проверять основания и выводы | Критическое мышление, сложные идеи, самообман |
| Mind maps / ментальные карты | Визуально разложить тему на ветви, связи и уровни | Обзор, систематизация, поиск связей |
| Dual coding / двойное кодирование | Соединять текст и релевантную схему | Процессы, системы, причинность, сравнения |
| Рукописная обработка | Писать от руки для осмысления, схем, вопросов и отбора главного | Новая концепция, сложная связь, первичная обработка |
| PRTR / Preview-Read-Think-Repeat | Обзор структуры, чтение, осмысление, повтор | Легкий универсальный цикл для статей, глав и видео |
| Таксономия Блума | Проверять глубину владения: запомнил, понял, применил, проанализировал, оценил, создал | Диагностика уровня знания |
| Модель Колба | Опыт, рефлексия, вывод, новый эксперимент | Практические навыки, проекты, обучение через действие |

## 3. Главное разделение: захват, конспект, знание

Нужно различать три вещи.

### Захват

Это первичная фиксация.

Примеры:

- записал лекцию;
- выписал тезисы;
- сделал пометки в книге;
- сохранил цитату;
- набросал список терминов;
- сделал скриншот;
- сохранил ссылку.

Захват полезен, но сам по себе слаб.

### Конспект

Это обработанный материал.

В хорошем конспекте есть:

- тема;
- структура;
- ключевые идеи;
- вопросы;
- примеры;
- связи;
- собственные формулировки;
- краткое резюме;
- точки для повторения.

### Знание

Это способность:

- вспомнить без подсказки;
- объяснить своими словами;
- отличить от похожего;
- применить в задаче;
- заметить ошибку;
- перенести в новый контекст;
- создать на основе темы собственное решение.

Конспект - мост между источником и знанием, а не финальная точка.

---

## 4. Базовый цикл работы с любым материалом

Используй этот цикл для книги, лекции, видео, статьи, документации, курса или PDF.

```text
1. Preview: осмотреть структуру.
2. Question: сформулировать вопросы.
3. Read/Watch/Listen: изучить фрагмент.
4. Process: переработать своими словами.
5. Recall: закрыть источник и вспомнить.
6. Check: свериться и исправить.
7. Apply: сделать задачу, пример или вывод.
8. Review: повторить через время.
```

Минимальная версия:

```text
обзор -> вопрос -> фрагмент -> пересказ -> проверка
```

Если времени мало, не делай красивый конспект. Сделай хотя бы вопросы и пересказ.

---

## 5. Когда какой метод выбирать

| Ситуация | Лучший метод |
| --- | --- |
| Быстрая лекция, нужно успеть фиксировать | Метод предложений, поля, сырой конспект |
| Учебник или глава | SQ3R, PRTR, Cornell |
| Новая тема без структуры | K-W-L |
| Нужно проверить понимание | Фейнман |
| Нужно проверить утверждения и слабые выводы | Сократ |
| Много связей между идеями | Mind map |
| Есть процесс, система, график, причинность | Dual coding |
| Сложная концепция, нужно замедлиться | Рукописная обработка |
| Нужно понять, на каком уровне владеешь темой | Таксономия Блума |
| Нужно учиться на практике и ошибках | Модель Колба |
| Нужно повторять позже | Cornell, вопросы, карточки, active recall |
| Нужно быстро восстановить тему | Опорный конспект |

# Часть I. Cornell Notes / метод Корнелла

## 6. Суть метода Корнелла

Метод Корнелла - это система конспектирования, где страница делится на зоны:

- шапка: тема, дата, источник;
- правая большая часть: основные заметки;
- левая колонка: ключевые слова, вопросы, подсказки для повторения;
- нижний блок: краткое резюме.

Главная ценность метода не в разметке страницы, а в том, что он заставляет превратить конспект в инструмент самопроверки.

Формула:

```text
записал -> выделил ключи -> закрыл детали -> пересказал -> написал резюме -> повторил позже
```

## 7. Классические 5R

| Шаг | Что делать | Зачем |
| --- | --- | --- |
| Record | Записать важное в правую колонку | Захватить материал |
| Reduce | Сжать в ключевые слова и вопросы в левой колонке | Выделить структуру |
| Recite | Закрыть правую колонку и пересказать по левой | Проверить память и понимание |
| Reflect | Написать нижнее резюме | Осмыслить главный вывод |
| Review | Повторять через время | Закрепить и обнаружить забывание |

## 8. Разметка страницы

Простой вариант:

```text
Тема / дата / источник
--------------------------------------------------
Ключи и вопросы | Основные заметки
                |
                |
--------------------------------------------------
Итоговое резюме:
```

Не нужно obsessively вымерять сантиметры. Важнее функции блоков:

- справа - материал;
- слева - вопросы и крючки;
- снизу - сжатый смысл.

## 9. Как вести Cornell Notes на лекции

Во время лекции:

1. Пиши в правую колонку.
2. Не пытайся сразу сделать идеальную структуру.
3. Фиксируй тезисы, аргументы, примеры и определения.
4. Помечай непонятные места знаком `?`.
5. Не выделяй цветом всё подряд.

После лекции в тот же день:

1. Дополни пропуски.
2. Расшифруй сокращения.
3. В левую колонку вынеси вопросы.
4. Сформулируй 3-7 ключевых идей.
5. Закрой правую колонку и перескажи по левой.
6. Напиши нижнее резюме.

## 10. Как использовать Cornell Notes для чтения

Перед чтением:

- посмотри заголовки;
- сформулируй вопросы;
- выпиши термины, которые ожидаешь встретить.

Во время чтения:

- справа фиксируй ответы;
- не копируй абзацы;
- пиши своими словами;
- точные определения выписывай только там, где важна формулировка.

После чтения:

- преврати заголовки в вопросы;
- сделай самопроверку;
- напиши резюме;
- запланируй повтор.

## 11. Вариации метода Корнелла

### Мини-Корнелл

Подходит для короткого видео, статьи или одной темы.

```text
Вопросы:
- ...
- ...

Заметки:
- ...
- ...

Итог:
...
```

### Две колонки

Подходит для экзаменов и повторения.

```text
Вопрос | Ответ
```

Плюс: быстро превращается в active recall.

Минус: может стать слишком сухим, если не добавлять примеры.

### Три колонки

Подходит для сложных тем, математики, программирования, права, науки.

```text
Вопрос | Развернутый ответ | Короткий вывод / пример
```

### Cornell в Obsidian

```markdown
# Тема

Дата:
Источник:
Статус:

## Вопросы

- 

## Заметки

- 

## Примеры

- 

## Итоговое резюме

...

## Проверка

1.
2.
3.

## Следующий повтор

...
```

## 12. Ошибки метода Корнелла

1. Делать красивую таблицу, но не пересказывать.
2. Заполнять левую колонку сразу случайными словами.
3. Писать в правой колонке стенограмму.
4. Не возвращаться к конспекту.
5. Писать резюме сразу механически, без паузы.
6. Считать, что метод обязан подходить всем.
7. Тратить больше времени на формат, чем на понимание.

Главное правило:

```text
Cornell работает только тогда, когда левая колонка становится инструментом самопроверки.
```

# Часть II. SQ3R

## 13. Суть SQ3R

SQ3R - это система активного чтения:

```text
Survey -> Question -> Read -> Recite -> Review
```

По-русски:

```text
Обзор -> Вопросы -> Чтение -> Пересказ -> Повтор
```

Метод нужен, чтобы не читать учебник как роман, а работать с ним как с источником задач, вопросов и проверок.

## 14. Survey / обзор

Перед чтением посмотри:

- название главы;
- подзаголовки;
- введение;
- выводы;
- выделенные термины;
- таблицы;
- схемы;
- вопросы в конце главы;
- примеры;
- структуру аргумента.

Цель:

```text
создать карту, куда потом будут прикрепляться детали
```

Фишка:

```text
Не начинай читать с первой строки, пока не понял, что за текст перед тобой.
```

## 15. Question / вопросы

Преврати заголовки в вопросы.

Пример:

```text
Заголовок: "Рабочая память"
Вопрос: "Что такое рабочая память и чем она ограничена?"
```

Хорошие вопросы:

- Что это такое?
- Зачем это нужно?
- Как это работает?
- Чем отличается от похожего?
- Где применяется?
- Какая типичная ошибка?
- Как это проверяется?
- Что автор пытается доказать?

Вопросы превращают чтение в поиск смысла.

## 16. Read / чтение

Читай не весь массив сразу, а фрагментами.

После каждого фрагмента остановись и спроси:

```text
Что я только что прочитал?
Какая главная мысль?
На какой вопрос это отвечает?
Какой пример это показывает?
```

Если ответить не можешь, возвращайся к фрагменту.

## 17. Recite / пересказ

Закрой источник и ответь на вопросы своими словами.

Не так:

```text
Я вроде помню, о чём было.
```

А так:

```text
Я могу проговорить ответ полными предложениями.
```

Если слова не складываются, проблема не в памяти, а в неполной обработке.

## 18. Review / повтор

Повтор - это не перечитывание.

Лучший порядок:

```text
сначала вспомнить -> потом открыть источник -> потом исправить
```

Через время:

- через несколько часов;
- на следующий день;
- через 3-7 дней;
- перед применением;
- перед экзаменом.

## 19. Вариации SQ3R

### Для сложного учебника

1. Обзор главы.
2. Разбиение на маленькие куски.
3. Вопросы к каждому куску.
4. Ознакомительное чтение.
5. Вдумчивое чтение.
6. Пересказ без источника.
7. Конспект только после понимания.

### Для статьи

1. Прочитать заголовок, аннотацию, вывод.
2. Спросить: какую проблему решает автор?
3. Прочитать разделы выборочно.
4. Выписать тезис, метод, доказательства, ограничения.
5. Сформулировать 3 вопроса к статье.

### Для видео

1. Посмотреть описание и структуру.
2. Сформулировать вопрос.
3. Смотреть фрагментами по 5-15 минут.
4. Ставить паузу и пересказывать.
5. После видео написать 5 выводов.

### Для документации

1. Обзор страницы.
2. Выделить задачу, ради которой читаешь.
3. Найти пример.
4. Запустить или переписать пример.
5. Изменить пример.
6. Зафиксировать правило и ошибку.

## 20. Ошибки SQ3R

1. Пропустить Survey и сразу читать.
2. Не формулировать вопросы.
3. Читать слишком большие куски.
4. Путать Recite с перечитыванием.
5. Делать Review сразу с открытым текстом.
6. Думать, что SQ3R сделает сложный текст легким.

SQ3R не отменяет сложность. Он делает работу с ней управляемой.

# Часть III. K-W-L

## 21. Суть K-W-L

K-W-L - это таблица для самообучения:

| K - Know | W - Want to know | L - Learned |
| --- | --- | --- |
| Что я уже знаю | Что хочу узнать | Что узнал |

Метод особенно полезен, когда тема большая, а внешней программы нет.

Главная польза:

```text
K-W-L защищает от хаоса, иллюзии компетентности и информационного перегруза.
```

## 22. Колонка K: что я уже знаю

Запиши:

- факты;
- термины;
- примеры;
- опыт;
- гипотезы;
- что уже пробовал;
- что кажется понятным.

Важно: это не экзамен. Можно писать приблизительно, но помечать уверенность.

Пример:

```text
Тема: конспекты.
Я знаю: переписывать дословно плохо; вопросы помогают; схемы могут облегчать понимание.
```

## 23. Колонка W: что хочу узнать

Формулируй конкретные вопросы.

Не так:

```text
узнать всё про конспекты
```

Лучше:

```text
Какой метод выбрать для быстрой лекции?
Когда писать от руки, а когда печатать?
Как превратить конспект в повторение?
```

Хорошая колонка W ограничивает поиск.

## 24. Колонка L: что узнал

После изучения отвечай коротко.

Если не можешь ответить кратко, возможно, тема еще плохо разобрана.

Пример:

```text
Узнал: метод Корнелла полезен не формой страницы, а тем, что превращает заметки в вопросы для самопроверки.
```

## 25. Вариации K-W-L

### K-W-L для новой темы

Используй перед стартом курса.

```text
Что знаю?
Что хочу понять?
Что узнал после первого блока?
```

### K-W-L для недели

```text
K: что уже умею по теме?
W: какие пробелы закрываю на этой неделе?
L: что стало понятнее к концу недели?
```

### K-W-L для экзамена

Добавь четвертую колонку:

```text
Как меня могут проверить?
```

### K-W-L для Obsidian

```markdown
# K-W-L: [тема]

## K - что уже знаю
- 

## W - что хочу узнать
- 

## L - что узнал
- 

## Пробелы
- 

## Следующий шаг
- 
```

## 26. Ошибки K-W-L

1. Писать слишком широко.
2. Не возвращаться к таблице.
3. Заполнять `L` длинными копиями из источника.
4. Не превращать `L` в следующий `K`.
5. Не ограничивать глубину погружения.

Главное правило:

```text
K-W-L должна вести учебное движение, а не становиться еще одной папкой с заметками.
```

# Часть IV. Метод Фейнмана

## 27. Суть метода Фейнмана

Метод Фейнмана - это проверка понимания через простое объяснение.

Формула:

```text
выбрать тему -> объяснить простыми словами -> найти пробел -> уточнить -> объяснить точнее -> применить
```

Главный вопрос:

```text
Могу ли я объяснить это умному человеку без специальных знаний и не исказить смысл?
```

## 28. Когда использовать

Метод особенно полезен, если:

- тема кажется понятной, но смутной;
- много терминов;
- нужно подготовиться к экзамену;
- нужно объяснить другому;
- нужно проверить иллюзию понимания;
- нужно понять, где именно пробел.

## 29. Протокол Фейнмана

1. Выбери узкую тему.
2. Напиши объяснение простыми словами.
3. Подчеркни туманные слова.
4. Проверь, можешь ли привести пример.
5. Найди, где объяснение ломается.
6. Вернись к источнику точечно.
7. Перепиши объяснение.
8. Добавь пример, ограничение и типичную ошибку.

## 30. Проверочные вопросы Фейнмана

1. Что это такое?
2. Зачем это нужно?
3. Как это работает?
4. Где применяется?
5. С чем это часто путают?
6. Какой пример это показывает?
7. Какие есть ограничения?
8. Что будет, если применить неправильно?
9. Как объяснить это через аналогию?
10. Как проверить это действием?

## 31. Вариации метода Фейнмана

### Фейнман на 5 минут

```text
1 минута: тема.
2 минуты: объяснение простыми словами.
1 минута: где запнулся.
1 минута: следующий источник или задача.
```

### Фейнман для конспекта

После чтения конспекта закрой его и напиши:

```text
Если совсем просто, идея в том, что...
```

### Фейнман для программирования

```text
объяснил -> написал минимальный код -> сломал код -> объяснил ошибку -> исправил
```

### Фейнман для науки

```text
тезис -> метод -> данные -> вывод -> ограничение -> вопрос
```

### Фейнман для гуманитарных тем

```text
понятие -> контекст -> аргумент -> пример -> контраргумент -> вывод
```

## 32. Ошибки метода Фейнмана

1. Упрощать до неверности.
2. Заменять практику объяснением.
3. Игнорировать точные термины.
4. Не возвращаться к источникам.
5. Не проверять примером.
6. Думать, что гладкая речь равна пониманию.

Главное правило:

```text
Простое объяснение должно вести к точности, а не заменять её.
```

# Часть V. Метод Сократа

## 33. Суть метода Сократа

Метод Сократа - это движение через точные вопросы.

Он помогает:

- уточнить утверждение;
- отделить факт от интерпретации;
- найти скрытые предпосылки;
- проверить основания;
- увидеть альтернативы;
- найти противоречия;
- сделать вывод аккуратнее.

Формула:

```text
утверждение -> основания -> предпосылки -> альтернативы -> противоречия -> вывод -> действие
```

## 34. Сократический шаблон для обучения

```text
1. Что именно я утверждаю?
2. Что я точно знаю?
3. Что я предполагаю?
4. Какие слова здесь расплывчатые?
5. Какие факты подтверждают мою версию?
6. Какие факты ей противоречат?
7. Какие есть альтернативные объяснения?
8. Что я пока не понимаю?
9. Какой следующий проверяемый шаг?
```

## 35. Как применять к материалу

Если читаешь главу или статью, спрашивай:

- Что автор утверждает?
- На чем стоит это утверждение?
- Какие доказательства приведены?
- Есть ли альтернативная трактовка?
- Где границы метода?
- Что автор не обсуждает?
- Какие понятия нужно уточнить?
- Какой пример подтверждает тезис?
- Какой пример может его ослабить?

## 36. Как применять к конспекту

После конспекта добавь блок:

```markdown
## Сократическая проверка

1. Какой главный тезис?
2. На каких основаниях он держится?
3. Какие есть ограничения?
4. Что может быть преувеличением?
5. Что я могу проверить практикой?
```

## 37. Вариации метода Сократа

### Для самообмана

```text
Мне кажется, что я понял.
Что именно я могу сделать без подсказки?
```

### Для тревоги при учебе

```text
"Я плохо учусь" -> что значит плохо? по каким фактам? какие есть контрпримеры?
```

### Для научной статьи

```text
тезис -> метод -> данные -> ограничения -> альтернативы -> практический вывод
```

### Для AI-ответа

```text
Что здесь факт?
Что гипотеза?
Что совет?
Что нужно проверить по первоисточнику?
```

## 38. Ошибки метода Сократа

1. Превращать вопросы в допрос.
2. Использовать метод для самокритики, а не для прояснения.
3. Искать абсолютную уверенность там, где есть только вероятности.
4. Задавать много вопросов, но не делать следующий шаг.
5. Подменять изучение бесконечным анализом.

Главное правило:

```text
Сократ нужен не чтобы себя поймать, а чтобы сделать мысль точнее и практичнее.
```

# Часть VI. Mind maps / ментальные карты

## 39. Суть ментальных карт

Ментальная карта - это визуальная схема темы:

- центральная идея;
- ветви;
- подветви;
- связи;
- примеры;
- стрелки;
- группы;
- контрасты.

Она полезна, когда нужно увидеть общую картину и связи между элементами.

## 40. Когда делать mind map

Лучшее время - после первичного знакомства с материалом.

Не обязательно делать карту во время первой лекции. Часто лучше:

```text
сначала воспринять -> затем выделить структуру -> потом сделать карту
```

Карта особенно полезна для:

- большой темы;
- подготовки к экзамену;
- планирования курса;
- мозгового штурма;
- сравнения разделов;
- поиска пробелов;
- визуального повторения.

## 41. Как делать ментальную карту

1. В центр поставь тему.
2. Выдели 3-7 главных ветвей.
3. На каждую ветвь добавь понятия и примеры.
4. Покажи связи стрелками.
5. Отметь непонятные места.
6. Убери лишние детали.
7. Проверь: можешь ли объяснить тему по карте.

## 42. Пример структуры

```text
Конспекты
├─ Захват
│  ├─ лекция
│  ├─ видео
│  └─ книга
├─ Обработка
│  ├─ вопросы
│  ├─ схемы
│  └─ свои слова
├─ Проверка
│  ├─ пересказ
│  ├─ задачи
│  └─ карточки
└─ Повтор
   ├─ через день
   ├─ через неделю
   └─ перед применением
```

## 43. Вариации mind maps

### Карта главы

Центр: название главы.

Ветви:

- главные идеи;
- термины;
- примеры;
- вопросы;
- выводы.

### Карта курса

Центр: предмет.

Ветви:

- разделы;
- навыки;
- проекты;
- экзамены;
- слабые места.

### Карта проблемы

Центр: проблема.

Ветви:

- причины;
- последствия;
- гипотезы;
- решения;
- проверки.

### Карта пробелов

Центр: тема.

Ветви:

- понимаю;
- частично понимаю;
- не понимаю;
- нужно применить;
- нужно повторить.

## 44. Ошибки mind maps

1. Делать карту слишком рано, когда материала еще нет.
2. Писать на карте длинные абзацы.
3. Украшать больше, чем думать.
4. Делать карту без связей.
5. Не проверять себя по карте.
6. Использовать готовую карту вместо собственной обработки.

Главное правило:

```text
Ментальная карта должна показывать связи, а не просто красиво разбрасывать слова.
```

# Часть VII. Dual coding / двойное кодирование

## 45. Суть dual coding

Dual coding - это сочетание слов и визуальной структуры.

Важно: речь не про "визуальный стиль обучения". Речь не о том, что одним людям нужны картинки, а другим нет.

Смысл другой:

```text
текст + релевантная схема помогают построить более богатую модель материала
```

Картинка помогает не сама по себе, а когда показывает структуру.

## 46. Что можно визуализировать

| Тип материала | Лучший визуальный формат |
| --- | --- |
| Процесс | стрелки, этапы, блок-схема |
| Сравнение | таблица |
| Система | компоненты и связи |
| Причинность | цепочка причина -> механизм -> следствие |
| Иерархия | дерево |
| Цикл | круговая схема |
| Аргумент | тезис -> основания -> вывод |
| Ошибка | путь сбоя |
| Временная линия | timeline |
| Противопоставление | две колонки |

## 47. Протокол dual coding

1. Прочитай идею.
2. Сформулируй ее словами.
3. Выдели элементы.
4. Определи связи.
5. Нарисуй схему.
6. Подпиши схему.
7. Закрой источник.
8. Восстанови схему по памяти.
9. Объясни по схеме.

## 48. Drawing effect / рисование для запоминания

Рисование может помогать памяти, но не из-за художественности.

Работает смысловая переработка:

```text
понятие -> смысл -> структура -> рисунок -> воспроизведение
```

Рисовать полезно:

- понятия;
- определения;
- процессы;
- связи;
- системы;
- маршруты;
- причины и следствия.

Не нужно красиво.

Нужно функционально.

## 49. Ошибки dual coding

1. Использовать декоративные картинки.
2. Срисовывать без понимания.
3. Делать схему, которая не отвечает на вопрос.
4. Путать dual coding со стилями обучения.
5. Не проверять восстановление схемы по памяти.
6. Рисовать всё подряд.

Главное правило:

```text
Схема полезна, если по ней легче объяснить, вспомнить или применить идею.
```

# Часть VIII. Рукописная обработка

## 50. Ручка против ноутбука: точная позиция

Не стоит делать культ ручки.

Более точная позиция:

```text
Ручка часто полезна для первичного осмысления, схем, вопросов и отбора главного.
Клавиатура полезна для скорости, поиска, редактирования, кода и долговременной базы знаний.
```

Рукописный конспект может помогать, потому что письмо медленнее и заставляет выбирать главное. Но если человек от руки механически переписывает текст, пользы мало.

Печатный конспект может быть хорошим, если человек:

- не стенографирует;
- пишет своими словами;
- отключает отвлечения;
- структурирует;
- возвращается к записи;
- проверяет себя.

## 51. Когда лучше писать от руки

Пиши от руки, если нужно:

- понять новую концепцию;
- нарисовать схему;
- разобраться в формуле;
- выделить связи;
- сформулировать вопросы;
- замедлиться;
- убрать цифровые отвлечения;
- подумать на черновике.

## 52. Когда лучше печатать

Печатай, если нужно:

- быстро фиксировать много фактов;
- работать с кодом;
- хранить заметки в Obsidian;
- делать ссылки;
- редактировать;
- искать по базе;
- собирать материалы из разных источников;
- готовить карточки;
- писать длинный текст.

## 53. Лучший гибрид

```text
Ручка -> первичная обработка.
Клавиатура -> хранение, структура, ссылки, повторение.
```

Пример:

1. 10 минут от руки: схема, вопросы, пробелы.
2. Затем цифровая заметка: структура, ссылки, выводы.
3. После этого active recall: закрыть и вспомнить.
4. Потом карточки или задачи.

## 54. Ошибки рукописной обработки

1. Думать, что рука сама улучшает обучение.
2. Переписывать красиво, но без понимания.
3. Терять записи без цифрового следа.
4. Писать слишком медленно там, где нужна скорость.
5. Отказываться от ноутбука там, где он объективно лучше.

Главное правило:

```text
Важен не инструмент, а глубина переработки.
```

# Часть IX. PRTR / Preview-Read-Think-Repeat

## 55. Что такое PRTR

PRTR - это легкий цикл работы с материалом:

```text
Preview -> Read -> Think -> Repeat
```

По-русски:

```text
Обзор -> Чтение -> Осмысление -> Повтор
```

В локальных источниках точного отдельного файла по PRTR не найдено, но сам цикл хорошо согласуется с материалами по SQ3R, активному чтению, работе с учебником, пересказу и повторению.

PRTR можно использовать как упрощенную версию SQ3R.

## 56. Preview

Сначала посмотри:

- что за материал;
- сколько его;
- какая структура;
- где заголовки;
- где схемы;
- где выводы;
- какие термины встречаются;
- зачем ты это читаешь.

## 57. Read

Читай фрагментами.

Для сложного текста:

- маленький кусок;
- пауза;
- пересказ;
- вопрос;
- следующий кусок.

## 58. Think

После фрагмента подумай:

- что здесь главное;
- на какой вопрос это отвечает;
- с чем связано;
- что непонятно;
- какой пример можно привести;
- где это применить.

## 59. Repeat

Повторять нужно активно:

```text
закрыть источник -> вспомнить -> свериться -> исправить
```

## 60. Когда PRTR лучше SQ3R

PRTR удобнее, если:

- материал короткий;
- нет времени на полную SQ3R-схему;
- читаешь статью;
- смотришь видео;
- делаешь быстрый учебный блок;
- нужен простой ритуал.

SQ3R лучше, если:

- учебник сложный;
- глава большая;
- нужно готовиться к экзамену;
- важно построить систему вопросов.

# Часть X. Таксономия Блума

## 61. Зачем нужна таксономия Блума

Таксономия Блума помогает понять, на каком уровне ты реально владеешь темой.

Не всё одинаково:

```text
прочитал ≠ понял ≠ применил ≠ оценил ≠ создал
```

## 62. Шесть уровней

| Уровень | Проверка |
| --- | --- |
| Запомнить | Могу назвать термин, факт, определение |
| Понять | Могу объяснить своими словами |
| Применить | Могу использовать в задаче |
| Проанализировать | Могу разобрать структуру и связи |
| Оценить | Могу сравнить варианты и обосновать выбор |
| Создать | Могу сделать собственный продукт, решение, план |

## 63. Как применять к конспекту

После темы добавь блок:

```markdown
## Проверка по Блуму

### 1. Запомнить
Какие термины нужно знать?

### 2. Понять
Как объяснить своими словами?

### 3. Применить
Какая мини-задача?

### 4. Проанализировать
Из каких частей состоит тема?

### 5. Оценить
Когда метод подходит, а когда нет?

### 6. Создать
Что я могу сделать на основе темы?
```

## 64. Пример

Тема: Cornell Notes.

| Уровень | Проверка |
| --- | --- |
| Запомнить | Назвать блоки страницы и 5R |
| Понять | Объяснить, зачем нужна левая колонка |
| Применить | Сделать конспект по одной лекции |
| Проанализировать | Понять, где метод замедляет или помогает |
| Оценить | Решить, подходит ли он для конкретного курса |
| Создать | Сделать свой шаблон Cornell для Obsidian |

## 65. Ошибки применения Блума

1. Использовать как жесткую лестницу.
2. Думать, что всегда надо проходить все уровни.
3. Останавливаться на "понял".
4. Не делать задач на применение.
5. Не учитывать цель обучения.

Главное правило:

```text
Блум - это диагностическая карта, а не бюрократическая форма.
```

# Часть XI. Модель Колба

## 66. Суть модели Колба

Модель Колба описывает обучение через опыт.

Цикл:

```text
опыт -> рефлексия -> принцип -> новый эксперимент
```

Или проще:

```text
сделал -> разобрал -> понял принцип -> попробовал иначе
```

## 67. Четыре этапа

| Этап | Вопрос |
| --- | --- |
| Конкретный опыт | Что произошло на практике? |
| Рефлексивное наблюдение | Что я заметил? |
| Абстрактная концептуализация | Какой принцип можно извлечь? |
| Активное экспериментирование | Что я попробую иначе? |

## 68. Как связать Колба с конспектами

После практической задачи конспект должен фиксировать не только теорию, но и опыт.

Шаблон:

```markdown
# Разбор по Колбу

## 1. Опыт
Что я сделал?

## 2. Наблюдение
Что получилось?
Что не получилось?

## 3. Вывод
Какой принцип я извлек?

## 4. Эксперимент
Что проверю в следующий раз?
```

## 69. Где модель Колба особенно полезна

- программирование;
- письмо;
- языки;
- публичные выступления;
- спорт;
- дизайн;
- переговоры;
- преподавание;
- работа с AI;
- любые навыки, где нужна практика.

## 70. Ошибки модели Колба

1. Читать про навык без опыта.
2. Много практиковаться без анализа.
3. Делать выводы без новой проверки.
4. Считать "стили обучения" жесткой научной истиной.
5. Не фиксировать ошибки.

Главное правило:

```text
Каждый практический блок должен закончиться выводом и следующим экспериментом.
```

# Часть XII. Другие форматы конспектов

## 71. Метод предложений

Подходит для быстрого захвата лекции, видео или потока объяснений.

Правило:

```text
одно предложение = одна мысль
```

Пример:

```text
Метод Корнелла превращает конспект в инструмент самопроверки.
Левая колонка нужна для вопросов и ключей.
Правая колонка хранит основную запись.
Нижнее резюме сжимает смысл страницы.
```

Минус: слабая структура.

Что делать после:

```text
предложения -> главное -> вопросы -> резюме -> повтор
```

## 72. Опорный конспект

Опорный конспект - это компактная схема восстановления темы.

Пример:

```text
Cornell
  -> notes
  -> cues/questions
  -> summary
  -> recall
  -> review
```

Подходит для:

- повторения;
- экзамена;
- объяснения;
- большой темы;
- сложной системы.

Риск: сжать слишком сильно, так что по схеме ничего не восстановить.

## 73. Метод двух колонок

Формат:

```text
Вопрос | Ответ
```

Подходит для:

- экзаменов;
- терминов;
- самопроверки;
- учебных билетов;
- подготовки к собеседованию.

## 74. Табличный конспект

Подходит для сравнения.

Пример:

| Метод | Когда полезен | Риск |
| --- | --- | --- |
| Cornell | повторение | формат ради формата |
| SQ3R | чтение | пропуск вопросов |
| Mind map | связи | хаос ветвей |

## 75. Flow notes

Свободный конспект со стрелками, связями, вопросами, маленькими схемами.

Подходит, когда:

- автор говорит нелинейно;
- тема развивается через связи;
- нужно думать во время записи.

Риск: потом трудно читать, если не сделать второй проход.

## 76. Визуальные заметки

Это заметки, где используются:

- схемы;
- рисунки;
- стрелки;
- символы;
- пространственное расположение;
- блоки;
- маленькие иконки.

Художественное качество не важно.

Важно, чтобы рисунок помогал вспомнить смысл.

## 77. Индексовые карточки

Карточки лучше делать не из сырого текста, а из уже проработанного конспекта.

Хорошая карточка:

- короткая;
- проверяет один элемент;
- требует вспоминания;
- не содержит огромный ответ;
- связана с примером или применением.

Плохая карточка:

- переписанный абзац;
- слишком много фактов;
- неясный вопрос;
- ответ невозможно проверить.

## 78. Сводный конспект из нескольких источников

Подходит, если тема изучается по лекции, учебнику, статье и видео.

Структура:

```markdown
# Тема

## Главная идея

## Источник 1: что добавил

## Источник 2: что уточнил

## Источник 3: что оспорил

## Итоговая модель

## Пробелы

## Проверочные вопросы
```

Главный риск - собрать много источников и не сделать вывод.

# Часть XIII. Работа с разными типами материала

## 79. Лекция

До лекции:

- посмотри тему;
- выпиши 2-3 вопроса;
- подготовь шаблон.

Во время:

- не оформляй слишком много;
- не пытайся писать всё;
- фиксируй тезис, аргумент, пример, вопрос;
- непонятное помечай.

После:

- в тот же день расшифруй;
- дополни пропуски;
- выдели структуру;
- сделай вопросы;
- перескажи без подсказки.

## 80. Онлайн-лекция или видео

Правила:

1. Не конспектируй весь поток.
2. Смотри фрагментами.
3. Ставь паузы для пересказа.
4. Записывай вопросы.
5. Не переписывай слайды.
6. После видео сделай 5 выводов.
7. Если есть практика, сделай ее сразу.

Фишка:

```text
Один фрагмент видео = один вопрос + один вывод + одна проверка.
```

## 81. Учебник

Используй SQ3R или PRTR.

Порядок:

1. Обзор главы.
2. Вопросы.
3. Чтение маленькими кусками.
4. Пересказ.
5. Разбор терминов.
6. Конспект после понимания.
7. Вопросы для повторения.

Не выделяй маркером на первом чтении.

Сначала пойми, потом выделяй.

## 82. Сложный текст

Для сложного текста нормально не понять с первого раза.

Рабочий порядок:

1. Определи цель.
2. Разбей текст на куски.
3. Сделай обзор.
4. Сформулируй вопросы.
5. Прочитай ознакомительно.
6. Прочитай вдумчиво.
7. Разбери термины.
8. Ответь без источника.
9. Посмотри дополнительный источник.
10. Сделай конспект.

## 83. Научная статья

Шаблон:

```markdown
# Статья

## Вопрос исследования

## Главный тезис

## Метод

## Данные / аргументы

## Вывод

## Ограничения

## Что это меняет в моей модели?

## Что проверить дополнительно?
```

## 84. Документация и технический материал

Не ограничивайся чтением.

Порядок:

```text
прочитал -> запустил пример -> изменил пример -> сломал -> исправил -> записал правило
```

Конспект должен содержать:

- задачу;
- минимальный пример;
- типичную ошибку;
- исправление;
- правило;
- ссылку на источник;
- проверочную мини-задачу.

## 85. Книга нон-фикшн

Не обязательно конспектировать всю книгу.

Можно делать:

- карту глав;
- список идей;
- вопросы к автору;
- применимые принципы;
- спорные места;
- 10 выводов;
- 3 действия.

Хороший вопрос:

```text
Что я изменю в действиях после этой книги?
```

## 86. AI-ответ

AI-ответ нужно не просто сохранять, а проверять и перерабатывать.

Шаблон:

```markdown
## Что AI утверждает

## Что похоже на факт

## Что является гипотезой или советом

## Что нужно проверить по источникам

## Что я беру в свою систему

## Как проверю на практике
```

# Часть XIV. Что делать после конспекта

## 87. В тот же день

После лекции, чтения или видео:

1. Перечитай записи.
2. Расшифруй сокращения.
3. Дополни пропуски.
4. Выдели структуру.
5. Выпиши вопросы.
6. Напиши 3-7 главных выводов.
7. Сделай одну проверку по памяти.

Это особенно важно для сырых лекционных записей.

## 88. Через день

Не открывай конспект сразу.

Сначала:

```text
Что я помню?
Какие были главные идеи?
Какие вопросы остались?
Какой пример могу привести?
```

Потом открой конспект и проверь.

## 89. Через неделю

Сделай повторение на уровне структуры:

- восстанови mind map;
- ответь на вопросы Cornell;
- объясни тему методом Фейнмана;
- реши задачу;
- добавь или удали карточки.

## 90. Когда переписывать конспект

Переписывание полезно, если первичная запись:

- сделана на быстрой лекции;
- полна сокращений;
- плохо читается;
- содержит пробелы;
- создана в хаотичном порядке;
- должна стать основой для экзамена или работы.

Переписывание бесполезно, если:

- это механическое копирование;
- ты не меняешь структуру;
- ты не задаешь вопросов;
- ты не проверяешь себя;
- ты просто делаешь красиво.

Правильное переписывание:

```text
сырой текст -> структура -> свои слова -> схемы -> вопросы -> повтор
```

# Часть XV. Фишки и техники

## 91. Фишки перед изучением

1. Перед чтением напиши один вопрос, ради которого открываешь материал.
2. Сначала смотри структуру, потом детали.
3. Превращай заголовки в вопросы.
4. Перед сложной главой запиши, что уже знаешь.
5. Установи критерий завершения: не "прочитать", а "ответить на 5 вопросов".
6. Не начинай с красивого шаблона, если не понял задачу.
7. Для новой темы сделай K-W-L.
8. Для плотного текста заранее разбей его на куски.
9. Если есть вопросы в конце главы, прочитай их до главы.
10. Если материал пугает, сначала сделай обзор без цели понять всё.

## 92. Фишки во время чтения

1. Читай маленькими фрагментами.
2. После фрагмента закрывай текст и пересказывай.
3. Не выделяй маркером на первом чтении.
4. Помечай непонятное знаком `?`.
5. Отделяй тезис от примера.
6. Отделяй факт от интерпретации.
7. Веди список терминов отдельно.
8. Используй карандашные пометки, если боишься выделить лишнее.
9. Для сложного абзаца спроси: "какую проблему он решает?"
10. Если не можешь пересказать, не двигайся дальше механически.

## 93. Фишки конспектирования

1. Пиши своими словами.
2. Сохраняй точные цитаты только там, где важна формулировка.
3. Одно предложение - одна мысль.
4. Используй пустые строки между блоками.
5. Не украшай то, что еще не понял.
6. Делай поля для вопросов.
7. В конце заметки добавляй резюме.
8. Добавляй блок "что осталось непонятным".
9. Для сравнения используй таблицы.
10. Для процессов используй стрелки.
11. Для систем используй схемы.
12. Для больших тем используй mind map.
13. Для терминов делай карточки только после понимания.
14. Для ошибок делай блок "симптом -> причина -> исправление".
15. Для AI-ответов добавляй блок проверки.

## 94. Фишки после конспекта

1. Не открывай конспект сразу при повторении.
2. Сначала вспомни максимум сам.
3. Потом сверяйся.
4. Ошибки отмечай как пробелы, а не как провал.
5. Через день попробуй восстановить карту темы.
6. Через неделю объясни тему методом Фейнмана.
7. Преврати главные тезисы в вопросы.
8. Удали лишние карточки, если они проверяют мусор.
9. Добавь практическую задачу.
10. Если конспект не помогает вспомнить, переделай структуру.

## 95. Фишки для Obsidian

1. Одна заметка - один учебный фрагмент или одна идея.
2. В начале заметки пиши вопрос, на который она отвечает.
3. В конце заметки пиши следующий шаг.
4. Используй раздел `Проверка`.
5. Используй раздел `Связи`.
6. Не превращай Obsidian в склад цитат.
7. Связывай заметки по смыслу, а не ради количества ссылок.
8. Делай MOC/карту темы для больших областей.
9. Отмечай статус: черновик, обработано, нужно повторить, применено.
10. Добавляй дату следующего повторения, если тема важная.

## 96. Фишки против самообмана

1. Знакомый текст не равен знанию.
2. Красивый конспект не равен пониманию.
3. Подсветка не равна обработке.
4. Сохраненная ссылка не равна изучению.
5. Большая база заметок не равна компетентности.
6. AI-саммари не равно твоему пониманию.
7. Рисунок не полезен, если не показывает структуру.
8. Карточка вредна, если сделана из непонятного текста.
9. Переписывание полезно только при переработке.
10. Повторение полезно, если сначала вспоминаешь, а не смотришь.

# Часть XVI. Готовые сценарии

## 97. Сценарий: нужно быстро прочитать и понять главу

1. Сделай обзор.
2. Преврати заголовки в вопросы.
3. Читай по разделам.
4. После каждого раздела перескажи.
5. Запиши 5-7 выводов.
6. Ответь на вопросы без текста.
7. Сделай краткое резюме.

Метод: `SQ3R` или `PRTR`.

## 98. Сценарий: лекция была быстрой и конспект сырой

1. В тот же день расшифруй сокращения.
2. Дополни пропуски.
3. Разбей на блоки.
4. Переведи в Cornell или опорный конспект.
5. Добавь вопросы.
6. Перескажи без подсказки.

Метод: `метод предложений -> Cornell -> active recall`.

## 99. Сценарий: тема новая и непонятно, куда двигаться

1. Сделай K-W-L.
2. Запиши, что уже знаешь.
3. Сформулируй вопросы.
4. Изучи только материал, который отвечает на вопросы.
5. Заполни `Learned`.
6. Новый блок `Learned` преврати в следующий `Know`.

Метод: `K-W-L`.

## 100. Сценарий: кажется, что понял, но не уверен

1. Объясни тему простыми словами.
2. Найди туманные места.
3. Приведи пример.
4. Сравни с похожим понятием.
5. Вернись к источнику точечно.
6. Объясни снова.

Метод: `Фейнман`.

## 101. Сценарий: нужно критически разобрать материал

1. Сформулируй главный тезис.
2. Выпиши основания.
3. Отдели факты от интерпретаций.
4. Найди альтернативы.
5. Укажи ограничения.
6. Сформулируй аккуратный вывод.

Метод: `Сократ`.

## 102. Сценарий: нужно подготовиться к экзамену

1. Сделай список тем.
2. Для каждой темы проверь уровень по Блуму.
3. Cornell-вопросы преврати в билеты.
4. Сложные связи вынеси в mind map.
5. Термины вынеси в карточки.
6. Каждую тему перескажи без подсказки.
7. Реши смешанные задачи.

Методы: `Cornell`, `Bloom`, `mind map`, `active recall`.

## 103. Сценарий: нужно освоить практический навык

1. Сделай действие.
2. Зафиксируй опыт.
3. Разбери ошибку.
4. Выведи правило.
5. Попробуй иначе.
6. Запиши вывод в конспект.

Метод: `Колб`.

# Часть XVIII. Промты для AI

## 113. Промт: превратить материал в активный конспект

```text
Ты мой учебный методист. Помоги превратить материал в активный конспект.
```

```text
3. Предложи первые источники/типы источников.
```

# Часть XIX. 7-дневный эксперимент

## 122. Как проверить систему на себе

Не внедряй все методы сразу.

На неделю выбери:

- один метод чтения;
- один метод конспекта;
- один метод проверки.

Например:

```text
SQ3R + Cornell + Фейнман
```

Или:

```text
K-W-L + mind map + Bloom
```

## 123. План на 7 дней

| День | Действие |
| --- | --- |
| 1 | Выбрать тему и сделать K-W-L или Preview |
| 2 | Прочитать/посмотреть первый фрагмент через SQ3R/PRTR |
| 3 | Сделать Cornell или структурный конспект |
| 4 | Проверить понимание методом Фейнмана |
| 5 | Сделать mind map или схему dual coding |
| 6 | Проверить тему по Блуму или применить на задаче |
| 7 | Сделать обзор: что оставить, что убрать, что повторить |

## 124. Метрики

| Метрика | Вопрос |
| --- | --- |
| Понимание | Могу объяснить своими словами? |
| Память | Могу вспомнить без источника? |
| Структура | Вижу связи между частями? |
| Пробелы | Понимаю, что именно непонятно? |
| Применение | Есть задача, пример или действие? |
| Время | Метод не съедает слишком много ресурса? |
| Повтор | Есть вопросы или карточки для возвращения? |

## 125. Решение после недели

Оставить метод, если:

- он помогает понять;
- он дает проверочные вопросы;
- после него проще повторять;
- он не превращается в украшательство;
- он ведет к действию.

Изменить метод, если:

- он полезен, но слишком тяжел;
- работает только для некоторых материалов;
- требует адаптации под Obsidian или бумагу.

Убрать метод, если:

- он создает видимость работы;
- ты больше оформляешь, чем понимаешь;
- он не помогает вспомнить;
- он мешает практике.

# Часть XX. Чего не стоит делать

## 126. Главные ошибки

1. Переписывать текст дословно.
2. Выделять маркером на первом чтении.
3. Считать лекционные записи финальным конспектом.
4. Не возвращаться к конспекту в тот же день.
5. Делать карточки из непонятного материала.
6. Собирать источники без итоговой модели.
7. Перечитывать вместо активного припоминания.
8. Делать mind map без понимания.
9. Делать красивые схемы без проверки.
10. Использовать AI вместо собственного пересказа.
11. Делать слишком длинные заметки без структуры.
12. Хранить цитаты без ответа, зачем они нужны.
13. Не фиксировать непонятные места.
14. Не применять материал.
15. Оценивать успех количеством страниц.

## 127. Что особенно важно учесть

1. Конспект должен быть удобен тебе, а не выглядеть правильно для других.
2. Любой метод можно адаптировать.
3. Чем сложнее материал, тем меньше фрагмент для одного цикла.
4. Вопросы важнее подсветки.
5. Пересказ важнее перечитывания.
6. Схема важна только при смысловой связи.
7. Ручка полезна для мышления, клавиатура - для системы.
8. Повторение нужно делать через вспоминание.
9. Практические темы требуют задач, а не только конспектов.
10. Хорошая заметка должна возвращать к действию.

# Часть XXI. Быстрые рецепты

## 128. Если есть 10 минут

```text
1. Обзор.
2. Один вопрос.
3. Один фрагмент.
4. Пересказ.
5. Один вывод.
```

## 129. Если есть 30 минут

```text
5 минут - обзор и вопросы.
15 минут - чтение/просмотр.
5 минут - пересказ без источника.
5 минут - резюме и следующий шаг.
```

## 130. Если есть 60 минут

```text
10 минут - SQ3R Survey + Question.
25 минут - чтение фрагментов.
10 минут - Cornell/структурный конспект.
10 минут - Фейнман или проверочные вопросы.
5 минут - план повторения.
```

## 131. Если материал очень сложный

```text
цель -> маленький кусок -> обзор -> вопрос -> чтение -> термин -> пересказ -> пауза -> повтор
```

Не пытайся победить сложный текст одним длинным заходом.

## 132. Если нужно готовиться к экзамену

```text
список тем -> Bloom -> вопросы -> Cornell -> active recall -> смешанные задачи -> повтор
```

## 133. Если нужно сделать заметку в Obsidian

```text
вопрос -> краткий ответ -> структура -> пример -> связи -> проверка -> следующий шаг
```

## 135. Ограничение по PRTR

По точному запросу `PRTR / Preview-Read-Think-Repeat` отдельного локального файла в папках `Learning`, `Memory`, `4sort` не найдено.

Поэтому в инструкции PRTR описан как практическая компактная версия цикла, поддержанная соседними локальными материалами:

- SQ3R;
- активное чтение;
- работа с учебником;
- пересказ;
- повторение;
- обработка информации.

Это не заявляется как отдельная доказательная техника из локальной базы, а используется как удобная рабочая аббревиатура.

# Часть XXIII. Финальная формула

## 136. Самое короткое правило

```text
Не собирай материал. Работай с ним.
```

## 137. Рабочая формула конспекта

```text
вопрос -> своими словами -> схема -> проверка -> применение -> повтор
```

## 138. Проверка качества

Конспект хороший, если по нему можно:

1. быстро восстановить структуру темы;
2. ответить на вопросы без подсказки;
3. найти пробелы;
4. повторить через время;
5. применить идею в задаче;
6. объяснить другому человеку;
7. решить, что делать дальше.

Если конспект только выглядит аккуратно, но не помогает ни вспомнить, ни применить, его нужно переработать.
