# Spaced repetition - инструкция пользователя

Режим: `ytb`  
Тема: интервальное повторение, распределенная практика, spaced retrieval, Anki/FSRS, карточки, вариативное повторение, successive relearning.  
Основные источники: `Learning`, `Memory`, `4sort`.

---

## 1. Короткая суть

**Spaced repetition** - это не "повторять много раз". Это система возвращения к материалу через интервалы, обычно с постепенным увеличением пауз.

Но сама интервальность - только половина метода.

Рабочая формула такая:

```text
понял -> вспомнил без подсказки -> проверил -> исправил -> повторил позже -> применил
```

Если убрать активное вспоминание, получится просто расписанное перечитывание. Оно может быть лучше полного хаоса, но слабее, чем попытка достать знание из памяти.

Главная мысль:

```text
Spaced repetition отвечает на вопрос "когда повторять".
Active recall / retrieval practice отвечает на вопрос "что делать при повторении".
Feedback отвечает на вопрос "не закрепляю ли я ошибку".
Практика применения отвечает на вопрос "могу ли я этим пользоваться".
```

---

## 2. Что считать spaced repetition

Spaced repetition - это семейство техник, а не одна программа.

К нему относятся:

- повторение по расписанию;
- карточки с интервалами;
- Anki, FSRS, SM-2, SuperMemo и аналоги;
- бумажная система Лейтнера;
- короткие тесты через несколько дней;
- повторение ошибок из журнала;
- возвращение к конспекту через 1, 3, 7, 14 дней;
- spaced retrieval - извлечение из памяти через интервалы;
- successive relearning - повторное достижение критерия знания через несколько сессий;
- distributed practice - распределенная практика вместо одного большого рывка;
- interleaved spaced practice - интервальное повторение с перемешиванием похожих тем;
- variable retrieval - проверка одной идеи разными способами.

Если совсем коротко:

```text
не один длинный подход, а несколько возвращений;
не узнавание текста, а вспоминание;
не склад карточек, а система проверяемых попыток.
```

---

## 3. Чем spaced repetition отличается от похожих методов

| Метод | Что делает | Пример |
|---|---|---|
| Spaced repetition | Разносит повторения во времени | сегодня, завтра, через 3 дня, через неделю |
| Distributed practice | Разносит учебные сессии во времени | 4 занятия по 30 минут в разные дни вместо 2 часов подряд |
| Retrieval practice | Заставляет вспоминать без подсказки | закрыть конспект и ответить на вопрос |
| Active recall | Практическая форма retrieval practice | пересказ, карточка, тест, задача |
| Testing effect | Проверка укрепляет память, а не только измеряет ее | мини-тест после темы |
| Successive relearning | Возврат к материалу до заданного уровня точности | снова довести карточки до 90-100% через несколько дней |
| Interleaving | Перемешивает похожие темы или задачи | interface, struct, pointer receiver, mock вперемешку |
| Variable retrieval | Проверяет одну идею разными формами | определение, пример, антипример, задача, ошибка |
| Overlearning | Повторение уже легкого материала | полезно ограниченно, но часто тратит время |

Важно не смешивать эти вещи в одну кашу. Лучший результат обычно дает связка:

```text
spacing + retrieval + feedback + применение
```

## 4. Что подтверждается хорошо, а где нужна осторожность

### Хорошо подтверждается

- Распределенная практика обычно лучше массированной зубрежки для долговременного удержания.
- Активное вспоминание обычно сильнее пассивного перечитывания.
- Повторение через интервалы особенно полезно для фактов, терминов, понятий, правил, процедур и экзаменационного материала.
- Обратная связь важна: после попытки вспомнить нужно свериться с источником, решением, тестом, преподавателем, документацией или эталоном.
- Карточки полезны, когда проверяют конкретное знание, различие, ошибку или применение.

### Требует осторожности

- Нет универсального расписания, которое идеально для всех тем и людей.
- Anki/FSRS не спасают плохие карточки.
- Для сложных навыков интервальное повторение не заменяет практику.
- Слишком раннее перемешивание тем может перегрузить новичка.
- Микросессии по 3-7 минут полезны для повторения, но не заменяют глубокую работу.
- Вариативность контекста может помогать переносу, но ее нужно дозировать.

Точная формулировка:

```text
Spaced repetition - сильный принцип для долговременной памяти.
Но конкретная реализация зависит от цели, материала, качества вопросов и наличия практики.
```

## 5. Когда использовать spaced repetition

Использовать, если материал нужно помнить не только сегодня.

Подходит для:

- иностранных слов;
- терминов;
- дат, имен, формул;
- команд и синтаксиса;
- профессиональных понятий;
- медицинских, юридических, технических фактов;
- экзаменационных вопросов;
- типовых ошибок;
- различий между похожими понятиями;
- алгоритмов действий;
- чек-листов;
- процедур;
- коротких объяснений;
- принципов, которые надо применять позже.

Подходит с оговорками:

- программирование;
- математика;
- system design;
- письмо;
- публичные выступления;
- аналитика;
- юридические кейсы;
- клиническое мышление;
- управленческие решения.

Здесь карточки помогают держать в памяти элементы, но навык строится через задачи, проекты, кейсы, обратную связь и применение.

Плохо подходит как единственный метод для:

- творческих проектов;
- физического навыка;
- сложного социального поведения;
- дебага реальной системы;
- написания большой работы;
- переговоров;
- глубокого проектирования.

В этих случаях spaced repetition может поддерживать память, но не должен становиться главным учебным действием.

## 6. Главный протокол

### Шаг 1. Выбрать знание, которое действительно нужно помнить

Не все заслуживает карточки.

Хорошие кандидаты:

- часто используемое правило;
- определение, без которого трудно думать дальше;
- отличие двух похожих понятий;
- ошибка, которую вы уже сделали;
- команда, которую нужно вспоминать без поиска;
- принцип, который понадобится в задачах;
- формула или структура ответа;
- вопрос, который может встретиться на экзамене или собеседовании.

Плохие кандидаты:

- случайная интересная деталь;
- длинный абзац из источника;
- цитата без будущего применения;
- то, что легко находится и не нужно держать в голове;
- чужая формулировка, которую вы не понимаете.

### Шаг 2. Сначала понять, потом повторять

Spaced repetition не предназначен для того, чтобы механически заучивать непонятое.

Перед карточкой ответьте:

- что это значит простыми словами;
- зачем это нужно;
- где это применяется;
- с чем это можно спутать;
- какой есть пример;
- какое ограничение у идеи.

Если на эти вопросы нет ответа, сначала доработайте понимание.

### Шаг 3. Сделать вопрос, а не выписку

Плохой вариант:

```text
Retrieval practice - это активное извлечение информации из памяти.
```

Лучше:

```text
Q: Чем retrieval practice отличается от перечитывания?
A: При retrieval practice я сначала пытаюсь достать ответ из памяти без подсказки; при перечитывании я в основном узнаю знакомый текст.
```

Еще лучше:

```text
Q: Почему перечитывание может создавать иллюзию знания?
A: Текст кажется знакомым, но это не доказывает, что я смогу воспроизвести идею без подсказки или применить ее в задаче.
```

### Шаг 4. Повторять через активное вспоминание

Правило:

```text
сначала ответ -> потом проверка
```

Не наоборот.

Порядок:

1. Увидеть вопрос.
2. Закрыть источник и ответ.
3. Сформулировать ответ вслух, письменно или в голове.
4. Открыть правильный ответ.
5. Сравнить.
6. Исправить ошибку.
7. Решить, когда повторять снова.

Если вы открыли ответ сразу, это было узнавание, а не тренировка извлечения.

### Шаг 5. Дать обратную связь

Без feedback можно закреплять ошибку.

Источники обратной связи:

- правильный ответ на карточке;
- учебник;
- документация;
- компилятор;
- тест;
- решение задачи;
- преподаватель;
- эксперт;
- эталонный пример;
- собственный журнал ошибок.

После ошибки спросите:

- я не знал факт;
- я перепутал два похожих понятия;
- вопрос был слишком широкий;
- ответ был написан чужим языком;
- мне не хватает примера;
- мне нужна задача, а не карточка;
- это вообще важное знание или лишняя деталь.

### Шаг 6. Вернуться позже

Стартовый шаблон:

```text
день 0: изучил и сделал первое вспоминание
день 1: повторил
день 3: повторил
день 7: повторил
день 14: повторил
день 30: проверил удержание
```

Это не закон. Это стартовая сетка.

Если легко - интервал можно увеличить.  
Если трудно - интервал сократить или улучшить карточку.  
Если совсем не вспоминается - вернуться к объяснению и примеру.

### Шаг 7. Добавить применение

Для важного материала карточки должны вести к действию.

Примеры:

- не только "что такое interface в Go", а написать маленький пример;
- не только "что такое active recall", а провести мини-экзамен по теме;
- не только "что такое spaced repetition", а составить расписание повторения;
- не только "что такое ошибка", а объяснить, почему вы ее сделали;
- не только "что говорит статья", а применить идею к своему учебному плану.

### Шаг 8. Чистить систему

Раз в неделю:

- удалить лишние карточки;
- объединить дубли;
- переписать слишком широкие вопросы;
- добавить примеры к провальным карточкам;
- приостановить карточки, которые не служат цели;
- вынести сложный материал в задачу или конспект;
- проверить, не вытесняет ли Anki реальную практику.

## 7. Расписания повторения

Точные интервалы не универсальны. Ниже - рабочие схемы, которые можно адаптировать.

### 7.1. Базовая схема для большинства тем

```text
0 день: изучение + первый recall
1 день: повтор без источника
3 день: повтор + исправление
7 день: повтор + пример
14 день: повтор + применение
30 день: контроль
```

Подходит для:

- учебных тем;
- статей;
- лекций;
- терминов;
- рабочих процедур;
- подготовки без жесткого дедлайна.

### 7.2. Схема "быстро стабилизировать"

```text
сразу после изучения
через несколько часов
на следующий день
через 2-3 дня
через неделю
```

Использовать, если материал новый, сложный и быстро распадается.

Ограничение: не повторяйте одну и ту же легкую карточку много раз в день. Это часто превращается в прокручивание в рабочей памяти, а не в полезное долговременное укрепление.

### 7.3. Схема перед экзаменом

Если до экзамена 14 дней:

```text
день 1: собрать список вопросов
день 2: первый полный recall
день 4: повтор слабых тем
день 7: смешанный тест
день 10: применение / задачи / билеты
день 12: слабые места
день 13: легкий контроль
день 14: сон, короткое повторение, без ночной зубрежки
```

Если времени мало:

```text
сначала самые вероятные и базовые вопросы;
потом ошибки;
потом похожие темы, которые легко спутать;
потом краткое повторение;
не пытаться за ночь создать идеальную систему.
```

### 7.4. Схема для долговременного знания

```text
1 день
3-4 дня
7-10 дней
2-3 недели
1 месяц
2-3 месяца
6 месяцев
по необходимости
```

Использовать для:

- языков;
- профессии;
- базовой теории;
- команд;
- принципов, к которым нужно возвращаться годами.

### 7.5. Схема для навыков

Для навыков интервал должен повторять не только вопрос, но и действие.

```text
день 0: разобрать пример
день 1: восстановить пример без подсказки
день 3: решить похожую задачу
день 7: решить задачу в другом контексте
день 14: найти ошибку или объяснить решение
день 30: применить в мини-проекте
```

Подходит для программирования, математики, письма, права, анализа кейсов.

### 7.6. Микросхема на 3-7 минут

Это не замена глубокой работе. Это короткое обслуживание памяти.

```text
1 вопрос
1 попытка ответа
1 проверка
1 исправление
1 отметка: повторить позже или закрыть
```

Хорошо подходит:

- перед началом учебного блока;
- после ошибки;
- между делами;
- утром для реактивации вчерашнего материала;
- перед Anki;
- после чтения короткой заметки.

Не подходит для первичного освоения сложной темы.

## 8. Вариации spaced repetition

### 8.1. Ручное интервальное повторение

Самая простая версия:

```text
в заметке после темы поставить даты повторения;
в каждый день открыть список;
сначала ответить без источника;
потом свериться;
перенести следующий повтор.
```

Подходит, если карточек мало или вы не хотите заводить отдельную систему.

### 8.2. Бумажные карточки

Подходит, если важно физически работать с материалом.

Правила:

- делать карточки самостоятельно;
- писать коротко;
- одна карточка - одна проверка;
- на лицевой стороне вопрос;
- на обратной стороне ответ;
- группировать по темам, но повторять вперемешку;
- сложные карточки возвращать чаще;
- легкие откладывать дальше.

### 8.3. Система Лейтнера

Бумажная или цифровая система коробок.

Пример:

```text
Коробка 1: каждый день
Коробка 2: через 2-3 дня
Коробка 3: раз в неделю
Коробка 4: раз в месяц
Коробка 5: редко
```

Ответил правильно - карточка движется дальше.  
Ошибся - возвращается ближе.

Плюс: просто и наглядно.  
Минус: менее гибко, чем современные алгоритмы.

### 8.4. Anki / FSRS

Anki - инструмент для карточек интервального повторения.  
FSRS - алгоритм планирования интервалов.

Практический смысл:

```text
приложение считает расписание;
пользователь отвечает;
качество карточки и честность оценки решают больше, чем бренд алгоритма.
```

Использовать Anki, если:

- карточек много;
- нужно повторять месяцами;
- есть языки, термины, команды, факты;
- нужно не забывать старое;
- удобно делать ежедневный короткий обзор.

Не использовать Anki как замену:

- решению задач;
- письму;
- проектам;
- чтению первоисточников;
- разбору ошибок;
- живой обратной связи.

### 8.5. Successive relearning

Смысл: не просто один раз повторить, а через интервалы снова доводить материал до критерия.

Пример критерия:

- 90-100% карточек правильны;
- 3 правильных ответа подряд;
- тема объяснена за 3 минуты без конспекта;
- задача решена без подсказки;
- ошибка исправлена и объяснена.

Протокол:

```text
день 1: выучить и воспроизвести до критерия
день 3: снова довести до критерия
день 7: снова проверить
день 14: применить
день 30: контроль удержания
```

Это полезно для материала, который действительно нужно держать долго.

### 8.6. Spaced retrieval

Это интервальное повторение через активное извлечение.

Не так:

```text
открыл конспект -> перечитал -> закрыл
```

А так:

```text
закрыл конспект -> вспомнил -> записал -> открыл -> проверил -> исправил
```

### 8.7. Interleaved spaced practice

Смысл: повторять с интервалами и перемешивать похожие темы.

Не так:

```text
10 карточек только про spacing
10 карточек только про retrieval
10 карточек только про interleaving
```

Лучше:

```text
spacing -> retrieval -> spacing -> interleaving -> retrieval -> cognitive load -> spacing
```

Особенно полезно, когда темы похожи и их можно спутать.

Примеры:

- `retrieval practice` / `spaced repetition` / `interleaving`;
- `errors.Is` / `errors.As` / прямое сравнение ошибок;
- `mock` / `fake` / `stub`;
- простое повторение / зубрежка / активное вспоминание;
- факт / гипотеза / преувеличение;
- правило / исключение / пример.

Ограничение: сначала нужна базовая ясность. Новичку слишком раннее перемешивание может мешать.

### 8.8. Variable retrieval

Одна идея проверяется разными способами.

Для темы "spaced repetition":

- дай определение;
- объясни простыми словами;
- приведи пример;
- назови ограничение;
- сравни с distributed practice;
- объясни, почему перечитывание слабее;
- составь расписание;
- найди плохую карточку;
- предложи улучшение.

Это помогает не застрять на одной формулировке.

### 8.9. Контекстная вариативность

Иногда полезно менять условия повторения:

- другое место;
- другая поза;
- другая тема оформления;
- другой формат вопроса;
- другой порядок карточек;
- другой тип задачи.

Смысл не в ритуале, а в снижении зависимости от одного контекста.

Ограничение: на раннем этапе это может повысить нагрузку. Если тема новая и хрупкая, сначала стабилизируйте базу.

### 8.10. Повторение через ошибки

Одна из самых практичных форм.

После ошибки создать не "карточку стыда", а карточку различения.

Плохая карточка:

```text
Я ошибся в Dockerfile.
```

Хорошая:

```text
Q: Какой признак показывает, что ошибка связана с рабочей директорией в Dockerfile?
A: Команда запускается из другого `WORKDIR`, относительный путь указывает не туда; проверить `WORKDIR`, `COPY`, текущую директорию и путь запуска.
```

Еще лучше:

```text
Q: Чем отличается ошибка "файл не найден при COPY" от ошибки "команда не нашла файл при RUN"?
```

## 9. Как делать карточки

### 9.1. Основные правила

Одна карточка должна проверять одно действие памяти.

Хорошая карточка:

- короткая;
- конкретная;
- написана вашим языком;
- содержит один вопрос;
- имеет проверяемый ответ;
- помогает будущей задаче;
- при необходимости содержит пример;
- при необходимости содержит ограничение;
- не требует перечитать длинный абзац.

Плохая карточка:

- "Расскажи все про тему";
- копирует абзац из источника;
- содержит 5 фактов сразу;
- проверяет узнавание, а не вспоминание;
- не имеет контекста;
- написана так, что через месяц непонятно, о чем вопрос;
- слишком легкая и бесполезная;
- слишком широкая и каждый раз проваливается.

### 9.2. Типы карточек

#### Термин

```text
Q: Что такое spaced repetition?
A: Возвращение к материалу через интервалы, обычно с увеличением пауз между повторениями.
```

#### Отличие

```text
Q: Чем spaced repetition отличается от retrieval practice?
A: Spaced repetition задает интервалы повторения; retrieval practice задает активную попытку вспомнить без подсказки.
```

#### Причина

```text
Q: Почему перечитывание может давать ложное чувство знания?
A: Текст узнается как знакомый, но это не доказывает, что знание можно извлечь без подсказки.
```

#### Пример

```text
Q: Приведи пример spaced repetition для статьи.
A: Прочитать, закрыть источник и вспомнить 3 идеи; повторить через 1 день, 3 дня, неделю и месяц.
```

#### Антипример

```text
Q: Что НЕ является хорошим spaced repetition?
A: Пять раз перечитать один и тот же текст подряд в один вечер и больше к нему не возвращаться.
```

#### Ошибка

```text
Q: Что делать, если карточка постоянно проваливается?
A: Проверить, не слишком ли она широкая; разделить, добавить пример, уточнить вопрос или вернуться к пониманию.
```

#### Применение

```text
Q: Как применить spaced repetition к изучению Go?
A: Делать карточки по понятиям и ошибкам, но регулярно добавлять задачи: написать пример, найти ошибку, объяснить различие.
```

#### Сравнение похожих понятий

```text
Q: Spacing, interleaving и retrieval - в чем разница?
A: Spacing - когда повторять; retrieval - вспоминать без подсказки; interleaving - перемешивать похожие темы или задачи.
```

#### Cloze / пропуск

```text
Q: Интервальное повторение сильнее всего работает вместе с [...], а не с пассивным перечитыванием.
A: активным вспоминанием / retrieval practice
```

Cloze полезен для терминов, формул, фраз, языков. Но если cloze превращается в угадывание слова по форме предложения, добавьте обычный вопрос.

#### Двусторонняя карточка

Для языков и терминов:

```text
Q: spaced repetition -> русский термин
A: интервальное повторение

Q: интервальное повторение -> английский термин
A: spaced repetition
```

Использовать осторожно: не все знания нужно гонять в обе стороны.

#### Карточка процедуры

```text
Q: Какой порядок активного повторения после лекции?
A: Закрыть конспект -> вспомнить главное -> записать -> свериться -> исправить -> назначить следующий повтор.
```

#### Карточка "когда не применять"

```text
Q: Когда карточки начинают мешать обучению?
A: Когда заменяют практику, плодят лишние детали, проверяют узнавание и не ведут к задачам, ошибкам или применению.
```

### 9.3. Формула хорошей карточки

```text
контекст + конкретный вопрос + короткий ответ + пример или ограничение
```

Пример:

```text
Контекст: учу эффективное обучение.
Q: Почему интервальное повторение лучше зубрежки для долговременной памяти?
A: Повторения распределены во времени; после частичного забывания попытка вспомнить требует усилия и укрепляет доступ к знанию.
Ограничение: если материал не понят, карточки не заменяют объяснение и практику.
```

## 10. Как пользоваться Anki / FSRS без самообмана

### 10.1. Минимальная настройка

Начать просто:

- 1-3 колоды максимум;
- теги вместо множества мелких колод;
- ежедневный короткий обзор;
- ограничение новых карточек;
- регулярная чистка;
- карточки только на нужное знание.

Плохой старт:

```text
20 колод, 300 новых карточек, чужие наборы, ноль практики.
```

Хороший старт:

```text
1 колода "Learning / Work";
5-15 новых карточек в день;
10-20 минут повторения;
карточки только по важным вопросам и ошибкам;
weekly review раз в неделю.
```

### 10.2. Как оценивать ответ

Оценивайте не настроение, а реальное извлечение.

Примерная логика:

- не вспомнил - снова;
- вспомнил с большой натяжкой - трудно;
- вспомнил нормально - хорошо;
- ответил быстро и точно - легко.

Не нажимайте "легко", если вы просто узнали ответ после взгляда на карточку.

### 10.3. Что делать с провальными карточками

Если карточка постоянно возвращается и раздражает, причина часто не в памяти.

Проверьте:

- вопрос слишком широкий;
- ответ слишком длинный;
- нет примера;
- непонятен контекст;
- материал не понят;
- карточка проверяет чужую формулировку;
- знание не нужно;
- нужно упражнение, а не карточка.

Действия:

- разделить карточку;
- переписать вопрос;
- добавить пример;
- добавить антипример;
- сделать карточку на отличие;
- вернуться к источнику;
- временно приостановить;
- удалить.

### 10.4. Колоды и теги

Для большинства случаев лучше не дробить колоды слишком мелко.

Плохо:

```text
Go interfaces
Go slices
Go maps
Go errors
Go tests
```

Если повторять их отдельно, получится блочная практика.

Лучше:

```text
Колода: Go core
Теги: interface, slice, map, errors, tests, mock, docker
```

Так легче получить interleaving: похожие темы перемешиваются, и вы учитесь выбирать нужный подход.

### 10.5. Новые карточки

Главное правило:

```text
новые карточки не должны ломать будущие повторения
```

Если каждый день добавлять слишком много, через неделю система станет долгом.

Старт:

- 5 новых карточек в день, если вы только начинаете;
- 10-15, если система уже стабильна;
- больше - только при понятном дедлайне и готовности к нагрузке.

### 10.6. Карточки после задачи, а не вместо задачи

Для практических тем полезный порядок:

```text
прочитал -> попробовал задачу -> ошибся -> понял причину -> сделал 1-3 карточки
```

Менее полезный порядок:

```text
прочитал -> сделал 40 карточек -> не применил
```

Карточка должна фиксировать то, что нужно будет вспомнить в действии.

## 11. "Фишки", которые реально полезны

### 11.1. Отвечать до переворота карточки

Нельзя считать повторением то, где вы сразу смотрите ответ.

Мини-правило:

```text
увидел вопрос -> сделал паузу -> ответил -> открыл ответ
```

### 11.2. Проговаривать вслух

Если тема сложная, ответ вслух быстрее выявляет пустоты.

Хорошо для:

- экзамена;
- собеседования;
- объяснений;
- языков;
- сложных понятий.

### 11.3. Делать карточки на различение

Самые ценные карточки часто не "что это", а "чем это отличается".

Примеры:

- spaced repetition vs distributed practice;
- retrieval practice vs rereading;
- interleaving vs blocked practice;
- understanding vs memorization;
- active recall vs recognition.

### 11.4. Добавлять антипример

Антипример защищает от механического знания.

```text
Q: Что НЕ является active recall?
A: Просто перечитать выделенные места и почувствовать знакомость текста.
```

### 11.5. Делать карточки из ошибок

После ошибки создайте карточку на признак ошибки, а не на факт "я ошибся".

Формула:

```text
Q: Как понять, что это ошибка типа X?
A: Признаки, проверка, отличие от похожей ошибки.
```

### 11.6. Использовать "пример + не пример"

Для понятий:

```text
Q: Пример и не пример spaced repetition.
A: Пример - вернуться к вопросу через день, 3 дня, неделю. Не пример - перечитать тему 5 раз подряд сегодня.
```

### 11.7. Сначала простая база, потом вариативность

Не начинайте с хаоса.

Порядок:

```text
понять базу -> сделать несколько прямых вопросов -> добавить различия -> добавить задачи -> перемешать
```

### 11.8. Менять форму вопроса

Одна и та же идея:

- что это;
- зачем это;
- когда применять;
- когда не применять;
- пример;
- ошибка;
- отличие;
- мини-задача.

Это снижает риск запомнить только одну фразу.

### 11.9. Использовать утреннюю реактивацию

3-5 минут утром:

```text
что я вчера учил?
какие 3 идеи помню?
какая была ошибка?
что повторить сегодня?
```

Это не обязательный ритуал, но хороший легкий вход в учебный день.

### 11.10. Короткое повторение перед сном

Полезно только если не мешает сну.

Формат:

```text
3-5 легких вопросов;
без новой сложной темы;
без тревожной гонки;
после - закрыть обучение.
```

Не превращать в ночную зубрежку.

### 11.11. Правило "3-7 вопросов после блока"

После учебного блока:

- 3 вопроса минимум;
- 7 вопросов максимум для обычной темы;
- если хочется 30 карточек, сначала спросить: "что из этого реально нужно помнить?"

### 11.12. Weekly purge

Раз в неделю удалять и переписывать.

Вопросы:

- какие карточки бесполезны;
- какие слишком широкие;
- какие постоянно проваливаются;
- какие заменить задачей;
- какие темы пора перемешать;
- какие старые карточки уже стали автоматическими.

### 11.13. Карточка должна возвращать к задаче

Если карточка про программирование, право, медицину, аналитику или обучение, периодически добавляйте:

```text
Q: Как это применить в реальной задаче?
```

Иначе можно хорошо помнить слова и плохо действовать.

### 11.14. Менять контекст дозированно

Иногда повторяйте:

- не за тем же столом;
- не в том же порядке;
- не в той же визуальной форме;
- не только в Anki, но и в задаче.

Но если материал новый, сначала закрепите основу.

### 11.15. Не создавать карточки в состоянии "сейчас все важно"

Сразу после лекции кажется, что важно все. Через день виднее.

Практичный вариант:

```text
сразу: 3-5 черновых вопросов;
на следующий день: оставить только нужное;
через неделю: удалить слабые карточки.
```

### 11.16. Сначала свой ответ, потом ИИ

Если используете ИИ:

```text
сначала ответить самому;
потом попросить ИИ проверить, найти пробелы и предложить вопросы;
потом вручную отобрать карточки.
```

ИИ хорошо помогает формулировать варианты, но не должен подменять вашу попытку извлечения.

## 12. Сценарии применения

### 12.1. После видео

1. Не пересматривать сразу.
2. Закрыть видео.
3. Ответить:
   - о чем было видео;
   - какие 3 идеи главные;
   - что применить;
   - где есть ограничение;
   - какой вопрос остался.
4. Создать 3-5 карточек.
5. Повторить через день.
6. Через неделю применить одну идею.

### 12.2. После статьи

Шаблон:

```text
Тезис:
Аргумент:
Ограничение:
Пример:
Вопрос для recall:
Следующий повтор:
```

Карточки:

- главная мысль;
- термин;
- ограничение;
- практический вывод;
- чем это отличается от похожей идеи.

### 12.3. После лекции

Сразу после:

```text
5 минут: восстановить структуру лекции
5 минут: написать вопросы
5 минут: свериться с конспектом
```

На следующий день:

```text
ответить на вопросы без конспекта;
слабые места превратить в карточки;
одно понятие объяснить вслух.
```

### 12.4. Для иностранного языка

Карточки:

- слово -> значение;
- значение -> слово;
- слово -> пример предложения;
- слово -> отличие от похожего;
- ошибка -> правильная форма;
- collocation / устойчивое сочетание;
- произношение, если нужно.

Обязательно добавлять контекст. Голое слово часто хуже удерживается и хуже переносится в речь.

### 12.5. Для программирования

Подход:

```text
понятие -> пример -> ошибка -> отличие -> мини-задача
```

Карточки:

- что делает команда;
- когда применять;
- типичная ошибка;
- отличие от похожего инструмента;
- что покажет компилятор;
- какой минимальный пример;
- как проверить результат.

Пример:

```text
Q: Чем `errors.Is` отличается от `errors.As`?
A: `errors.Is` проверяет соответствие конкретной ошибке в цепочке; `errors.As` ищет ошибку конкретного типа и извлекает ее.
```

После карточки нужна задача:

```text
написать минимальный пример с wrapped error и проверить оба случая.
```

### 12.6. Для экзамена

Не повторять "главы". Повторять вопросы.

Схема:

```text
список билетов -> вопросы -> активный ответ -> проверка -> слабые места -> повтор позже -> смешанный тест
```

Фишка:

```text
в конце подготовки перемешать вопросы, чтобы не зависеть от порядка конспекта.
```

### 12.7. Для юридических / медицинских / профессиональных знаний

Полезные карточки:

- критерии;
- признаки;
- исключения;
- сроки;
- документы;
- дифференциальные различия;
- типовые ошибки;
- "какой факт меняет вывод";
- "какое доказательство нужно".

Важно: карточки не заменяют кейсы. Они держат базу, а мышление тренируется на задачах.

### 12.8. Для книг

Не делайте карточку на каждый абзац.

Выбирайте:

- главные идеи;
- новые понятия;
- сильные различия;
- практические правила;
- вопросы, которые хотите помнить;
- идеи, к которым нужно вернуться.

После главы:

```text
1 карточка на главную мысль;
1 карточка на ограничение;
1 карточка на применение;
1 заметка в Obsidian, если идея связана с проектом.
```

### 12.9. Для Obsidian

Obsidian лучше использовать как внешнюю память и контекст, а spaced repetition - как механизм возврата.

Шаблон заметки:

```text
## Суть

## Почему важно

## Пример

## Антипример

## Вопросы для повторения

- [ ] ...
- [ ] ...
- [ ] ...

## Следующий повтор

## Связанные заметки
```

Главное: заметка должна возвращать к действию, а не только хранить текст.

## 13. Чему не стоит уделять слишком много внимания

### 13.1. Идеальному расписанию

Расписание важно, но не надо искать магическую формулу.

Лучше несовершенная, но работающая система:

```text
день 1 -> день 3 -> день 7 -> день 14
```

чем вечный поиск идеального алгоритма.

### 13.2. Количеству карточек

Много карточек не равно много знания.

Лучший вопрос:

```text
что мне нужно будет вспомнить или различить в реальной ситуации?
```

### 13.3. Красоте карточек

Карточка должна работать, а не выглядеть как мини-постер.

Исключение: схемы, изображения и цвет помогают, если они реально кодируют смысл.

### 13.4. Чужим наборам карточек

Чужие карточки могут быть полезны как черновик, но слабее для понимания.

Причина: вы не прошли обработку материала сами.

Если используете чужой набор:

- перепишите важные карточки своим языком;
- удалите лишнее;
- добавьте примеры из своих задач;
- проверьте точность.

### 13.5. Ощущению легкости

Легкость при перечитывании часто обманывает.

Лучший индикатор:

```text
могу ли я ответить без подсказки через несколько дней?
могу ли я применить это в новой задаче?
```

## 14. Частые ошибки

### Ошибка 1. Повторять перечитыванием

Проблема: вы узнаете текст, но не тренируете извлечение.

Исправление:

```text
сначала закрыть источник и ответить;
потом открыть и проверить.
```

### Ошибка 2. Делать карточки на все подряд

Проблема: система разрастается и превращается в долг.

Исправление:

```text
карточка только на то, что понадобится вспомнить, различить или применить.
```

### Ошибка 3. Слишком широкие вопросы

Плохо:

```text
Расскажи про интервальное повторение.
```

Лучше:

```text
Почему интервальное повторение лучше массированной зубрежки для долговременной памяти?
```

### Ошибка 4. Нет feedback

Проблема: можно активно вспоминать неправильную модель.

Исправление:

```text
после ответа свериться с эталоном и исправить.
```

### Ошибка 5. Карточки заменяют практику

Проблема: вы помните определения, но не умеете действовать.

Исправление:

```text
после карточек делать задачу, пример, объяснение или кейс.
```

### Ошибка 6. Слишком много нового

Проблема: будущие повторы становятся неподъемными.

Исправление:

```text
ограничить новые карточки и чистить старые.
```

### Ошибка 7. Учить непонятое

Проблема: механическая зубрежка держится плохо и плохо переносится.

Исправление:

```text
сначала объяснение и пример, потом карточка.
```

### Ошибка 8. Не различать цели

Факт, понятие, навык и проект требуют разных повторений.

Исправление:

```text
факт -> карточка;
понятие -> объяснение + пример;
навык -> задача;
проект -> применение и feedback;
ошибка -> карточка различения.
```

## 15. Как проверить, что система работает

Не по ощущению "я много повторял".

Проверять по результатам:

- через неделю я могу ответить без источника;
- ошибок одного типа стало меньше;
- я быстрее возвращаюсь к теме после перерыва;
- я могу объяснить понятие простыми словами;
- я могу привести пример и антипример;
- я могу отличить похожие идеи;
- я могу решить задачу без подсказки;
- повторение занимает умеренное время;
- карточки не вытесняют практику.

Плохие признаки:

- ежедневный долг растет;
- карточки вызывают избегание;
- много карточек не связано с целями;
- одни и те же карточки постоянно проваливаются;
- вы помните ответы, но не можете применить;
- Anki заменил учебу;
- система стала важнее результата.

## 16. 7-дневный эксперимент

Цель: проверить spaced repetition без самообмана.

### День 1

Выбрать одну тему.

Сделать:

- 5-10 вопросов;
- 1 пример;
- 1 антипример;
- 1 вопрос на отличие;
- 1 карточку по возможной ошибке.

### День 2

Повторить без источника.

Отметить:

- что вспомнилось легко;
- что забыто;
- где ответ был неточным;
- какую карточку надо переписать.

### День 3

Не добавлять много нового.  
Сделать одну мини-задачу или объяснение.

### День 4

Повторить слабые вопросы.  
Добавить feedback.

### День 5

Смешать тему с похожей.

Пример:

```text
spaced repetition + retrieval practice + interleaving
```

Ответить, чем они отличаются.

### День 6

Применить в реальной задаче:

- объяснить тему;
- решить задачу;
- написать пример;
- составить план повторения;
- найти ошибку.

### День 7

Weekly review:

- что реально запомнилось;
- какие карточки удалить;
- какие переписать;
- что требует задачи, а не карточки;
- продолжать ли систему.

Метрики:

- процент ответов без источника;
- число повторных ошибок;
- время на повторение;
- качество применения;
- не стало ли больше учебной бюрократии.

## 17. Готовые шаблоны

### 17.1. Шаблон карточки

```text
Q:

A:

Пример:

Антипример:

Когда применять:

Когда не применять:

Источник:
```

Не все поля нужны всегда. Используйте их для сложных понятий.

### 17.2. Шаблон карточки ошибки

```text
Ошибка:

Как я понял неправильно:

Правильная модель:

Какой признак надо было заметить:

С чем можно спутать:

Следующая проверка:
```

### 17.3. Шаблон повторения темы

```text
Тема:
Дата изучения:
Следующие повторы:

1. Что я помню без источника?
2. Какие 3 главные идеи?
3. Какой пример?
4. Где можно ошибиться?
5. Как это применить?
6. Что проверить по источнику?
```

### 17.4. Шаблон weekly review

```text
Какие карточки помогли?
Какие карточки были мусором?
Какие вопросы слишком широкие?
Какие темы надо перемешать?
Какие ошибки повторились?
Какие карточки заменить задачами?
Что удалить?
```

## 18. Промты для ИИ

ИИ можно использовать как помощника, но не как замену active recall. Сначала попытка своими силами, потом ИИ.

### 18.1. Сделать вопросы по тексту

```text
Я изучаю тему для долговременного понимания.
Сделай 10 вопросов для active recall по тексту ниже.
Требования:
- не копируй формулировки источника;
- добавь вопросы на отличие, пример, антипример и ограничение;
- отдели простые факты от вопросов на применение;
- не делай карточки на неважные детали.

Текст:
...
```

### 18.2. Улучшить плохие карточки

```text
Ниже мои карточки для интервального повторения.
Найди плохие карточки: слишком широкие, неоднозначные, длинные, механические или без контекста.
Для каждой предложи улучшенную версию.
Сохрани смысл, но сделай вопрос проверяемым.

Карточки:
...
```

### 18.3. Сделать карточки по ошибке

```text
Я сделал ошибку при изучении темы.
Помоги превратить ее в 3 карточки:
1. признак ошибки;
2. отличие от похожей ситуации;
3. правильное действие.

Ошибка:
...

Правильное объяснение:
...
```

### 18.4. Сделать вариативное извлечение

```text
Для темы ниже сделай набор variable retrieval:
- определение;
- объяснение простыми словами;
- пример;
- антипример;
- вопрос на отличие;
- мини-задача;
- типичная ошибка;
- вопрос "когда не применять".

Тема:
...
```

### 18.5. Составить план повторения

```text
У меня есть тема, которую нужно помнить через [срок].
Составь план spaced repetition.
Учитывай:
- первые повторы должны быть короткими;
- каждое повторение начинается с active recall;
- нужны feedback и применение;
- не перегружай план.

Тема:
...
Срок:
...
```

## 19. Минимальная система на каждый день

Если не хочется строить большую систему, достаточно этого:

```text
1. После учебного блока сделать 3 вопроса.
2. На следующий день ответить без источника.
3. Ошибки исправить и превратить в 1-2 хорошие карточки.
4. Через неделю повторить вперемешку с похожими темами.
5. Раз в неделю удалить лишнее.
```

Это уже намного сильнее, чем:

```text
прочитать -> подчеркнуть -> забыть -> перед дедлайном перечитать все заново
```

## 20. Главные правила

1. Повторение начинается до открытия источника.
2. Интервалы нужны для долговременной памяти, а не для красоты расписания.
3. Хороший вопрос важнее красивой карточки.
4. Feedback обязателен, если есть риск закрепить ошибку.
5. Карточки должны вести к применению.
6. Точные интервалы подбираются по материалу и цели.
7. Не все нужно превращать в карточки.
8. Плохие карточки надо переписывать или удалять.
9. Для навыков нужны задачи.
10. Система должна уменьшать забывание, а не увеличивать учебную бюрократию.

Короткая финальная формула:

```text
Вспомни сам.
Проверь.
Исправь.
Вернись позже.
Примени иначе.
Удали лишнее.
```
