# Память и повторение - инструкция пользователя

Режим: `ytb`  
Тема: память, повторение, активное припоминание, вариативное извлечение, последовательное переучивание, утренняя реактивация, контроль оверлернинга, пауза с последующим припоминанием.  
Основные источники: `Learning`, `Memory`, `4sort`.

---

## 1. Главная идея

Память укрепляется не от того, что материал много раз проходит перед глазами, а от повторяющегося цикла:

```text
вспомнить без подсказки -> проверить -> исправить -> вернуться позже -> применить в другой форме
```

Повторение полезно, когда оно заставляет достать знание из памяти. Если повторение сводится к перечитыванию знакомого текста, оно легко создает иллюзию знания: "я это видел, значит, я это знаю".

Более точный критерий:

```text
Я знаю материал, если могу восстановить его без источника,
проверить точность,
объяснить простыми словами,
привести пример,
найти ошибку
и применить в задаче.
```

---

## 2. Пять технологий из этой инструкции

| Технология | Для чего нужна | Короткая формула |
|---|---|---|
| Variable retrieval / вариативное извлечение | Чтобы не застревать на одной формулировке и переносить знание в задачи | одна идея -> разные проверки |
| Successive relearning / последовательное переучивание | Чтобы доводить материал до устойчивого воспроизведения через несколько сессий | выучил -> критерий -> интервал -> снова критерий |
| Morning reactivation / утренняя реактивация | Чтобы мягко вернуть вчерашний материал в рабочее состояние | 3-5 минут утром -> вспомнить -> свериться |
| Overlearning control / контроль оверлернинга | Чтобы не тратить время на легкое и не перегружать систему | стало легко -> остановись -> следующий повтор позже |
| Pause + recall / пауза с последующим припоминанием | Чтобы снизить интерференцию и проверить, что осталось после отдыха | блок -> пауза без контента -> recall |

Эти технологии лучше использовать вместе, но не одновременно в максимальной версии. Их задача - сделать повторение умным, а не бесконечным.

## 3. Карта всей системы

Минимальная рабочая система:

```text
1. Изучил маленький блок.
2. Закрыл источник.
3. Вспомнил главное.
4. Проверил себя.
5. Сделал паузу без нового контента.
6. Через время вернулся к материалу.
7. Проверил его другой формой.
8. Если легко - отложил дальше.
9. Если трудно - исправил вопрос, пример или понимание.
```

Расширенная система:

```text
первичное понимание
-> active recall
-> feedback
-> pause + recall
-> spaced repetition
-> variable retrieval
-> successive relearning
-> application
-> weekly cleanup
```

## 4. Что важно понимать о памяти

### 4.1. Узнавание не равно вспоминание

Когда вы перечитываете конспект, текст кажется знакомым. Это приятно, но не доказывает, что вы сможете ответить без подсказки.

Разница:

| Действие | Что проверяет |
|---|---|
| Перечитать | узнавание |
| Подчеркнуть | внимание к фрагменту |
| Пересказать без источника | извлечение |
| Ответить на вопрос | доступ к знанию |
| Решить задачу | применение |
| Объяснить другому | понимание структуры |
| Найти ошибку | различение и перенос |

Главный тест:

```text
могу ли я достать это из головы, когда источника нет?
```

### 4.2. Забывание - не всегда провал

Частичное забывание делает повторение полезнее. Если материал чуть-чуть "ушел", попытка восстановить его требует усилия. Это усилие и укрепляет доступ к знанию.

Но есть граница:

- если все слишком легко, повторение почти ничего не добавляет;
- если все полностью забыто, повторение превращается в новое изучение;
- полезная зона - умеренная трудность.

### 4.3. Память не является идеальной записью

Память реконструирует материал. При каждом воспроизведении можно уточнить знание, но можно и исказить его.

Отсюда правило:

```text
recall без feedback опасен для точности.
```

После ответа нужна проверка:

- по источнику;
- по решению;
- по тесту;
- по документации;
- по преподавателю;
- по рабочему примеру;
- по собственному журналу ошибок.

### 4.4. Для навыков одной памяти мало

Можно помнить определение и не уметь пользоваться знанием.

Для навыков нужен следующий слой:

```text
вопрос -> ответ -> пример -> задача -> ошибка -> повторная задача
```

Например, если вы учите программирование, карточка может помочь вспомнить термин. Но навык появляется, когда вы пишете код, получаете ошибку, проверяете модель и повторяете похожую задачу.

## 5. Базовый протокол повторения

### Шаг 1. Завершить учебный блок вопросом

После материала не спрашивайте "я понял?". Это слишком расплывчато.

Спросите:

- что я могу воспроизвести без источника;
- какие 3 идеи главные;
- какой пример я могу привести;
- где я могу ошибиться;
- что надо проверить;
- что я смогу сделать с этим знанием.

### Шаг 2. Закрыть источник

Повторение начинается до открытия конспекта.

```text
сначала память -> потом источник
```

### Шаг 3. Восстановить

Форматы:

- устный пересказ;
- письменный ответ;
- чистый лист;
- схема по памяти;
- ответ на карточку;
- мини-тест;
- решение задачи;
- объяснение простыми словами.

### Шаг 4. Проверить

После попытки:

- что вспомнил точно;
- что исказил;
- что забыл;
- что понял поверхностно;
- где нужна карточка;
- где нужна задача;
- где нужно вернуться к объяснению.

### Шаг 5. Исправить

Не переписывайте весь источник. Исправляйте конкретную дыру:

- уточнить определение;
- добавить пример;
- сделать карточку на отличие;
- решить короткую задачу;
- записать ошибку;
- выделить условие применения.

### Шаг 6. Назначить следующий контакт

Стартовая сетка:

```text
сегодня -> завтра -> через 3 дня -> через неделю -> через 2 недели -> через месяц
```

Это не магическая формула. Это рабочая заготовка. Интервал меняется по трудности и цели.

## 6. Variable retrieval / вариативное извлечение

### 6.1. Что это

**Variable retrieval** - это проверка одной идеи разными способами.

Не просто:

```text
Что такое spaced repetition?
```

А набор:

```text
Дай определение.
Объясни простыми словами.
Приведи пример.
Приведи антипример.
Сравни с похожей идеей.
Реши мини-задачу.
Найди ошибку.
Назови ограничение.
```

Смысл: вы тренируете не одну заученную фразу, а гибкий доступ к знанию.

### 6.2. Когда использовать

Использовать, когда:

- знание нужно применять, а не только помнить;
- темы похожи и путаются;
- есть риск выучить формулировку без понимания;
- нужно готовиться к экзамену, собеседованию или практике;
- материал относится к программированию, праву, медицине, психологии, математике, языкам;
- важно переносить идею в новые ситуации.

Не обязательно использовать для каждой мелкой детали. Для простого факта иногда достаточно обычной карточки.

### 6.3. Основной протокол

Для каждой важной идеи создать 4-8 разных проверок:

1. Определение.
2. Объяснение простыми словами.
3. Пример.
4. Антипример.
5. Отличие от похожего.
6. Типичная ошибка.
7. Мини-задача.
8. Ограничение.

Пример для `retrieval practice`:

```text
Q1: Что такое retrieval practice?
Q2: Почему перечитывание слабее активного извлечения?
Q3: Приведи пример retrieval practice после видео.
Q4: Что может пойти не так, если нет feedback?
Q5: Чем retrieval practice отличается от spaced repetition?
Q6: Как применить retrieval practice к изучению Go?
Q7: Когда карточки не заменяют задачу?
Q8: Какой антипример active recall?
```

### 6.4. Варианты проверок

#### Определение

```text
Что это?
```

Пример:

```text
Что такое variable retrieval?
```

#### Смысл

```text
Зачем это нужно?
```

Пример:

```text
Зачем одну идею проверять разными формами?
```

#### Пример

```text
Где это выглядит правильно?
```

Пример:

```text
Приведи пример variable retrieval для английского слова.
```

#### Антипример

```text
Что выглядит похоже, но не является этим?
```

Пример:

```text
Почему 10 одинаковых карточек на одно определение - слабая версия повторения?
```

#### Отличие

```text
С чем это можно спутать?
```

Пример:

```text
Чем variable retrieval отличается от interleaving?
```

#### Ошибка

```text
Где я обычно ошибаюсь?
```

Пример:

```text
Какая ошибка возникает, если вспоминать без проверки?
```

#### Применение

```text
Что я сделаю с этим знанием?
```

Пример:

```text
Составь 5 разных вопросов по одной заметке Obsidian.
```

### 6.5. Фишки variable retrieval

- Делайте наборы "что это / зачем / пример / ошибка / отличие".
- Для сложных понятий добавляйте "когда не применять".
- Для похожих тем делайте карточки на различение.
- Для программирования добавляйте мини-код или прогноз ошибки.
- Для права добавляйте "какой факт меняет вывод".
- Для языков добавляйте контекст, пример предложения и обратное направление.
- Для книг добавляйте "какой вывод автора и где его ограничение".
- Не вводите слишком много вариативности до первичного понимания.

### 6.6. Ошибки variable retrieval

Ошибка: проверять одну идею десятью формами сразу, когда база еще не понята.

Исправление:

```text
сначала прямое определение и простой пример;
потом отличие, ошибка и применение.
```

Ошибка: путать вариативность с хаосом.

Исправление:

```text
вариативность должна менять форму проверки, а не распылять внимание на случайные темы.
```

Ошибка: делать вопросы без feedback.

Исправление:

```text
после ответа сверяться с источником, задачей или эталоном.
```

## 7. Successive relearning / последовательное переучивание

### 7.1. Что это

**Successive relearning** - это метод, где вы несколько раз возвращаетесь к материалу через интервалы и каждый раз доводите воспроизведение до заданного критерия.

Формула:

```text
выучить -> воспроизвести до критерия -> пауза -> снова воспроизвести до критерия -> повторить через интервал
```

Это не одно повторение. Это серия восстановлений.

### 7.2. Чем отличается от обычного повторения

Обычное повторение:

```text
перечитал -> вроде знакомо -> пошел дальше
```

Successive relearning:

```text
ответил без подсказки -> проверил -> исправил -> добился критерия -> вернулся через несколько дней -> снова добился критерия
```

### 7.3. Когда использовать

Использовать для материала, который нужен надолго:

- иностранные слова;
- формулы;
- команды;
- алгоритмы;
- экзаменационные вопросы;
- медицинские и юридические понятия;
- программные конструкции;
- базовые принципы;
- типовые ошибки;
- профессиональные чек-листы.

Не использовать на все подряд. Это сильная, но дорогая техника.

### 7.4. Критерии готовности

Критерий должен быть проверяемым:

- 90-100% карточек отвечены правильно;
- 3 правильных ответа подряд;
- объяснение темы за 3 минуты без конспекта;
- схема восстановлена по памяти;
- задача решена без подсказки;
- ошибка найдена и объяснена;
- билет рассказан с планом и примерами;
- команда использована в рабочем контексте.

Плохой критерий:

```text
кажется, я понял.
```

Хороший критерий:

```text
я объяснил тему без источника, допустил 1 неточность, исправил ее и через 3 дня повторил без ошибки.
```

### 7.5. Базовый протокол

```text
День 1:
изучить материал;
ответить на вопросы;
исправить ошибки;
довести до критерия.

День 3:
снова ответить без подсказки;
исправить;
довести до критерия.

День 7:
проверить удержание;
добавить задачу или пример.

День 14:
повторить в смешанном наборе.

День 30:
контрольное воспроизведение.
```

### 7.6. Фишки successive relearning

- Заранее задайте критерий, иначе будете повторять бесконечно.
- Не добивайтесь идеальности для неважного материала.
- На второй и третьей сессии начинайте с чистого recall, а не с чтения.
- Если материал проваливается снова и снова, проблема может быть в понимании или карточке.
- Для навыков добавляйте задачу после достижения критерия.
- Для экзамена делайте критерий ближе к формату экзамена.
- Для работы делайте критерий ближе к реальному применению.

### 7.7. Successive relearning и навыки

Для навыков критерий не должен быть только текстовым.

Пример для программирования:

```text
Критерий 1: объяснить понятие.
Критерий 2: написать минимальный пример.
Критерий 3: найти ошибку.
Критерий 4: применить в новой маленькой задаче.
```

Пример для юридического материала:

```text
Критерий 1: назвать норму.
Критерий 2: выделить юридически значимые факты.
Критерий 3: отличить факт от вывода.
Критерий 4: применить к кейсу.
```

## 8. Morning reactivation / утренняя реактивация

### 8.1. Что это

**Morning reactivation** - короткое восстановление вчерашнего материала утром.

Формула:

```text
3-5 минут утром -> вспомнить вчерашнее -> свериться -> выбрать следующий шаг
```

Это не полноценная учебная сессия. Это запуск памяти.

### 8.2. Зачем это нужно

Утром можно быстро проверить:

- что осталось после сна;
- что нужно повторить;
- какая ошибка всплывает снова;
- с чего начать день;
- где знание уже доступно, а где требует поддержки.

Это помогает не начинать учебу с нуля каждый раз.

### 8.3. Мини-протокол на 3-5 минут

```text
1 минута:
Что я вчера изучал?

1 минута:
Какие 3 идеи помню без источника?

1 минута:
Какая была ошибка или трудность?

1 минута:
Открыть источник и сверить только главное.

1 минута:
Записать первый шаг на сегодня.
```

Если времени мало:

```text
Что вчера было главным?
Что я могу вспомнить без подсказки?
Что повторить сегодня?
```

### 8.4. Что не делать утром

Не превращать утреннюю реактивацию в:

- часовой повтор;
- чтение всех конспектов;
- новую лекцию;
- просмотр видео;
- чистку всей Anki-системы;
- тревожную проверку "я все забыл".

Цель не в том, чтобы утром выучить заново. Цель - вернуть след памяти и определить следующий шаг.

### 8.5. Форматы

#### Формат "3 идеи"

```text
Вчера я изучал:

Помню:
1.
2.
3.

Не помню:

Сегодня первый шаг:
```

#### Формат "ошибка"

```text
Вчерашняя ошибка:

Почему я ошибся:

Как правильно:

Какую маленькую проверку сделать сегодня:
```

#### Формат "карточки"

```text
Открыть 3-5 карточек по вчерашней теме.
Ответить без подсказки.
Не добавлять новые карточки во время утренней реактивации.
```

#### Формат "чистый лист"

```text
На пустом листе восстановить схему вчерашней темы.
Потом открыть источник и отметить пробелы.
```

### 8.6. Фишки morning reactivation

- Делайте до нового контента.
- Не открывайте сразу YouTube, новости или чат.
- Начинайте с памяти, а не с конспекта.
- Проверяйте только главное.
- Если вспоминается плохо, не ругайте себя: это данные для повторения.
- Если вспоминается легко, не повторяйте долго; назначьте следующий более поздний контакт.
- Используйте утро для выбора первого действия, а не для планирования всего дня.

### 8.7. Когда особенно полезно

- после сложной лекции;
- после вечернего учебного блока;
- при изучении языка;
- перед продолжением проекта;
- перед практикой программирования;
- при подготовке к экзамену;
- после вчерашней ошибки;
- если вы часто забываете, на чем остановились.

## 9. Overlearning control / контроль оверлернинга

### 9.1. Что такое оверлернинг

**Overlearning** - продолжение повторения после того, как материал уже воспроизводится правильно.

Иногда это полезно:

- для автоматизма;
- для выступления;
- для экзаменационного билета;
- для аварийных процедур;
- для базовых команд;
- для языка;
- для навыков, где цена ошибки высока.

Но часто это просто трата времени:

```text
легко вспомнил -> повторил еще 10 раз -> устал -> не осталось сил на слабые места
```

### 9.2. Главный принцип контроля

```text
Если стало легко, следующий полезный повтор часто должен быть позже.
```

Не надо бесконечно гонять то, что уже доступно. Лучше увеличить интервал, перейти к применению или заняться слабым местом.

### 9.3. Как понять, что пора остановиться

Остановиться, если:

- ответ быстрый и точный;
- вы дали пример;
- вы объяснили ограничение;
- вы не путаете с похожей темой;
- задача решена без подсказки;
- карточка несколько раз подряд легкая;
- повторение стало механическим;
- есть более слабый материал.

Продолжить, если:

- ответ узнается, но не воспроизводится;
- есть ошибка в ключевой детали;
- вы путаете похожие понятия;
- нет примера;
- не можете применить;
- материал нужен завтра в точной форме;
- цена ошибки высокая.

### 9.4. Stop rules

#### Правило одного легкого ответа

Если карточка легкая, не повторяйте ее снова сегодня. Отложите.

#### Правило двух форм

Если ответ легкий, проверьте другой формой:

```text
определение -> пример
пример -> отличие
отличие -> задача
```

Если и это легко - остановитесь.

#### Правило слабого места

После легкой карточки спросите:

```text
где сейчас слабее?
```

И переключитесь туда.

#### Правило применения

Если знание легко вспоминается, следующий шаг - не еще одно повторение, а применение.

```text
вспомнил -> применил -> проверил
```

### 9.5. Оверлернинг и Anki

В Anki риск оверлернинга появляется, когда вы:

- вручную гоняете легкие карточки;
- создаете слишком много дублей;
- нажимаете "снова" из тревоги, хотя ответ был нормальный;
- держите карточки, которые уже не нужны;
- повторяете колоду ради ощущения контроля.

Практика:

- честно нажимайте "легко", если ответ действительно легкий;
- удаляйте и приостанавливайте лишнее;
- не добавляйте дубли на один факт;
- используйте теги для weak spots;
- weekly review обязателен.

### 9.6. Когда оверлернинг оправдан

Оверлернинг полезен, если нужно:

- выступить без запинки;
- быстро вызвать команду или процедуру;
- автоматизировать базовую операцию;
- помнить критический чек-лист;
- сдать экзамен с точными формулировками;
- убрать задержку в базовом навыке.

Даже тогда лучше ограничить объем:

```text
1-2 дополнительных правильных воспроизведения,
а не бесконечный прогон.
```

### 9.7. Фишки overlearning control

- Легкое не повторять много раз подряд.
- После легкости увеличивать интервал.
- Вместо лишнего повторения делать вопрос на применение.
- При тревоге не добавлять повторы автоматически; сначала проверить реальный результат.
- Разделять "я не уверен" и "я реально не помню".
- Для важного материала использовать критерий, а не ощущение.
- Для неважного материала разрешать себе забыть.

## 10. Pause + recall / пауза с последующим припоминанием

### 10.1. Что это

**Pause + recall** - это связка:

```text
учебный блок -> короткая пауза без нового контента -> припоминание
```

Пауза снижает информационный шум. Recall проверяет, что осталось доступным после небольшой паузы.

### 10.2. Связь с wakeful rest

В локальных источниках wakeful rest описан как спокойный бодрствующий отдых после обучения: без телефона, новых задач, видео, переписок и новой учебной информации.

Аккуратная формулировка:

```text
Короткая пауза после сложного материала может снизить интерференцию и помочь памяти.
Но она не заменяет повторение, feedback и практику.
```

Не стоит обещать себе точные проценты улучшения. Эффекты зависят от материала, условий и проверки.

### 10.3. Базовый протокол

```text
1. Закончить учебный блок.
2. Записать одну фразу: "Что сейчас было главным?"
3. 5-10 минут без нового контента.
4. Не открывать телефон, видео, новости, чат, новую лекцию.
5. После паузы восстановить 3 идеи по памяти.
6. Свериться.
7. Записать следующий шаг.
```

### 10.4. Что можно делать во время паузы

Можно:

- посидеть спокойно;
- пройтись без аудио;
- выпить воды;
- посмотреть в окно;
- сделать легкую разминку;
- убрать стол;
- принять душ;
- просто ничего не делать.

Нежелательно:

- листать ленту;
- смотреть короткие видео;
- читать новости;
- открывать новую тему;
- решать сложную задачу;
- вступать в эмоциональную переписку;
- слушать насыщенный подкаст.

### 10.5. Когда использовать

Использовать после:

- сложной лекции;
- новой темы;
- интенсивного чтения;
- debug-блока;
- сильной ошибки;
- экзаменационной подготовки;
- изучения большого концепта;
- сессии, где было много новых связей.

Не нужно делать отдельный ритуал после каждой мелочи.

### 10.6. Варианты pause + recall

#### 5 минут

```text
2 минуты пауза
3 минуты recall
```

Подходит для короткого блока.

#### 10 минут

```text
5 минут без контента
5 минут восстановить и проверить
```

Подходит для обычной учебной сессии.

#### 20 минут

```text
10-15 минут прогулка без аудио
5 минут записать главное
```

Подходит после тяжелого материала.

### 10.7. Фишки pause + recall

- Сначала коротко закройте блок, потом отдыхайте.
- Не превращайте паузу в медитационный экзамен.
- Если тишина раздражает, сделайте спокойную ходьбу.
- После паузы вспоминайте, а не перечитывайте.
- Если после паузы ничего не вспоминается, это не провал; вернитесь к источнику и сделайте 2-3 вопроса.
- Не используйте паузу как прокрастинацию перед практикой.

## 11. Как пять технологий работают вместе

### 11.1. Стандартная сессия 45 минут

```text
0-25 минут:
изучить или решить задачу.

25-30 минут:
закрыть источник и восстановить главное.

30-38 минут:
пауза без нового контента.

38-43 минуты:
recall после паузы: 3 идеи, 1 ошибка, 1 следующий шаг.

43-45 минут:
назначить повтор или сделать 1 карточку.
```

### 11.2. На следующий день

```text
утром 3-5 минут:
вспомнить вчерашнее без источника.

днем:
сделать одну вариативную проверку.

вечером:
не гонять легкое; повторить только слабое или назначенное.
```

### 11.3. Через несколько дней

```text
вернуться к теме;
проверить по другому формату;
довести важное до критерия;
легкое отложить дальше;
сложное переписать или применить.
```

### 11.4. Недельный обзор

```text
Что запомнилось?
Что воспроизводится только при подсказке?
Какие вопросы слишком легкие?
Где нужен variable retrieval?
Что довести через successive relearning?
Что удалить?
Где нужна практика, а не повторение?
```

## 12. Готовые сценарии

### 12.1. После видео

1. Закрыть видео.
2. 2 минуты ничего не смотреть.
3. Записать 3 главные идеи.
4. Свериться с заметками.
5. Сделать 3 вопроса:
   - определение;
   - пример;
   - ошибка или ограничение.
6. Утром восстановить суть.
7. Через неделю проверить другой формой.

### 12.2. После статьи

```text
Суть статьи:
Главный аргумент:
Ограничение:
Пример:
Антипример:
Вопрос на применение:
Следующий повтор:
```

### 12.3. После лекции

Сразу:

```text
закрыть конспект;
восстановить структуру;
записать 5 вопросов;
свериться;
5-10 минут без нового контента.
```

Утром:

```text
3 идеи;
1 забытая деталь;
1 следующий шаг.
```

Через 3-7 дней:

```text
смешанный мини-тест;
вопросы на отличие;
задача или объяснение.
```

### 12.4. Для экзамена

Не повторять главы. Повторять ответы.

Схема:

```text
билет -> план ответа -> ответ вслух -> сверка -> слабые места -> повтор через интервал
```

Добавить variable retrieval:

- объяснить коротко;
- ответить подробно;
- привести пример;
- назвать ограничение;
- сравнить с похожим билетом;
- найти типичную ошибку.

Контроль оверлернинга:

```text
если билет уже легко рассказывается 2 раза подряд,
перенести его дальше и заняться слабым.
```

### 12.5. Для языка

Проверки:

- слово -> перевод;
- перевод -> слово;
- слово -> пример;
- слово -> отличие от похожего;
- фраза -> ситуация применения;
- ошибка -> правильная форма;
- аудио/произношение -> повторение вслух.

Фишка:

```text
утром восстановить 5 вчерашних слов в предложениях, а не просто переводом.
```

### 12.6. Для программирования

Проверки:

- что это;
- зачем нужно;
- минимальный пример;
- ошибка компиляции;
- отличие от похожего;
- задача без подсказки;
- изменение условия;
- объяснение новичку.

Пример:

```text
Тема: context.WithTimeout

Определение:
Что делает?

Пример:
Написать минимальный код.

Ошибка:
Что будет, если не вызвать cancel?

Отличие:
Чем timeout отличается от manual cancel?

Применение:
Добавить timeout в маленький HTTP-запрос.
```

### 12.7. Для Obsidian

Шаблон заметки:

```text
## Суть

## Что вспомнить завтра

- [ ] 
- [ ] 
- [ ] 

## Variable retrieval

Определение:
Пример:
Антипример:
Ошибка:
Отличие:
Применение:

## Pause + recall

После паузы вспомнил:

## Следующий повтор

## Что удалить / не повторять
```

### 12.8. Для Anki

Anki использовать как часть системы, а не как всю систему.

Карточки:

- короткие;
- конкретные;
- на одно различие;
- с примером при необходимости;
- с вопросами на ошибки;
- без больших абзацев.

Не делать:

- карточки на все подряд;
- дубли легких фактов;
- повтор легкого материала из тревоги;
- карточки без применения для практических тем.

## 13. Фишки памяти и повторения

### 13.1. "Сначала память, потом источник"

Перед любым повторением:

```text
что я помню без подсказки?
```

Только потом открыть конспект.

### 13.2. "Одна идея - пять входов"

Для важной идеи:

```text
определение;
пример;
антипример;
ошибка;
применение.
```

### 13.3. "Легкое отправлять в будущее"

Если ответ легкий, не тратьте на него текущую энергию.

```text
легко -> позже
трудно -> исправить
важно -> применить
```

### 13.4. "После паузы не перечитывать"

После отдыха сначала вспомнить:

```text
что осталось?
```

Потом свериться.

### 13.5. "Утро без нового контента"

Первые 3-5 минут учебного дня:

```text
вчерашний материал -> recall -> следующий шаг
```

### 13.6. "Ошибка как карточка различения"

Не писать:

```text
Я ошибся.
```

Писать:

```text
Какой признак показывает, что здесь нужен другой подход?
```

### 13.7. "Критерий перед повторением"

Перед serious relearning:

```text
что будет считаться достаточным?
```

Без критерия легко уйти в бесконечное повторение.

### 13.8. "Смена формы вместо увеличения количества"

Если хочется повторить еще раз, лучше сменить форму:

```text
карточка -> пример;
пример -> задача;
задача -> объяснение;
объяснение -> ошибка.
```

### 13.9. "Не путать комфорт и результат"

Комфортное перечитывание часто кажется продуктивным. Активное извлечение часто ощущается труднее, но лучше показывает реальное знание.

### 13.10. "Не всё должно жить в памяти"

Часть информации должна жить во внешней системе:

- Obsidian;
- чек-листы;
- документация;
- заметки;
- ссылки;
- проекты;
- шаблоны.

В память стоит помещать то, что нужно быстро извлекать, различать и применять.

## 14. Частые ошибки

### Ошибка 1. Повторять слишком рано и слишком часто

Если материал еще свежий в рабочей памяти, повторение может быть слишком легким.

Лучше:

```text
дождаться умеренной трудности;
увеличивать интервалы;
не гонять легкое много раз подряд.
```

### Ошибка 2. Сразу открывать конспект

Так вы тренируете узнавание.

Лучше:

```text
закрытый источник -> ответ -> открытый источник -> correction.
```

### Ошибка 3. Делать variable retrieval слишком рано

Если база не понята, вариативность перегружает.

Лучше:

```text
сначала прямой вопрос и простой пример;
потом отличие, ошибка и перенос.
```

### Ошибка 4. Successive relearning для всего

Это дорогой метод. Используйте его для важного материала.

### Ошибка 5. Утренняя реактивация превращается в полноценную учебу

Если утром вы час "реактивируете", это уже не micro-review.

Ограничение:

```text
3-5 минут, максимум 10.
```

### Ошибка 6. Пауза превращается в ленту

Телефон после обучения - не wakeful rest. Это новый поток стимулов.

### Ошибка 7. Feedback отсутствует

Можно запомнить ошибку уверенно.

### Ошибка 8. Карточки заменяют задачи

Для навыка нужны действия.

### Ошибка 9. Стремление помнить всё

Запоминать абсолютно всё не нужно и невозможно. Выбирайте опорное.

### Ошибка 10. Оценивать систему по количеству повторов

Лучше оценивать по самостоятельным ответам, задачам и снижению повторных ошибок.

## 15. Проверка качества повторения

Хорошее повторение:

- начинается с попытки вспомнить;
- имеет конкретный вопрос;
- заканчивается feedback;
- приводит к исправлению;
- повторяется через интервал;
- иногда меняет форму проверки;
- не тратит время на уже легкое;
- связано с задачей или будущим применением.

Плохое повторение:

- начинается с перечитывания;
- создает ощущение знакомости;
- не имеет проверки;
- повторяет легкое из тревоги;
- копит карточки без цели;
- заменяет практику;
- не обновляет расписание;
- не удаляет лишнее.

## 16. 7-дневный эксперимент

Цель: проверить систему "память и повторение" без самообмана.

### День 1

Выбрать одну тему.

Сделать:

- 5 вопросов;
- 1 пример;
- 1 антипример;
- 1 вопрос на ошибку;
- 1 критерий успеха.

После блока:

```text
5 минут без контента;
после паузы вспомнить 3 идеи.
```

### День 2

Утром:

```text
3-5 минут восстановить вчерашнее.
```

Днем:

```text
повторить слабые вопросы;
легкие отложить.
```

### День 3

Variable retrieval:

- объяснить простыми словами;
- привести новый пример;
- найти ошибку;
- сравнить с похожей идеей.

### День 4

Successive relearning:

```text
довести важный блок до критерия.
```

Например:

- 90% правильных ответов;
- 3 правильных ответа подряд;
- задача без подсказки.

### День 5

Pause + recall:

```text
после сложного блока 10 минут без контента;
затем чистый лист по памяти.
```

### День 6

Применение:

- решить задачу;
- объяснить человеку;
- написать мини-инструкцию;
- применить к своему проекту;
- пройти смешанный тест.

### День 7

Review:

- что реально помню;
- где помогла утренняя реактивация;
- где повторял лишнее;
- какие вопросы надо переписать;
- что удалить;
- что оставить как недельный протокол.

Метрики:

- количество ответов без источника;
- количество повторных ошибок;
- время на повторение;
- качество применения;
- число удаленных лишних карточек;
- субъективная перегрузка 1-5.

```text
8. Ограничение:
```

### 17.2. Шаблон successive relearning

```text

- результат:
- ошибки:
- исправление:

Сессия 2:
- результат:
- ошибки:
- исправление:

Сессия 3:
- применение:
```

### 17.3. Шаблон morning reactivation

### 17.5. Шаблон pause + recall

## 18. Промты для ИИ

ИИ использовать после собственной попытки, а не до нее.

### 18.1. Сделать variable retrieval

```text
Я изучаю тему и хочу проверить ее через variable retrieval.
Сделай набор вопросов:
- определение;
- простое объяснение;
- пример;
- антипример;
- отличие от похожей темы;
- типичная ошибка;
- мини-задача;
- ограничение.

Не делай вопросы на неважные детали.
Тема:
...
```

### 18.2. Проверить мой recall

```text
Я сначала ответил без источника.
Проверь мой ответ:
- что точно;
- что искажено;
- что пропущено;
- какие 2 вопроса добавить для повторения;
- какая одна мини-задача поможет применить знание.

Мой ответ:
...

Источник / эталон:
...
```

### 18.3. Найти оверлернинг

```text
Посмотри на список карточек/вопросов.
Найди, где я повторяю слишком легкое или дублирую один и тот же факт.
Предложи:
- что удалить;
- что объединить;
- что отложить;
- что заменить вопросом на применение.

Список:
...
```

### 18.4. Сделать successive relearning план

```text
Для темы ниже сделай план successive relearning.
Укажи:
- критерий успешного воспроизведения;
- 3-4 сессии с интервалами;
- что проверять на каждой сессии;
- когда остановиться;
- где добавить практическую задачу.

Тема:
...
Срок:
...
```

### 18.5. Сделать pause + recall протокол

```text
Я закончил учебный блок.
Помоги составить короткий pause + recall:
- что записать до паузы;
- сколько отдыхать;
- что не делать во время паузы;
- какие 3 вопроса задать после паузы;
- как назначить следующий повтор.

Тема блока:
...
```

## 19. Минимальная инструкция на каждый день

Если упростить до минимума:

```text
После учебного блока:
1. Закрыть источник.
2. Вспомнить 3 идеи.
3. Проверить.
4. Сделать паузу без контента.

Утром:
5. За 3-5 минут восстановить вчерашнее.

Через несколько дней:
6. Проверить тему другой формой.

Раз в неделю:
7. Удалить лишнее и отложить легкое.
```

## 20. Чек-лист перед завершением учебного блока

- Я закрыл источник и вспомнил сам?
- Я проверил точность?
- Я записал ошибку или пробел?
- Я сделал паузу без нового контента?
- Я знаю, когда вернусь к теме?
- Я понимаю, что не надо повторять сегодня?
- Я добавил вопрос на применение, если тема практическая?
- Я не превратил повторение в перечитывание?

## 21. Главные правила

1. Повторение начинается с памяти, а не с конспекта.
2. Вариативность нужна после базового понимания, а не вместо него.
3. Successive relearning применять к важному материалу, а не ко всему подряд.
4. Утренняя реактивация должна быть короткой.
5. Легкое не надо повторять много раз подряд.
6. Пауза после обучения полезна как гигиена, но не как магия.
7. Feedback защищает от закрепления ошибок.
8. Для навыков нужны задачи.
9. Система повторений должна удалять лишнее.
10. Лучший признак результата - самостоятельное воспроизведение и применение через время.

Короткая формула:

```text
Вспомнил.
Проверил.
Отдохнул.
Вернулся позже.
Проверил иначе.
Остановился вовремя.
```
