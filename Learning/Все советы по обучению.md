# Все советы по обучению: инструкция пользователя

## О чём эта инструкция

Эта инструкция собирает основные советы по эффективному обучению из локальных материалов: техники памяти, режим дня, работа с вниманием, конспекты, ИИ, Anki, Zettelkasten, ошибки, обратная связь, самообучение, онлайн-обучение и подготовка к экзаменам.

Главная идея:

```text
эффективное обучение - это не одна секретная техника,
а система коротких проверяемых циклов.
```

Рабочий цикл выглядит так:

```text
цель -> маленький блок -> попытка без подсказки -> ошибка или факт -> feedback -> исправление -> повтор позже -> применение
```

Если упростить ещё сильнее:

```text
не просто читать,
а вспоминать, делать, проверять, исправлять и возвращаться позже.
```

## Обновление 2026-06-01: что добавлено из новых материалов `4sort`

Новые заметки уточняют общий совет: эффективное обучение зависит не только от техник памяти, но и от того, как устроены паузы, состояние, формат источников и момент дня.

В практическую систему добавлены следующие поправки:

- retrieval practice работает лучше пассивного перечитывания даже в неидеальных условиях, но не отменяет интервальное повторение;
- паузы полезны, когда после них есть попытка вспомнить, а не просто возврат к чтению;
- оверлернинг лёгкого материала часто даёт меньше пользы, чем перенос следующего повторения на позже;
- утро можно использовать для короткой реактивации вчерашнего материала: сначала вспомнить, потом проверить;
- сложные задачи желательно ставить в личное окно ясности, а не обязательно в абстрактное "правильное утро";
- короткие видео лучше использовать как демонстрацию или вход, но не как основной способ учить сложную связанную тему;
- сон, движение и настроение стоит отслеживать как систему, без поиска одной магической нормы;
- кофеин может быть временной опорой после плохой ночи, но не заменяет сон и не должен становиться стратегией обучения.

## Главный принцип

Не считать обучение завершённым, пока не было проверяемого действия.

Пассивное обучение:

- прочитал;
- посмотрел видео;
- подчеркнул;
- сохранил ссылку;
- попросил ИИ объяснить;
- почувствовал, что "вроде понятно".

Проверяемое обучение:

- закрыл источник и воспроизвёл;
- объяснил своими словами;
- решил задачу;
- написал пример;
- получил ошибку;
- проверил по источнику;
- исправил;
- повторил через интервал;
- применил в другом контексте.

Главный вопрос:

```text
Что я теперь могу сделать без подсказки?
```

## Какие технологии обучения использовать

Ниже - список технологий без углубления: название и короткое описание.

### Ядро системы

| Технология                                          | Краткое описание                                                                                 |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Retrieval practice / практика извлечения            | Закрыть источник и попытаться вспомнить материал без подсказки.                                  |
| Active recall / активное воспроизведение            | То же ядро: отвечать, объяснять, восстанавливать, а не перечитывать.                             |
| Spaced repetition / интервальное повторение         | Возвращаться к материалу через интервалы: завтра, через несколько дней, через неделю.            |
| Distributed practice / распределённая практика      | Учиться несколькими короткими сессиями, а не одним длинным рывком.                               |
| Feedback loop / контур обратной связи               | Быстро получать сигнал: правильно, неправильно, где ошибка, что исправить.                       |
| Error-based learning / обучение на ошибках          | Ошибку использовать как данные: что ожидал, что вышло, почему модель была неверной.              |
| Deliberate practice / осознанная практика           | Тренировать конкретное слабое место с полной концентрацией и обратной связью.                    |
| Self-regulated learning / саморегулируемое обучение | Планировать, отслеживать прогресс, замечать ошибки и менять стратегию.                           |
| Проверяемый учебный цикл                            | Любой блок заканчивать конкретным выходом: ответ, задача, ошибка, исправление, следующий повтор. |

### Память и повторение

| Технология | Краткое описание |
|---|---|
| Anki | Программа для интервального повторения карточек. |
| FSRS | Планировщик повторений в Anki, который подбирает интервалы по истории ответов. |
| Flashcards / учебные карточки | Короткие вопросы и ответы для активного вспоминания. |
| Index cards / бумажные карточки | Физические карточки, которые заставляют сжимать мысль до главного. |
| Cloze cards / карточки с пропусками | Формат, где нужно восстановить скрытый элемент в предложении. |
| Карточки на различия | Проверяют, чем похожие понятия отличаются друг от друга. |
| Карточки из ошибок | Фиксируют типовую ошибку и правильный принцип. |
| Variable retrieval / вариативное извлечение | Проверять одну идею разными способами: определение, пример, задача, ошибка. |
| Successive relearning / последовательное переучивание | Повторять материал до устойчивого воспроизведения через несколько сессий. |
| Morning reactivation / утренняя реактивация | 3-5 минут восстановить вчерашний материал по памяти, затем свериться. |
| Overlearning control / контроль оверлернинга | Лёгкое не повторять много раз подряд; следующий полезный повтор часто должен быть позже. |
| Pause + recall / пауза с последующим припоминанием | После отдыха сначала вспомнить материал, а не сразу перечитывать. |

### Понимание и перенос

| Технология | Краткое описание |
|---|---|
| Interleaving / чередование | Перемешивать похожие типы задач, чтобы учиться выбирать правильный метод. |
| Worked examples / разобранные примеры | Сначала изучить готовое решение, потом постепенно решать самому. |
| Fading | Постепенно убирать подсказки из разобранного примера. |
| Self-explanation / самообъяснение | Объяснять себе, почему шаг работает и где можно ошибиться. |
| Learning by teaching / обучение через объяснение | Объяснить тему другому человеку или воображаемому ученику. |
| Generation effect / эффект генерации | Самостоятельно формулировать ответ, пример или правило. |
| Pre-testing / предварительное тестирование | До изучения попробовать ответить на вопросы, чтобы усилить внимание. |
| Transfer practice / практика переноса | Применить идею в новой задаче, а не только повторить знакомый пример. |
| Сенсорная обратная связь | Для речи и произношения: образец, попытка, запись себя, сравнение, точечная коррекция. |

| Технология | Краткое описание |
|---|---|

| Desirable difficulties / желательные трудности | Делать обучение достаточно трудным, но посильным. |
| Pomodoro | Работать короткими сфокусированными блоками с паузами. |
| Implementation intentions / планы "если - то" | Заранее задать действие для типовой ситуации. |
| Weekly review / недельный обзор | Раз в неделю смотреть, что работает, что убрать и что тренировать дальше. |

| First-step planning / первый шаг заранее | В конце дня или блока заранее определить первое действие следующего учебного блока. |

## Как лучше обучаться

### 1. Начинать с цели действия

Плохая цель:

```text
Изучить тему.
```

Лучше:

```text
После блока я должен объяснить принцип, решить одну задачу и назвать типичную ошибку.
```

Цель должна указывать не только материал, но и действие.

Вопросы:

- что я хочу уметь делать?
- где я это применю?
- какой минимальный результат покажет, что я продвинулся?
- как я проверю себя без подсказки?

### 2. Делить обучение на маленькие блоки

Один учебный блок - это не "весь курс".

Хороший блок:

- одна тема;
- один принцип;
- один тип задач;
- 20-60 минут;
- один проверяемый результат.

Примеры:

```text
Не "учить английский", а "10 фраз для описания прошлого опыта".
Не "изучать память", а "объяснить разницу между retrieval и rereading".
Не "учить программирование", а "написать один тест с fake dependency".
```

### 3. Сначала понять минимально, потом вспоминать

Активное вспоминание не заменяет первичное понимание.

Рабочая последовательность:

```text
коротко изучить -> закрыть источник -> вспомнить -> проверить -> исправить -> применить
```

Если материал совсем непонятен, сначала нужны:

- более простой пример;
- объяснение;
- схема;
- разобранная задача;
- словарь терминов;
- уменьшение сложности.

### 4. После каждого блока делать retrieval

Минимальный протокол:

```text
1. Закрыть источник.
2. За 3-5 минут записать главную мысль.
3. Восстановить 3-5 тезисов.
4. Привести пример.
5. Назвать ограничение.
6. Проверить по источнику.
7. Исправить одну ошибку.
```

Не спрашивать только:

```text
Я понял?
```

Спрашивать:

```text
Что я могу воспроизвести без подсказки?
Что я могу применить?
Где я ошибся?
Что повторить позже?
```

### 5. Делать задачу как можно раньше

Для навыков чтения мало.

После теории нужна маленькая попытка:

- решить задачу;
- написать пример;
- объяснить вслух;
- сделать мини-проект;
- найти ошибку;
- применить принцип к своему случаю.

Правило:

```text
Если навык должен работать в действии, обучение должно включать действие.
```

### 6. Получать обратную связь

Обратная связь может быть разной:

- правильный ответ;
- преподаватель;
- тест;
- компилятор;
- решение задачи;
- ИИ;
- сверка с источником;
- коллега;
- собственный журнал ошибок.

Важно не просто увидеть ответ, а понять расхождение:

```text
что я ожидал?
что получилось?
почему моя модель была неверной?
какой следующий пример это проверит?
```

### 7. Повторять через интервалы

Один раз понял - не значит удержал.

Простой график:

```text
сразу после блока
на следующий день
через 3-4 дня
через неделю
через 2-4 недели
```

Повтор должен быть активным:

```text
сначала вспомнить,
потом проверить.
```

### 8. Чередовать похожие задачи после базы

На первом этапе можно учить блоками.

Когда база появилась, полезно чередовать похожие задачи:

```text
A B C A C B
```

Это учит не только решать, но и выбирать метод.

Не стоит делать interleaving слишком рано, если базовые элементы ещё не различаются.

### 9. Вести простую систему следов

Нужны не идеальные заметки, а следы, которые помогают учиться:

- список тем;
- журнал ошибок;
- Anki-кандидаты;
- конспекты источников;
- Zettelkasten-заметки;
- недельный фокус;
- список открытых вопросов;
- мини-проекты.

Главный критерий:

```text
эта запись помогает сделать следующую проверяемую попытку?
```

## Режим дня для обучения

Ниже - не медицинский режим и не универсальная норма. Это практическая рамка, которую нужно подстраивать под работу, сон, здоровье и реальные обязанности.

### Утро

Подходит для:

- сложной новой темы;
- практических задач;
- письма;
- короткой реактивации материала после сна;
- планирования учебного дня.

Протокол:

```text
1. Проверить состояние: сон, энергия, тревога, отвлечения, ясность шага.
2. Не открывая конспект, за 3-5 минут восстановить вчерашнюю тему.
3. Свериться с источником и исправить одно искажение.
4. Выбрать одну главную учебную задачу.
5. Сделать 25-60 минут фокусной работы.
6. Закрыть источник и вспомнить главное.
7. Зафиксировать один вывод или ошибку.
```

Если утром мало времени:

```text
10 минут Anki или retrieval
5 минут план дня
1 маленький вопрос
```

Если утро объективно слабое из-за хронотипа, недосыпа или обязанностей, не нужно ломать систему. Перенеси самую сложную новую тему в своё устойчивое окно ясности, а утром оставь короткий retrieval, Anki, планирование или лёгкую обработку ошибок.

### День

Подходит для:

- учебных блоков средней сложности;
- повторов;
- задач;
- созвонов;
- обратной связи;
- чтения источников.

Протокол:

```text
1. Работать блоками 25-50 минут.
2. Между блоками делать 5-10 минут паузы.
3. Не смешивать тяжёлую учёбу с лентами, мессенджерами и фоном.
4. После блока делать короткое извлечение.
```

### Вечер

Подходит для:

- лёгкого повторения;
- Anki;
- конспектов;
- обработки ошибок;
- чтения без перегруза;
- подготовки следующего дня.

Не лучший момент для:

- героического добивания сложной темы после истощения;
- многочасовой зубрёжки вместо сна;
- просмотра бесконечных видео "ещё чуть-чуть".

Протокол:

```text
1. Повторить 10-20 минут.
2. Закрыть 1-2 открытых вопроса.
3. Записать следующий шаг на завтра.
4. Не начинать тяжёлый новый блок прямо перед сном, если он возбуждает и мешает отдыху.
```

### После учебного блока

Минимальная пауза:

```text
2-5 минут без нового контента.
```

Можно:

- посидеть в тишине;
- пройтись;
- закрыть глаза;
- записать главный вывод;
- ничего не смотреть.

Смысл: не забивать мозг новым потоком сразу после обработки.

Если была тяжёлая попытка вспомнить или решить задачу, пауза особенно уместна. Но после длинной паузы не начинай с перечитывания: сначала восстанови, что осталось в памяти, и только потом сверяйся.

### Сон

Сон - не бонус, а часть обучения.

Что учитывать:

- хронический недосып ухудшает внимание и память;
- ночь перед экзаменом не должна быть основной учебной сессией;
- после большого объёма новой информации восстановление особенно важно;
- лучше сократить позднюю зубрёжку, чем разрушить следующий день;
- личная норма сна может отличаться, но её нужно проверять по устойчивым паттернам, а не по одному удачному дню;
- кофеин может временно повысить бодрость, но не восстанавливает полноценную консолидацию, внимание и эмоциональную устойчивость после регулярного недосыпа.

Практическое правило:

```text
Если выбор между ещё часом пассивного чтения ночью и сном, часто лучше выбрать сон.
```

### Физическая активность

Физическая активность не должна восприниматься только как отвлечение от учёбы.

Рабочие варианты:

- короткая прогулка после сложного блока;
- лёгкая разминка перед учёбой;
- 5 минут движения при сонливости;
- регулярная нагрузка несколько раз в неделю;
- ходьба для обдумывания и повторения.

Смысл не в том, что движение магически обучает, а в том, что оно поддерживает состояние, внимание и восстановление.

## Пример дневного режима

### Вариант для обычного дня

```text
Утро:
- 10 минут повторения / Anki.
- 45 минут главной учебной задачи.
- 5 минут retrieval.

День:
- 1-2 блока по 25-50 минут.
- После каждого блока короткая проверка.
- Один практический шаг или задача.

Вечер:
- 15 минут повторения.
- Разбор одной ошибки.
- Записать следующий шаг на завтра.
```

### Вариант при занятости

```text
10 минут утром:
- вспомнить вчерашнюю тему;
- ответить на 3 вопроса.

25 минут днём или вечером:
- один маленький учебный блок;
- одна попытка;
- один вывод.

5 минут перед сном:
- записать следующий шаг;
- не открывать новый поток материалов.
```

### Вариант на выходной

```text
Блок 1:
- новая тема и разобранный пример.

Пауза:
- движение / отдых.

Блок 2:
- самостоятельная задача.

Пауза.

Блок 3:
- разбор ошибок, карточки, заметки, план повторения.
```

Не превращать выходной в 8 часов пассивных видео.

## Недельная система

Раз в неделю делать короткую калибровку на 20-30 минут.

Вопросы:

```text
1. Что я изучал?
2. Что теперь могу сделать без подсказки?
3. Где были ошибки?
4. Какие ошибки повторяются?
5. Что требует Anki или повторения?
6. Что требует задач и feedback?
7. Что нужно убрать из плана?
8. Какая одна компетенция будет главной на следующей неделе?
```

Главный результат недельного обзора:

```text
не отчёт,
а решение: что продолжать, что убрать, что тренировать дальше.
```

Минимальная недельная структура:

```text
1 главная компетенция
1 мини-проект или задача
1 список ошибок
1 список повторений
1 вещь, которую убрать
```

## Как обучаться в разных сценариях

### Если изучаешь новую тему с нуля

```text
1. Сделать карту темы: что входит, какие базовые понятия, какой результат нужен.
2. Найти 1-2 основных источника, не 20.
3. Разобрать один простой пример.
4. Закрыть источник и объяснить базовую модель.
5. Сделать маленькую задачу.
6. Записать ошибки.
7. Повторить через день.
```

Главное: сначала база, потом вариативность.

### Если тема сложная и пугает

```text
1. Уменьшить блок.
2. Найти worked example.
3. Выписать термины.
4. Сформулировать один вопрос.
5. Сделать 10-25 минут, а не ждать идеального дня.
6. После блока записать, что стало понятнее.
```

Не пытаться сразу понять всю область.

### Если нужно быстро подготовиться к экзамену

```text
1. Составить список вопросов.
2. Отвечать без конспекта.
3. Отмечать слабые темы.
4. Делать интервальные возвраты.
5. Тренировать формат экзамена.
6. Спать перед проверкой.
```

Что не делать:

- только перечитывать;
- учить всю ночь;
- откладывать тестирование до последнего дня;
- заниматься только лёгкими темами.

### Если учишься онлайн

```text
1. Сделать расписание.
2. Разбить курс на недели.
3. После каждого видео закрывать источник и вспоминать.
4. Делать задания, а не только смотреть.
5. Поддерживать социальное присутствие: вопросы, обсуждения, группа, наставник.
6. Раз в неделю проверять, что реально сделано.
```

Онлайн-обучение требует больше саморегуляции, потому что внешней структуры меньше.

### Если смотришь обучающее видео

```text
1. Перед видео сформулировать 2-3 вопроса.
2. Смотреть один связный фрагмент, а не цепочку случайных роликов.
3. Не конспектировать всё подряд.
4. После фрагмента закрыть видео.
5. Восстановить главную мысль, пример и действие.
6. Сделать одну задачу или пример.
7. Вернуться только к непонятному месту.
8. Перед сложным блоком не разогреваться лентой коротких видео.
```

Видео без пауз часто создаёт иллюзию понимания.

Короткое видео допустимо как демонстрация одного действия, но не должно быть основной средой для сложной темы, где нужна связная линия.

### Если читаешь книгу или статью

```text
1. Перед чтением понять цель.
2. После раздела закрыть текст.
3. Записать главную мысль своими словами.
4. Сформулировать 1-3 вопроса.
5. Сделать заметку источника.
6. Вынести 1-3 постоянные Zettelkasten-заметки.
7. При необходимости сделать Anki-карточки.
```

Не нужно превращать каждый абзац в заметку.

### Если учишь практический навык

```text
1. Найти простое правильное выполнение.
2. Разобрать пример.
3. Повторить с подсказкой.
4. Повторить без подсказки.
5. Получить feedback.
6. Исправить.
7. Повторить в похожем, но не идентичном контексте.
```

Навык требует действия. Конспект не заменяет попытку.

### Если используешь ИИ

Хороший сценарий:

```text
1. Сам сформулировал прогноз.
2. Сам сделал попытку.
3. Получил ошибку или затруднение.
4. Попросил ИИ объяснить расхождение.
5. Попросил не готовое решение, а вопрос или подсказку.
6. Сам сделал повторную задачу.
```

Плохой сценарий:

```text
ИИ решил -> я прочитал -> мне кажется, что понял.
```

ИИ должен быть тьютором, редактором, проверяющим и генератором задач, а не заменой мышления.

## Что особенно важно учесть

### 1. Понимание и знакомость - разные вещи

Материал может казаться знакомым при повторном чтении, но быть недоступным без подсказки.

Проверка:

```text
могу ли я объяснить это без источника?
```

### 2. Система должна быть маленькой

Слишком много методов перегружает.

Лучше:

```text
retrieval + задачи + feedback + интервалы + недельный обзор
```

Чем:

```text
20 приложений, 8 шаблонов, 15 типов заметок и ни одной завершённой попытки.
```

### 3. Ошибки нужно фиксировать без самообвинения

Ошибка - это не доказательство неспособности.

Правильный формат:

```text
что ожидал?
что вышло?
почему?
какой принцип?
какой следующий пример?
```

### 4. Усталость меняет качество обучения

Если два и более параметра провалены:

- сон;
- тревога;
- энергия;
- отвлечения;
- ясность следующего шага;

то не начинать с самой тяжёлой темы. Сначала стабилизировать среду: закрыть лишнее, записать один шаг, подвигаться, уменьшить задачу.

Но это не должно становиться избеганием. Цель настройки состояния - быстрее вернуться к действию.

### 5. Новые материалы нужно ограничивать

Одна из частых ошибок самообучения - распыление на десятки курсов и книг.

Правило:

```text
1-2 основных источника на тему,
дальше - практика, проверка и расширение.
```

### 6. Сложность должна быть посильной

Слишком легко - нет роста.

Слишком трудно - угадывание, тревога и перегруз.

Рабочая зона:

```text
есть усилие,
есть шанс ошибки,
но можно проверить и исправить.
```

### 7. Повторение должно вести к применению

Anki, конспект, Obsidian и Zettelkasten полезны, если ведут к:

- вопросу;
- задаче;
- объяснению;
- проекту;
- исправлению ошибки;
- повторной попытке.

Если они только копят материалы, они начинают мешать.

### 8. Нужны разные режимы мышления

Фокусный режим:

- читать;
- решать;
- писать;
- проверять.

Рассеянный режим:

- прогулка;
- пауза;
- сон;
- свободное обдумывание.

Не все проблемы решаются давлением. Иногда после честной попытки нужна пауза.

### 9. Время блока влияет на качество, но не является магией

Хронотип и время суток могут влиять на внимание. Практичный вывод не в том, чтобы объявить себя "совой" или "жаворонком", а в том, чтобы 7-14 дней отследить, когда чаще получается сложное мышление.

Правило:

```text
сложное - в окно ясности,
рутина и лёгкое повторение - в слабое окно.
```

### 10. Оверлернинг нужно дозировать

Если факт, термин или карточка уже легко вспоминаются, дополнительные повторы подряд часто дают меньше пользы, чем повтор позже.

Лучше:

```text
легко вспомнил -> проверил -> отложил -> вернулся через интервал
```

Для навыков иначе: после первого правильного выполнения нужно повторить в разных условиях, чтобы закрепить действие, но не автоматизировать ошибку.

### 11. Фрагментированный контент ухудшает связность

Короткие видео, ленты и быстрые переключения могут мешать удерживать непрерывную смысловую линию. Для сложной темы нужен один источник, фокусный блок и самостоятельный выход после него.

### 12. Сон, движение и настроение работают как система

Если внимание проседает, не всегда проблема в мотивации. Проверь связку:

```text
сон -> энергия -> движение -> тревога -> качество учебного блока
```

Это не диагностика и не медицинский вывод, а способ не планировать тяжёлую учёбу так, будто состояние всегда одинаковое.

## Чему не стоит уделять слишком много внимания

### 1. Поиску идеального курса

Часто лучше пройти один нормальный источник с задачами, чем бесконечно выбирать лучший.

### 2. Красивым конспектам

Красота конспекта не равна пониманию.

Полезнее:

```text
короткий вывод + вопрос + задача + ошибка
```

### 3. Длинному перечитыванию

Перечитывание полезно для уточнения, но плохо как главный способ запоминания.

Сначала:

```text
вспомнить,
потом перечитать.
```

### 4. Подсветке текста

Маркер может помочь навигации, но сам по себе не создаёт знания.

### 5. Сохранению ссылок

Сохранённая ссылка - не обработанный материал.

Минимум:

```text
почему я это сохранил?
какой вывод?
что сделать?
```

### 6. Настройке приложений

Obsidian, Anki, Notion, плагины, графы, шаблоны полезны только как инфраструктура.

Если настройка стала главным занятием, система мешает обучению.

### 7. ИИ-ответам без попытки

Чем раньше ИИ дал готовое решение, тем меньше тренировки получил ученик.

### 8. Мотивационным ритуалам

Мотивация помогает начать, но навык строят:

- расписание;
- попытки;
- feedback;
- повторение;
- восстановление.

### 9. Многозадачности

Переключения между темами, окнами и лентами снижают качество глубокого обучения.

### 10. Количеству часов как главной метрике

Важнее:

- сколько было самостоятельных попыток;
- сколько ошибок разобрано;
- что повторено через интервал;
- что теперь можно сделать без подсказки.

### 11. Идеальным "золотым часам"

Полезно учитывать окно ясности, но вредно превращать хронотип в жёсткий ярлык. Смотри на повторяющиеся данные по себе, а не на универсальную формулу.

### 12. Бесконечному добиванию лёгкого материала

Лёгкая карточка не должна съедать сессию. Если ответ уже быстрый и точный, перенеси следующий повтор.

### 13. Коротким видео как основному способу обучения

Они удобны для входа и демонстрации, но сложная тема требует связного источника, пауз и проверки памяти.

### 14. Кофеину как решению проблемы сна

Кофеин может помочь пережить плохую ночь, но не делает день физиологически восстановленным. После недосыпа лучше уменьшить риск ошибок и восстановить сон.

## Что считать прогрессом

Хорошие признаки:

- могу объяснить без источника;
- могу решить задачу;
- быстрее замечаю ошибки;
- меньше перечитываю;
- лучше отличаю похожие понятия;
- могу применить идею в новой ситуации;
- знаю, что повторить;
- могу назвать следующий шаг;
- система не перегружает.

Слабые признаки:

- много просмотренных видео;
- много сохранённых ссылок;
- много красивых заметок;
- длинный список курсов;
- ощущение "я сегодня много занимался";
- много ИИ-объяснений без своих попыток.

## Минимальная система обучения

Если нужна простая версия, достаточно этого:

```text
1. Одна главная тема на неделю.
2. Один основной источник.
3. Каждый день 25-60 минут.
4. После блока - активное вспоминание.
5. Каждый день - одна маленькая задача.
6. Ошибки - в журнал.
7. Важные различия - в Anki.
8. Главные выводы - в Obsidian/Zettelkasten.
9. Через интервалы - повтор.
10. Раз в неделю - обзор и удаление лишнего.
```

## Подробный учебный блок на 60 минут

```text
0-5 минут:
Проверить состояние и выбрать один результат.

5-20 минут:
Изучить короткий фрагмент или разобрать пример.

20-35 минут:
Сделать самостоятельную попытку.

35-45 минут:
Получить feedback и исправить одну ошибку.

45-52 минуты:
Сделать похожую повторную попытку.

52-57 минут:
Записать вывод, ошибку или карточку.

57-60 минут:
Запланировать повтор или следующий шаг.
```

## Короткий блок на 25 минут

```text
1. 2 минуты - выбрать вопрос.
2. 10 минут - изучить или вспомнить.
3. 8 минут - сделать попытку.
4. 3 минуты - проверить.
5. 2 минуты - записать следующий шаг.
```

## 7-дневный старт

### День 1. Выбрать одну тему

Не пять направлений. Одну тему и один измеримый результат.

### День 2. Сделать карту темы

Записать:

- что входит в тему;
- какие базовые понятия;
- какие задачи нужно уметь делать;
- что будет первым блоком.

### День 3. Внедрить retrieval

После каждого блока закрывать источник и вспоминать.

### День 4. Добавить feedback

Найти способ проверки: тест, задача, ответ, преподаватель, ИИ, документация.

### День 5. Добавить интервалы

Повторить материал первого и третьего дня без подсказки.

### День 6. Сделать мини-проект или кейс

Применить несколько элементов вместе.

### День 7. Провести недельную калибровку

Ответить:

```text
что теперь могу сделать?
где ошибался?
что повторить?
что убрать?
какой фокус следующей недели?
```

```md
# Учебный блок

```

```md
# Недельный обзор обучения

## Что убрать

## Главная компетенция следующей недели

## Один 7-дневный эксперимент
```

```md
# Перед учебным блоком

Отвлечения: закрыты / открыты
Следующий шаг: ясен / неясен

Решение:
- тяжёлый блок
- лёгкий блок
- повторение
- 5 минут движения и вернуться
- уменьшить задачу
```

## Итоговая формула

Эффективное обучение держится на нескольких опорах:

```text
ясная цель,
маленький блок,
```
